오늘은 소명처리시스템의 SSO 관련 commit 5개를 마무지며 bit-scalp 봇의 testnet-to-live 마이그레이션을 진행하고, Obsidian Vault에서 claude.md 계열 파일들을 체계화하는 데 Claude를 집중적으로 활용한 하루였다. 단순 지시가 아닌 "순서대로 정확히"나 "기존 기능 확인"처럼 구체적 컨텍스트를 먼저 주고 시작하니 AI의 응답 정확도가 올라갔고, 그 덕에 디버깅 시간을 단축할 수 있었다.

## 소명처리시스템: 단계별 commit 지시의 효과
소명처리시스템에서 SSO 테스트 후 .gitignore, 사용자 테넌트 분리 MD 문서들(001~004), 리프레시 토큰 관련 파이썬 스크립트(059, 060), 그리고 AcceptInvitePage.tsx 같은 파일들을 commit 5개로 나누어 진행했다. Claude에게 "SSO 직접 한 번 테스트 후 문제 없으면 commit 5 진행"이라고 단순히 던지면 항상 "테스트 방법부터 설명해달라"고 되물었지만, 오늘은 미리 "grep으로 파일 확인, docker 환경 ls/cd 후 git pwd 상태 점검, echo/mkdir로 임시 디렉터리 만들고 순서대로 commit"처럼 의도와 필요한 명령 카테고리를 나열해 주니 한 번에 실행 가능한 체크리스트를 뱉어냈다. 특히 MD 문서와 코드 파일을 섞인 상태에서 "테넌트 분리부터 SSO 통합 순으로 그룹화"라고 컨텍스트를 주자 논리적 분할을 제안해 20분 만에 git 히스토리가 깔끔해졌다. 반대로 처음에 "전체 commit 해"라고 광범위하게 물으니 파일 간 의존성 놓치고 헤매서, 배운 점은 "작업 단계를 3~5개로 쪼개서 프롬프트에 미리 매핑"하는 거다. AI는 세부 실행보단 구조화에 강하니, 인간이 grep/ls 같은 진단을 먼저 하고 결과를 피드백으로 주는 하이브리드 방식이 최적.

## bit-scalp: 로그 분석과 마이그레이션 순서 고정화
bit-scalp에서 대시보드 웹소켓 문제(P&L, 수익률 동기화 안 됨)와 텔레그램 발송, 거래소 보호 기능 확인을 요청했다. "현재가 상황 불러오지 못함, 동기화 안 되는 값들 확인해"라고 로그 스니펫 없이 물으니 Claude가 "일반적 웹소켓 재연결 로직"만 제안해 헛다리를 쳤지만, app.py, binance_client.py, executor.py 등 편집 파일 목록과 "기존과 동일하게 한 거 맞지? USDT에 남아 있는지, 매매 시작 후 notifier.py 발송 체크"처럼 구체적 의심 포인트를 나열하니 "curl로 API 엔드포인트 테스트, tail로 로그 모니터링, pgrep/nohup으로 프로세스 상태 확인"하는 진단 순서를 바로 줬다. 특히 일일 자동 최적화와 testnet-to-live 마이그레이션에서 "STEP 1: DB environment 컬럼 추가 + 기존 데이터 testnet 라벨링, 아래 Python 스크립트 Bash로 실행"이라고 단계 번호와 스크립트 의도를 명확히 프롬프트에 넣었더니, models.py/repository.py 같은 파일 수정 지점을 정확히 짚고 python3 실행 가이드를 출력했다. 어제 로그 분석(Bash로 날짜 계산 후 로그 읽기, 거래 목록 추출)도 "logs/bitscalp_YYYY-MM-DD.log에서 Opening/C 추출" 컨텍스트 주니 패턴 매칭 제안이 딱 들어맞아 nohup/pkill로 프로세스 재시작까지 커버. 안 통한 건 "한 번에 모든 체크" 시도 – sleep/tail 중복으로 응답이 길어지니, "하나씩 확인 후 다음 단계"로 끊으니 40% 효율 up. AI의 한계는 실시간 curl/pgrep 실행이 안 되니, 내가 직접 검증 후 "결과: XX" 피드백 루프를 도는 게 핵심.

## Obsidian Vault: 파일 관계성 시각화 프롬프트 실험
Downloads와 Obsidian Vault에서 claude.md, memory.md, agents.md 같은 파일들의 프로젝트 간 연결을 정리하려 했다. "claude.md가 전역/로컬에 있을 때 프로젝트 적용 여부"부터 물었는데, 단순 질문으로 "둘 다 독립적"이라고 일반 답만 오다 "모든 프로젝트의 claude.md와 메모리 관련 .md를 Obsidian에 정리, 관계성 연결하고 시각화"라고 프로젝트 목표를 주고 "너 스스로 mcp 만들어도 되고, 같은 프로젝트 파일끼리 그룹화, b+추후 c 진행" 옵션을 나열하니 sync.py 생성과 find/echo/mkdir/chmod/rm 명령으로 폴더 구조화 스크립트를 제안했다. 잘 통한 건 "관계성 연결" 컨텍스트 – AI가 Obsidian 그래프 뷰를 텍스트 다이어그램으로 그려주며 "프로젝트 A의 claude.md가 B의 memory.md와 링크"처럼 유기적 매핑을 해줘 초기 설계가 빨라졌다. 안 통한 건 "편집한 걸 원본에도 적용" 같은 모호 지시 – AI가 무한 루프 제안으로 빠지니 "편집은 Obsidian 한정, 원본 동기화 불필요"라고 명확히 제한하자 안정적. 여기서 노하우는 AI에게 "다양한 방법 고려"라고 열어두되, "ls/find로 파일 목록 먼저 스캔" 결과를 프롬프트에 넣어 구체화하는 거. python3/chmod로 스크립트 실행 후 검증하니 분업이 명확해졌다.
