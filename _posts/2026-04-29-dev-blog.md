오늘은 여러 프로젝트를 오가며 AI의 모델 스위칭과 프롬프팅 세밀화에 집중한 하루였다. joyuno.github.io의 개발일지 생성을 최적화하면서부터 시작해 소명처리시스템의 스키마 설계, bit-scalp의 자동화 루틴까지, 단순 요청이 아닌 "반복 테스트 + 컨텍스트 보강" 패턴으로 AI를 다루니 효율이 올라갔다.

## 모델 변경과 시각화 프롬프트로 생성 품질 끌어올리기
joyuno.github.io에서 28일자 개발일지 생성이 중간에 끊긴 문제를 해결하려고, 먼저 기존 프롬프트를 분석한 뒤 "코드나 SVG로 시각적으로 표현해 개발 피드백이 직관적으로 보이게 최적화"하라고 지시했다. 그런데 Claude 기본으로는 토큰 한계로 희석된 결과가 나와서, nvidia/nemotron-3-nano-omni-30b-a3b-reasoning과 x-ai/grok-4.1-fast로 모델을 바꿔 테스트. Grok으로 재생성하니 로그 세션이 덜 희석되고 시각 요소가 잘 반영됐다. 잘 통한 점은 "max_token과 모델을 싹 갈아엎자"처럼 과감한 스위칭이었고, 안 통한 건 초기 프롬프트에서 시각화 지시가 모호해 텍스트 위주로 나왔다는 거. 배운 점: 모델별 강점(예: Grok의 빠른 생성)을 미리 비교 테스트하고, 프롬프트에 "SVG 다이어그램으로 컴포넌트 흐름 표현"처럼 구체적 예시를 넣자. config.json과 summarize.py 수정까지 1시간 만에 끝났다.

## Q&A 세션으로 복잡한 설계 결정 가속화
소명처리시스템에서 스키마와 UI 설계를 q1부터 q25까지 순차 질의응답으로 풀었다. "티켓당 한 유저만 들어가도록 시나리오 설정"이나 "수동 생성 user는 별도 테이블로"처럼 회의 결과를 미리 공유하고, 각 q에 "a/b/c 중 선택" 형식으로 물으니 AI가 일관된 제안을 줬다. 세션이 길어져 컨텍스트가 끊길 때 summary를 활용해 이어갔는데, 이게 잘 통했다. 반면 POST API 500 에러(Internal Server Error)가 반복되니 AI에게 로그만 던졌더니 "일반론"만 나와 직접 grep과 tail로 원인 추적 중. 안 통한 건 한 번에 여러 q를 묶으니 응답이 산만해진 점 – 1~2개씩 끊어 물으니 정확도 2배. LoginPage.tsx나 055_org_sync_schema.py 같은 파일 편집에서 AI 제안을 검증하며 인간이 최종 스키마 조정. 노하우: 비즈니스 시나리오를 "회의 결과: ~"로 프롬프트 앞에 박아넣으면 hallucination 줄었다.

## 자동화 루틴 등록과 피드백 루프 강화
bit-scalp에서 "현재 로직 문제점 판단하고, 매일 00:00 한국시간에 피드백/알고 업데이트 확인"을 요청했는데, /schedule 루틴 등록이 안 돼 반복 지시해야 했다. "strategy대로 진행하고 /schedule claude 루틴 등록"처럼 명시적 반복이 통했고, 어제 로그 분석(Bash로 날짜 계산 후 tail/grep)은 AI에게 "거래 목록 추출 항목: Opening/C" 프롬프트로 넘겨 MD 정리까지. ficeptterminal도 API 오류로 중지됐을 때 "이전 진행 단계부터 다시"라고 하면 바로 재개. 잘 안 통한 건 스케줄 지시가 모호해 "너 스스로 피드백" 대신 "매일 한국시간 00:00에 /schedule로 수행"으로 고치니 안정적. settings.py의 max_hold_minutes 연장(1440→2880)처럼 세부 수정은 AI가 제안하고 내가 python3 실행 검증. 배운 점: 자동화는 "작업 디렉터리: ~" 컨텍스트 + "정확히 5개 구현" 숫자 제한으로 오류 줄임.

## 컨텍스트 엔지니어링 'pulse' 고도화 제안
하네스에서 pulse(새 컨텍스트 엔지니어링 + 멀티에이전트)를 "더 간략하고 핵심적으로 고도화 방안 제안"하라고 했고, "제안 1+3+5만 묶어 통합 설계"로 프롬프트 좁히니 기존 프레임워크 변질 없이 구체적 아이디어가 나왔다. 사주 프로젝트에서도 "LLM 메모리 개인별 유지, 공유 안 되게" 지시가 잘 먹혔고, admin_dashboard_screen.dart 같은 UI 생성에서 "결제 API 건너뛰고 예시로 pass"라고 하면 프로토타입 빨리 뽑음. 안 통한 건 광범위 제안 시 "너무 큰 프레임워크로 변질" 경고를 안 넣어 산만해진 점.
