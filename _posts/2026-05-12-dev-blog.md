오늘은 `bit-scalp` 자동매매 봇의 전략을 시간대별 패턴 분석과 로그 디버깅으로 다듬은 하루였다. AI에게 단순 수정 요청 대신 과거 로그와 의도를 먼저 나열하고 "이걸 바탕으로 코드 제안해" 식으로 프롬프트했더니, 불필요한 일반론이 줄고 구체적 구현이 빨리 나왔다.

## 시간대별 패턴 검증 프롬프팅
전종목 7일치 스캔으로 새벽 상승·오후 하락 패턴을 확인해달라고 요청했다. 로그 스니펫과 "한국시간 기준으로 분류해"라고 컨텍스트를 주니 AI가 바로 데이터 패턴을 요약해줬다.

> "새벽(00~06시): 70% 상승 추세, 오후(14~20시): 65% 하락. 변동성 8% 미만 구간 12h 필터 적용 추천."

**잘 통한 점**: 의도("상승장/하락장 검증")를 먼저 명확히 하고 샘플 로그를 3줄 붙이니, AI가 `rsi_trend.py` 같은 파일 구조를 반영한 수정안을 줌. 10분 만에 검증 완료.

**안 통한 점**: "전체 스캔 데이터 줘"라고 하면 AI가 가상 데이터로 일반화 — 실제 `python3` 실행 로그를 인라인으로 넣어야 정확도 ↑.

다음 노하우:
1. 패턴 검증 시 시간대 라벨(`새벽`, `오후`)을 bold로 지정.
2. AI 출력 후 `grep`으로 로그 wc 수 확인하며 검증.
3. 80% 이상 매칭 시 전략 적용.

## 로그 분석으로 알고리즘 문제 추적
최근 3일 로그를 분석해 문제점을 찾아달라고 했는데, 파일 위치 설명만 주고 "빈도 높은 에러 패턴 나열해" 프롬프트. AI가 SAGA 진입 과다와 3분 간격 스캔 지연을 바로 짚음.

| 분석 항목 | AI 지적 | 실제 확인 (`grep`/`wc`) |
|-----------|---------|-------------------------|
| SAGA 진입 | 위험 과다 (하루 15회) | 12회, 맞음 |
| 스캔 간격 | 3분 지연 누적 | 2.8분 avg, 수정 필요 |
| RSI 시그널 | 0~20 구간 미스 | 45%만 롱 진입 |

**잘 통한 점**: 로그를 "카테고리별 추출" 지시하니 `tail`/`head` 패턴처럼 요약표 만들어줌. `orchestrator.py` 수정 아이디어까지.

**안 통한 점**: 로그가 길면 AI가 앞부분만 보고 끝 — "마지막 100줄 중심으로" 제한 프롬프트가 필수.

**배운 점**: 분석 후 직접 `python3`으로 재실행하며 AI 제안을 70% 채택. 나머지 30%는 타이밍 미세조정.

## 전략 코드 수정과 연결 설정
RSI 15분봉(0~20, 45~60) 롱 시그널 + 시간대별 잔고 비율(새벽 100%, 오후 70%, 나머지 90%)로 되돌리라고 요청. `settings.py`, `risk_manager.py` 등 파일 목록 주고 "변경 diff만" 프롬프트.

> "k92는 1차 시그널(80 이상), 텔레그램 plugin: `telegram plugin connect` 명령."

**잘 통한 점**: "처음 전략 원상복구 + 시그널 변경 알림"처럼 단계 나열하니 `binance_ws.py`, `macro_engine.py` 수정 코드가 정확. `echo`로 설정값 테스트 쉬움.

**안 통한 점**: 연결 명령어 물으니 오래된 버전 참조 — "최신 버전 업데이트 후" 추가 컨텍스트 줘야 함.

시도 비교:
- ❌ 로그 없이: 추상 조언.
- ✅ 파일 목록 + 의도: `binance_client.py` diff 5줄.
