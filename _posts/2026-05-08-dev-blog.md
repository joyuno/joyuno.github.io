오늘은 여러 프로젝트의 UI 수정부터 트레이딩 봇 리팩터링, 워크플로우 최적화까지 `Claude`를 활용해 산발적으로 작업한 하루였다. 단순 "이거 고쳐줘" 대신 **목표 중심 지시 + 세부 의도 설명**으로 프롬프팅하니 응답이 더 실용적이었지만, 컨텍스트 누락 시 반복 질문을 유발했다.

## 웹 UI 수정에서의 프롬프팅 효과
포트폴리오와 옵시디언 정리에서 `index.html`, `book.html` 등 파일 수정 요청이 잦았다. 이전엔 파일 내용 전체 던지면 AI가 과도한 변경을 제안했지만, 오늘은 **변경 지점만 나열 + 이미지 참조**로 좁히니 정확도가 올랐다.

> "contact에 blog 란 추가하고 joyuno.github.io 링크 걸어줘. 헤더에도 blog 버튼 추가. APM · OTel · AI 추론 → APM · OTel · AI 어시스턴트로 변경."

**잘 통한 점**: 이미지 첨부([Image #3]) 후 "github email 옆에 blog 버튼" 지시로 즉시 맞춤 HTML 생성. `grep`으로 확인 후 1회 수정으로 끝.
**안 통한 점**: ai-tech-digest 제목/이미지 불일치 버그 설명 시 "최근 이틀"처럼 모호하면 AI가 전체 스캔 대신 일반론만. 

| 접근법 | 결과 | 시간 절감 |
|--------|------|-----------|
| 전체 파일 던지기 | 과도 변경 | - |
| 지점+이미지 | 정확 수정 | +20분 |
| 버그 세부 로그 | 반복 질문 | ❌ |

**노하우**: 다음엔 `default.html` 같은 파일명 + "제목 번역 이상, 검색 후보 안 나옴" 리스트로 프롬프트 시작. 검증은 `curl`로 즉시.

## 트레이딩 봇 리팩터링 지시
`bit-scalp`와 `ficeptterminal`에서 포지션 슬롯 3개화, 보고 주기 4시간 변경, RegimeDetector vs. ensemble 결정 요청. AI에게 **현재 로직 설명 + 목표 수식** 주니 코드 스니펫이 바로 나왔다.

> "1포지션 → 3슬롯 동시. 잔고를 remaining_slots로 균등 분배. zecusdt PnL -10% 청산 확인."

**잘 통한 점**: `orchestrator.py`, `risk_manager.py` 리팩터링 시 "weight 비율 표시 + 청산 로직" 예시 주자 `python` 실행 전 검증 OK. 퀀트 전문가 관점으로 "ensemble" 추천 정확.
**안 통한 점**: KIS API 키 위치 물으니 "2.0+ 향상" 지시 무시하고 기본 답변. 세션 컨텍스트 길면 잊음.

시도 순서:
1. 로직 질문 → 확인.
2. "너가 결정해" → 전문가 시뮬.
3. 코드 생성 → `tail`, `pgrep`으로 로컬 테스트.

**배운 점**: 리팩터링 전 `mean_reversion_kr.py` 현재 상태 요약부터. AI 코드 70% 채택, 나머지 `pkill` 후 직접 디버그.

`하네스`, `joyuno.github.io`에서 pulse 이름 변경 추천(`ezimplan`), omc 의존성 질문, plugin 최적화. **Q1~Q4 태스크 리스트**로 프롬프트 주니 단계별 실행.

> "pulse → ezimplan으로 싹 바꿔. CLAUDE.md, plugin.json, git repo rename. 최적화 제안: pulse-dna/immunity 통합."

**잘 통한 점**: `git`, `gh` 명령 카테고리 + "session.md 제외 commit"으로 안전 제안. `marketplace.json` remove 정확.
**안 통한 점**: "omc 전역 md 필수?" 질문에 장황 설명. 의도 "ralph-loop만 사용" 미리 주니 워크플로우 수정 플랜 나옴.

| 옵션 | 추천 이유 | 채택 |

**노하우**: 이름 추천 시 "유쾌한데 실용적" 기준 명시. 변경 후 `find`, `sed`로 전체 grep 검증.
