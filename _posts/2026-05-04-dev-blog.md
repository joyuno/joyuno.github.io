오늘은 소명처리시스템의 테넌트 동기화와 로그프레소 티켓 라우팅 문제를 Claude와 함께 파헤치고, 드롭쉬핑자동화 프로젝트에서 보안 강화와 AliExpress 전환을 추진한 하루였다. 여러 프로젝트를 오가며 AI를 썼는데, 복잡한 비즈니스 로직에서 "단계별 질문" 접근이 가장 효과적이었다.

## 소명처리시스템: 한 번에 여러 이슈 던지면 엉키는 법
테넌트 관리 페이지에서 인원수 동기화가 안 되고, 로그프레소 티켓이 일부만 가져와지는 문제를 Claude에게 넘겼다. 처음엔 "테넌트 인원수 0명으로 동기화 안 됨 + 티켓 전체 안 가져옴 + SIEM 부모-자식 설정"을 한 프롬프트에 다 넣었더니, 응답이 산만해져서 "tr-box 로그와 임직원 매핑" 부분만 반복적으로 설명했다. 그래서 1단계씩 쪼개서 물어봤다: 먼저 "인원수 동기화 원인 진단"으로 org_sync.py와 TenantStatsPage.tsx를 공유하니, "임직원 수로 매핑해야 하니 사용자 수가 아닌 emp_name 기준으로 변경"이라고 정확히 지적. 다음으로 "_site 빈 티켓까지 가져와서 email로 라우팅" 요청 시, 002-ticket-routing-by-email.md처럼 마크다운으로 로직을 미리 스케치해 주니 Claude가 그걸 기반으로 admin.py와 auth.py 수정 코드를 바로 제안했다. 

잘 통한 점: 로그 진단 스크립트(_lp_diag.sh 등) 결과를 텍스트로 추출해 "이 출력에서 원인 추적"이라고 지정하니, grep 기반 패턴 분석이 빨라졌다. 안 통한 점: 서버 적용 시 "34:8443에 배포" 지시를 넣었는데 모델이 보안 문맥을 무시하고 가정만 해서 직접 검증해야 했다. 노하우: AI는 "현재 상태 + 다음 한 걸음"으로 유도하면 인간 검증 부담 줄음. 결과적으로 티켓 UI에 사이트 컬럼 추가와 operator-to-guest 변경까지 commit까지 갔다.

## 드롭쉬핑자동화: 보안부터 사업 전략까지 컨텍스트 전환
드롭쉬핑에서 "사이트 보안 철저히 보완 후 journal 페이지 완성 + 판매 물품 인기 검증 + 리뷰 기능"을 요청했다. 보안 부분은 "NestJS rate limiting 추가"로 구체적으로 물으니 app.module.ts와 main.ts에 @nestjs/throttler 통합 코드를 바로 줬다. 하지만 "물품이 정말 잘 팔릴까? K-Beauty로 해외 타겟"처럼 사업 판단을 섞으니 Claude가 "CJ에서 AliExpress로 전환 추천" 하며 sourcing.service.ts와 trend-crawler.service.ts를 제안했지만, 구체적 트렌드 데이터 없이 일반론으로 끝났다. 그래서 ALIEXPRESS_SETUP.md에 "한국 제품 드랍쉬핑 파이프라인 구조"를 먼저 그려 공유하니, configuration.ts 수정과 cron 기반 크롤링을 세밀하게 다듬었다.

잘 통한 점: 파일 목록(CLAUDE.md, STRATEGY.md, harden.sh 등)을 나열하고 "이 파일들 기반으로 보안 강화 스크립트"라고 하니 deploy-korea.sh처럼 순차적 배포 가이드를 줌. 안 통한 점: "1번 journal, 2번 물품 판단, 3번 사업 우선순위"를 나열하니 응답이 중단됐고, 재시작 시 요약 컨텍스트가 유용했다. 노하우: 보안/기능 작업은 "사전 확인: sourcing/aliexpress 모듈 상태"부터 물어보니 불필요한 재작업 줄음. 리뷰 기능(create-review.dto.ts)은 dto부터 시작해 UI(layout.tsx 등)까지 자연스럽게 연결됐다.

## 다중 프로젝트: 세션 요약으로 컨텍스트 유지
사주 프로젝트에서 관리자 대시보드와 구독제 vs 광고 전략을 논의 중이었는데, 세션이 끊어져 iap_service.dart와 entitlement.py 수정이 필요했다. "이전 요약 + 관리자 로그인 시 대시보드 UI 추가" 프롬프트로 재개하니 PDF 갱신과 IAP 과정까지 안내했다. bit-scalp 일일 최적화는 로그 분석부터("어제 로그에서 거래 목록 추출") 지정하니 python 스크립트 제안이 정확. 하네스와 Obsidian Vault는 session.md 정리와 run.sh 위치 확인처럼 가벼운 작업.

잘 통한 점: 프로젝트 전환 시 "session 요약 저장"을 Claude에게 시키니(하네스 session.md처럼), 컨텍스트 손실 없이 이어짐. 안 통한 점: macOS date 명령어 설명 시 모델이 플랫폼 가정 실수. 노하우: git 로그나 find/ls 결과를 공유하며 "이 출력 분석"하면 AI가 도구처럼 작동.

### 오늘의 한줄
복잡 로직은 단계 쪼개기 + 파일/로그 컨텍스트 공유가 AI 활용의 80%를 결정짓는다.
