오늘은 두 가지 일을 처리했다. 하나는 크리마 리뷰 마케팅 플랫폼의 대체 가능성을 조사한 건데, 예상과 달리 네이버 스마트스토어 쪽에 리뷰 API 자체가 존재하지 않는다는 벽을 확인했다. 다른 하나는 비트스캘프 봇의 거래량 게이트가 너무 높아 진입이 거의 없는 상태였는데, 백테스트 결과를 근거로 게이트를 낮추고 사이즈 차등을 도입했다.

## 크리마 리뷰 마케팅 대체 가능성 조사 — 네이버 스마트스토어 리뷰 API 부재

현재 스마트스토어 + 자사몰에서 제품을 판매하는 기업이 크리마 리뷰 마케팅을 사용 중인데, 대체 플랫폼으로 전환할 수 있는지 검토가 필요했다. 처음엔 "스마트스토어 API와 자사몰 API만 묶으면 자체 구축으로 충분하지 않을까"라는 가설을 세웠다. 자사몰 쪽은 게시판이나 리뷰 API가 있을 테니, 네이버 스마트스토어의 공식 리뷰 API만 확인하면 될 거라 생각했다.

네이버 커머스 API 문서를 샅샅이 뒤졌다. 하지만 스마트스토어의 **상품 리뷰 조회**에 해당하는 엔드포인트는 어디에도 없었다. 오히려 GitHub Discussion에서 2024-08-30에 올라온 기능 요청 쓰레드에서 "리뷰 조회 API는 현재 계획에 없음"이라는 운영진의 답변을 발견했다. 이게 결정적 단서였다.

> **네이버 커머스 API에는 공식 리뷰 조회 API가 없고, GitHub Discussion에 기능 요청만 올라와 있는 상태.**

이후 경쟁 솔루션인 알파리뷰, 브이리뷰의 연동 방식을 추적했다. 알파리뷰는 스마트스토어 관리자 권한을 위임받아 리뷰를 수집하는 구조였고, 자체 WebFetch 시도에서 cre.ma 도메인이 **403**으로 차단된 걸 보면 크리마 역시 동일한 메커니즘(봇 차단 + 권한 위임)을 사용한다는 게 확인됐다. 즉, 모든 외부 리뷰 솔루션은 공식 API가 아닌 **스토어 관리자의 직접 권한 위임**에 의존하고 있었다.

결론: 자사몰 쪽은 자체 API로 리뷰 연동이 가능하지만, **스마트스토어 쪽은 공식 API가 존재하지 않아** 크리마 같은 플랫폼을 완전히 대체할 수 없다. 스마트스토어 리뷰를 다루려면 현재로서는 크리마나 알파리뷰 같은 외부 솔루션을 계속 사용하거나, 네이버 쇼핑의 자체 리뷰 관리 기능을 활용하는 수밖에 없다. 자체 크롤링은 법적 리스크와 유지보수 비용이 너무 크다.

**검증**: 조사 결과를 팀에 공유했다. 추가로 특정 자사몰 게시판 API 스키마나 마이그레이션 시 데이터 이관 절차를 더 깊이 파볼 계획이다.

## 비트스캘프 거래량 게이트 하향 — $70M에서 $15M으로, 그리고 사이즈 차등 도입

최근 v4 아키텍처로 전환 후 거래량 게이트를 **$70M**으로 설정해놨는데, 스캔 결과 21개 후보 중 14개가 거래량 미달로 차단됐다. 나머지도 추세나 유동성 문제로 빠지면서 실질적으로 진입이 전혀 없었다. 사용자가 "모든 걸 다 청산하게 생겼다"고 알려왔다.

처음엔 게이트가 너무 높아서라고 생각했다. v4 안정화 당시 보수적으로 잡은 값이 현재 시장 상황에 맞지 않는다는 가설. `volume_bin_tp_sl_backtest.py` 백테스트 결과를 다시 열어봤다.

| 거래량 구간 | 평균 PnL (LONG) | 평균 PnL (SHORT) |
|---|---|---|
| $15M ~ $30M | +0.44% | +1.28% |
| $30M ~ $70M | +0.47% (SL 27%) | +1.22% |
| $70M ~ $150M | +0.45% (SL 22%) | +1.35% |
| > $150M | +0.50% | +1.44% (안전) |

$15M~$70M 구간의 PnL이 상위 구간과 거의 동일했다. 단, $30~70M 구간에서 LONG의 SL이 27%로 위험하지만, SHORT는 안정적이다. **게이트를 $15M까지 낮춰도 성능 손실이 없다**는 걸 확인했다. 오히려 $70M 게이트가 대부분의 알트코인에서 진입 기회를 차단하고 있었다. 현재 시장에서 거래량 $15M~$70M 사이에 몰려 있는 종목이 대부분이라, 이걸 놓치면 봇이 유의미한 거래를 할 수 없다.

사용자와 협의해 최종 방향을 결정했다. 거래량 게이트는 **$15M**으로 낮추고, $15M~$70M 구간은 메이저 대비 **50% 포지션 사이즈**만 진입하도록 차등을 두기로 했다. $70M 이상은 정상 사이즈. 위험을 완전히 배제하는 대신 부분 진입으로 리스크를 조절하는 쪽.

**바뀐 점**
- `risk_manager.calculate_position_size`에 `vol_mu` 파라미터를 추가해 거래량에 따른 사이즈 차등 로직 삽입
- `executor.py`, `orchestrator.py`에 해당 파라미터 전달 경로 수정
- `data/tunable_params.json` 갱신 (거래량 게이트 $15M)

**검증**: 봇 재시작(PID 96660) 후 스캔에서 PENDLE($23M), TIA($18M), FARTCOIN($18M) 등이 통과되면서 진입이 재개됐다. $15M 미만만 차단된다.

**남은 과제**: $15M 미만 종목에 대한 백테스트 데이터가 부족해 추가 검증이 필요하다. 또한 시장 변동에 따라 게이트를 동적으로 조정하는 로직을 고려 중이다.

### 오늘의 한줄
네이버 스마트스토어 리뷰 API는 존재하지 않으며, 봇의 거래량 게이트는 너무 높으면 오히려 봇을 굶게 만든다 — 정답은 문턱을 낮추고 사이즈로 제어하는 것.
