오늘은 포트폴리오 다듬기, `joyuno.github.io` 설정 업데이트, 사주 풀이에 Claude를 활용한 산만하지만 생산적인 하루였다. 직설적 질문보다는 **이전 컨텍스트 공유**나 **이미지 첨부**를 먼저 시도하니 응답이 구체적이었지만, 사주처럼 주관적 영역에선 **참조 프롬프트 재사용**이 핵심이었다.

## 포트폴리오 커리어 조언
LinkedIn용 콘텐츠와 연봉 추정을 위해 Claude에게 이력을 공유하며 물어봤다. 평소처럼 "연봉 얼마?" 식이 아닌, **한국 시장 + 이전 이력**을 명시한 프롬프트가 효과적이었다.

> "너 생각에 카야에 내가 들어간다면 어느정도의 연봉을 받을거같아? 이전 이력이나 한국시장 진출을 생각해봤을때 고려해서 판단해줘"

**잘 통한 점**: 시장 데이터와 이력을 결합한 응답이 현실적 — 20% 정도의 범위로 추정해줘서 바로 참고 가능.  
**안 통한 점**: 구체적 숫자보단 "경쟁사 벤치마킹" 식 일반론으로 끝날 때가 있어, 후속으로 "숫자 범위만" 재질문.

Core Competencies 정리도 직무 설명을 던지니 500자 내 맥락식으로 잘 뽑아줬다.

| 프롬프트 스타일 | 결과 품질 | 시간 절감 |
|---------------|----------|----------|
| 직무 설명만 | 보통 (템플릿 느낌) | 중간 |
| **이력+시장 추가** | **높음** (맞춤형) | **높음** |

**노하우**: 다음엔 `claude.md`에 이력 템플릿 저장 후 복붙 — 컨텍스트 누락 줄임.

## joyuno.github.io 설정 업데이트
Google Analytics 태그를 `default.html`에 삽입하고 `_config.yml`, `robots.txt`를 수정했다. Claude에게 스크립트 구조를 물어보며 진행, `git` 커밋과 `until` 루프로 배포 확인.

AI에게는 "gtag.js 삽입 코드"를 간단히 요청했지만, **기존 파일 컨텍스트 없이** 물으니 boilerplate만 나왔다.

> "Google tag (gtag.js) 스크립트 async src로 넣는 법 알려줘"

**잘 통한 점**: 짧은 코드 스니펫은 즉시 검증 가능 — 5분 만에 `default.html` 적용.  
**안 통한 점**: 사이트 구조(`_config.yml` 연동) 설명 안 주니, robots.txt 충돌 가능성 놓침. 직접 `ls`와 `grep`으로 파일 검토 후 수정.

다음 접근:
1. 파일 목록 먼저 공유 (`ls` 결과 inline).
2. "이 파일에 gtag 추가" 식으로 **위치 지정**.
3. 변경 후 `git log`으로 diff 확인.

**배운 점**: 웹 설정은 AI 코드 생성 > 인간 검증 분업이 딱 — AI의 "완벽 코드" 믿지 말고 로컬 테스트 필수.

## 사주 풀이 컨텍스트 재활용
사주 관련해서 이전 Claude API 프롬프트를 참조하며 물었고, **[Image #5]** 첨부로 여자 종합사주 → 궁합 순으로 분석. 월지(12→1 변경) 질문도 던졌다.

> "전에 내가 클로드에서 사주 봣던 애한테 작성해서 너 참고하라햇던 현재 클로드 api 프롬프트로 잇는 거 참고해서 봐줄수있어?"

**잘 통한 점**: **이전 프롬프트 + 이미지** 조합으로 일관된 해석 — 자우충 설명이 세밀해 10분 삽질 줄임.  
**안 통한 점**: 주관적 영역이라 "월지 왜 바뀌었음에도 자우충?" 같은 세부 질문에 **모순된 답변** 나옴. `grep`으로 과거 로그 추적하며 직접 검증.

| 시도 | 통한 이유 | 개선점 |
|------|-----------|--------|
| 프롬프트 참조만 | 기본 구조 OK | - |
| **이미지 첨부** | **시각적 정확도 ↑** | 텍스트 추출 먼저 |
| 세부 질문 | 부분 OK | **로그 grep** 병행 |

**노하우**: 사주처럼 반복 주제는 `prompts/saju.md`에 저장 — 컨텍스트 길어져도 Sonnet 3.5가 잘 소화.
