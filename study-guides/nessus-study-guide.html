1. Nessus란?

Nessus는 Tenable에서 개발한 세계적으로 가장 널리 사용되는 취약점 평가(Vulnerability Assessment) 솔루션입니다. 네트워크 장비, 서버, 웹 애플리케이션, 클라우드 인프라 등 다양한 IT 자산에 대한 보안 취약점을 자동으로 탐지, 분석, 보고합니다.

취약점 탐지

100,000개 이상의 CVE, CWE, CVSS 기반 취약점 플러그인을 활용해 정밀 스캔. 패치 미적용, 잘못된 구성, 알려진 익스플로잇 존재 여부를 식별합니다.

컴플라이언스 검증

PCI-DSS, HIPAA, CIS Benchmarks, NIST, GDPR 등 주요 규정 및 보안 기준에 대한 준수 상태를 자동으로 점검하고 보고서를 생성합니다.

자산 관리 및 시각화

스캔된 모든 자산(호스트, 서비스, 소프트웨어)을 카탈로그화하고, 위험도에 따라 우선순위를 부여하여 대시보드로 제공합니다.

AI 엔지니어 관점: Nessus는 보안 데이터의 구조화된 원천입니다. 스캔 결과(예: .nessus 파일)는 머신러닝 모델이 위협 트렌드를 분석하거나 예측 정밀도를 높이는 데 활용할 수 있는 풍부한 피처 세트를 제공합니다.

3. 핵심 개념

Nessus는 클라이언트-서버 아키텍처를 기반으로 하며, 주요 컴포넌트는 다음과 같습니다.

+-------------------+ 스캔 정책 & +-----------------------+ | Nessus Client |<<---템플릿 정의--->>| Nessus Server | | (Web UI / CLI) | | (Scanner Engine) | +-------------------+ 결과 보고 +----------+------------+ | ^ | | | | (스캔 프로브) V | V +------------------+ +---------------------------+ | 보고서 저장 | | Target Network/Assets | | (.nessus, | | (Hosts, Web Apps, DB, | | .csv, .pdf) | | Cloud Instances...) | +------------------+ +---------------------------+

주요 용어

  • 플러그인(Plugin): 특정 취약점이나 검사를 수행하는 코드 모듈. 각 플러그인은 고유 ID를 가짐.
  • 스캔 정책(Policy): 어떤 플러그인을 사용할지, 스캔 강도, 자격 증명 등을 정의한 템플릿.
  • 스캔(Scan): 정책과 타겟을 지정해 실행하는 단일 평가 작업.
  • 자격 증명(Credentials): 인증된 스캔(더 깊은 검사)을 위해 제공되는 OS/App 로그인 정보.
  • 리포지토리(Repository): 스캔 결과가 저장되는 논리적 공간.

4. 퀵스타트: 설치 → 설정 → 첫 예제

Step 1: 설치 (Ubuntu 22.04 기준)

# Tenable 공식 리포지토리 추가 및 Nessus 설치
curl -sSL https://www.tenable.com/downloads/api/v2/pages/nessus/files/Nessus-10.7.2-ubuntu1404_amd64.deb -o Nessus.deb
sudo dpkg -i Nessus.deb

# Nessus 서비스 시작
sudo systemctl start nessusd
sudo systemctl enable nessusd

# 상태 확인 (활성화될 때까지 약간 대기 필요)
sudo systemctl status nessusd

브라우저에서 https://localhost:8834 접속, 초기 설정 수행.

Step 2: 첫 스캔 생성 (Web UI)

  1. 로그인 후 Policies → Basic Network Scan 템플릿 선택.
  2. 타겟에 스캔할 IP 또는 CIDR (예: 192.168.1.0/24) 입력.
  3. 스캔 이름 지정 후 Launch 클릭.

Step 3: CLI 및 API를 통한 자동화 (Python 예제)

Nessus REST API를 사용해 스캔을 생성하고 결과를 수집합니다.

import requests
import json
import time
import warnings
warnings.filterwarnings('ignore') # SSL 경고 무시 (테스트용)

NESSUS_URL = "https://localhost:8834"
ACCESS_KEY = "YOUR_ACCESS_KEY"  # Settings > API Keys에서 생성
SECRET_KEY = "YOUR_SECRET_KEY"

headers = {
    "X-ApiKeys": f"accessKey={ACCESS_KEY}; secretKey={SECRET_KEY}",
    "Content-Type": "application/json"
}

# 1. 정책 ID 조회
def get_policy_id(policy_name="Basic Network Scan"):
    resp = requests.get(f"{NESSUS_URL}/policies", headers=headers, verify=False)
    policies = resp.json()["policies"]
    for p in policies:
        if p["name"] == policy_name:
            return p["id"]
    return None

# 2. 새 스캔 생성
def create_scan(policy_id, target, scan_name="API Test Scan"):
    scan_data = {
        "uuid": policy_id,
        "settings": {
            "name": scan_name,
            "description": "Created via API for automation",
            "text_targets": target,
            "enabled": False,  # 생성 후 수동 실행
            "launch": "ON_DEMAND"
        }
    }
    resp = requests.post(f"{NESSUS_URL}/scans", headers=headers, json=scan_data, verify=False)
    return resp.json()["scan"]["id"]

# 3. 스캔 실행
def launch_scan(scan_id):
    resp = requests.post(f"{NESSUS_URL}/scans/{scan_id}/launch", headers=headers, verify=False)
    return resp.json()["scan_uuid"]

# 4. 결과 내보내기 (예: .nessus 형식)
def export_scan_results(scan_id, file_format="nessus"):
    # 내보내기 요청
    resp = requests.post(f"{NESSUS_URL}/scans/{scan_id}/export",
                         headers=headers,
                         json={"format": file_format},
                         verify=False)
    file_id = resp.json()["file"]
    
    # 파일 준비 대기
    while True:
        status_resp = requests.get(f"{NESSUS_URL}/scans/{scan_id}/export/{file_id}/status",
                                   headers=headers, verify=False)
        if status_resp.json()["status"] == "ready":
            break
        time.sleep(2)
    
    # 파일 다운로드
    download_resp = requests.get(f"{NESSUS_URL}/scans/{scan_id}/export/{file_id}/download",
                                 headers=headers, verify=False)
    with open(f"scan_{scan_id}.nessus", "wb") as f:
        f.write(download_resp.content)
    print(f"[+] 결과 저장됨: scan_{scan_id}.nessus")

# 실행 예시
if __name__ == "__main__":
    policy_id = get_policy_id()
    if policy_id:
        scan_id = create_scan(policy_id, "192.168.1.10", "Automated API Scan")
        print(f"[*] 스캔 생성됨 (ID: {scan_id})")
        launch_uuid = launch_scan(scan_id)
        print(f"[*] 스캔 실행됨 (UUID: {launch_uuid})")
        # 실제 운영시에는 스캔 완료를 기다리는 로직 추가 필요
        # time.sleep(300)  # 5분 대기 예시
        # export_scan_results(scan_id)
    else:
        print("[!] 정책을 찾을 수 없습니다.")

성능 Max Simultaneous Checks 5-10 (호스트당 동시 검사 수) 높을수록 깊이 있는 스캔 but 리소스 소모多

스케줄링 Schedule 비업무 시간대 설정 (예: 매주 토요일 02:00) 운영 중인 서비스에 대한 영향 최소화

6. 실무 팁 & 트러블슈팅

팁 #1: 항상 "Discovery Scan"으로 먼저 네트워크의 라이브 호스트와 오픈 포트를 식별한 후, 타겟을 좁혀서 상세 취약점 스캔을 실행하세요. 시간을 절약하고 불필요한 트래픽을 줄입니다.
팁 #2: 컴플라이언스 스캔을 위해 CIS Benchmarks 템플릿을 사용하세요. 시스템이 보안 베이스라인을 준수하는지 자동으로 점검하고, 미준수 항목에 대한 상세 조치 가이드를 제공합니다.
문제 #1: 스캔이 매우 느리거나 멈춤
  • 원인: 동시 호스트/체크 수 과다, 네트워크 지연, 타겟 호스트 응답 지연.
  • 해결: 정책에서 Max Simultaneous Hosts를 5-10으로 낮추고, Performance 설정을 조정. "Ping the remote host" 옵션을 활성화해 죽은 호스트를 빠르게 스킵.
문제 #2: 많은 False Positive (잘못된 양성)
  • 원인: 공격적 스캔 모드, 오래된 플러그인, 특정 네트워크 장비/방화벽의 비정상 응답.
  • 해결: 스캔 정책을 "Normal" 또는 "Paranoid"로 변경. 플러그인 피드를 최신 상태로 유지(sudo /opt/nessus/sbin/nessuscli update). 결과를 확인해 특정 플러그인을 비활성화.
문제 #3: API 호출 시 "403 Forbidden" 또는 인증 오류
  • 원인: 잘못된 API 키, 키 권한 부족, Nessus 서버의 API 접근 제한.
  • 해결: Nessus Web UI의 Settings > API Keys에서 새 키 쌍 생성. 헤더 형식(X-ApiKeys)이 정확한지 확인. 서버 설정에서 API 접근이 허용되어 있는지 확인.
AI 연동 팁: .nessus XML 리포트를 파싱하여 Pandas DataFrame으로 변환한 후, 위험도(CVSS 점수), 자산 중요도, 패치 가용성 등을 피처로 한 머신러닝 모델을 구축해 실제 공격 발생 가능성이 높은 취약점의 우선순위를 재조정할 수 있습니다.
