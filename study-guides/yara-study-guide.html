1. YARA란?

YARA는 "Yet Another Recursive Acronym"의 약자로, 파일이나 메모리에서 패턴을 식별하기 위한 도구이자 규칙 언어입니다. 주로 악성코드 연구원과 보안 전문가가 특정 악성코드 패밀리나 위협 지표(IoC)를 식별하는 데 사용합니다.

YARA 규칙은 문자열, 바이트 시퀀스, 정규 표현식 등을 기반으로 한 서술적 패턴을 정의하며, 복잡한 논리 조건을 결합하여 정확한 식별을 가능하게 합니다.

패턴 매칭

텍스트 문자열, 바이트 시퀀스, 정규 표현식을 파일이나 메모리에서 검색하여 정확히 매칭합니다. 해시 기반 검색보다 변형된 악성코드에 강력합니다.

메타데이터 활용

규칙에 작성자, 설명, 참조 정보 등의 메타데이터를 포함시켜 협업과 규칙 관리를 효율화합니다. 위협 인텔리전스 공유에 적합합니다.

조건부 로직

AND, OR, NOT 연산자를 활용해 복잡한 매칭 조건을 구성할 수 있습니다. 단순 패턴 이상의 정교한 식별 로직 구현이 가능합니다.

2. 왜 써야 하는가?

YARA는 기존의 정적 시그니처 검사나 해시 기반 검색을 보완하는 강력한 도구입니다. 아래 비교표를 통해 YARA의 장점을 확인할 수 있습니다.

방식 장점 단점 YARA 비교
해시 기반 (MD5/SHA256) 매우 빠름, 정확도 100% 최소한의 변경에도 무력화, 규모 확장 비효율 패턴 기반으로 변형에도 강함
정적 시그니처 (ClamAV 등) 광범위한 패밀리 커버리지 복잡한 규칙 작성 어려움, 성능 이슈 직관적인 규칙 문법, 높은 성능
머신러닝/AI 모델 미지의 위협 탐지 가능 설명 가능성 낮음, 오탐률 관리 어려움 명확한 규칙 기반, 설명 가능성 높음
YARA 높은 정확도, 변형 강인성, 협업 용이 규칙 작성 학습 필요, 패턴 관리 부담 -

AI 엔지니어에게 중요한 점

YARA는 AI 기반 위협 탐지 시스템의 전처리 필터나 후처리 검증 계층으로 효과적입니다. AI 모델이 탐지한 샘플을 YARA 규칙으로 재검증하거나, YARA로 선별한 샘플을 AI 모델 학습 데이터로 활용할 수 있습니다.

3. 핵심 개념

YARA 규칙은 크게 세 부분으로 구성됩니다: Rule Header, Strings Section, Condition Section.

┌─────────────────────────────────────────────────────────────┐ │ YARA Rule Structure │ ├─────────────────────────────────────────────────────────────┤ │ rule <rule_name> : <tag1> <tag2> ... { │ │ meta: │ │ author = "..." │ │ description = "..." │ │ reference = "..." │ │ │ │ strings: │ │ $str1 = "malicious" │ │ $str2 = { 6A 40 68 00 30 00 00 } │ │ $reg1 = /http:\/\/[a-z0-9]+\.(com|net)/ │ │ │ │ condition: │ │ ( $str1 or $reg1 ) and filesize < 200KB │ │ } │ └─────────────────────────────────────────────────────────────┘

주요 구성 요소

  • Rule Header: 규칙 이름과 선택적 태그 정의
  • Meta Section: 규칙에 대한 설명 메타데이터
  • Strings Section: 검색할 문자열/바이트 패턴 정의
  • Condition Section: 매칭 조건을 정의하는 부울 표현식

주의사항

YARA 규칙은 대소문자를 구분합니다. 규칙 이름, 문자열, 조건문 모두 정확한 대소문자를 사용해야 합니다.

4.2 첫 번째 YARA 규칙 작성

간단한 악성코드 패턴을 탐지하는 규칙을 작성해봅니다.

malware_rule.yar
rule Detect_Suspicious_PE {
    meta:
        author = "AI Security Team"
        description = "의심스러운 PE 파일 특징 탐지"
        date = "2026-03-01"
        threat_level = "high"
    
    strings:
        $mz_header = "MZ"
        $pe_header = "PE"
        $suspicious_string = "This program cannot be run in DOS mode"
        $url_pattern = /https?:\/\/[a-zA-Z0-9\.\-]+\.[a-z]{2,}\/[a-zA-Z0-9\/\.\-]+/
        $hex_pattern = { 68 ?? ?? 00 00 68 ?? ?? 00 00 }
    
    condition:
        // MZ 헤더와 PE 헤더가 모두 존재 (PE 파일)
        $mz_header at 0 and $pe_header
        and 
        // 의심 문자열이 있거나 URL 패턴이 존재
        ( $suspicious_string or $url_pattern )
        and
        // 파일 크기가 100KB에서 10MB 사이
        filesize > 100KB and filesize < 10MB
}

4.3 Python에서 YARA 사용하기

Python
import yara
import os

# 규칙 컴파일
try:
    rules = yara.compile(filepath='malware_rule.yar')
except yara.SyntaxError as e:
    print(f"규칙 문법 오류: {e}")
    exit(1)

# 파일 스캔
def scan_file(file_path):
    if not os.path.exists(file_path):
        print(f"파일 없음: {file_path}")
        return
    
    try:
        matches = rules.match(file_path)
        if matches:
            print(f"[탐지] {file_path}")
            for match in matches:
                print(f"  규칙: {match.rule}")
                print(f"  태그: {', '.join(match.tags) if match.tags else '없음'}")
                print(f"  네임스페이스: {match.namespace}")
                print(f"  매칭 문자열:")
                for string in match.strings:
                    print(f"    - {string[1]} at offset {string[0]}")
        else:
            print(f"[정상] {file_path}")
    except Exception as e:
        print(f"스캔 오류: {e}")

# 디렉토리 배치 스캔
def batch_scan(directory):
    for root, dirs, files in os.walk(directory):
        for file in files:
            file_path = os.path.join(root, file)
            scan_file(file_path)

# 사용 예제
if __name__ == "__main__":
    # 단일 파일 스캔
    scan_file("suspicious.exe")
    
    # 디렉토리 배치 스캔
    # batch_scan("/path/to/scan")

4.4 CLI에서 YARA 사용

CLI
# 기본 스캔
yara malware_rule.yar suspicious_file.exe

# 태그 필터링
yara -t high malware_rule.yar suspicious_file.exe

# 재귀적 디렉토리 스캔
yara -r malware_rule.yar /path/to/directory

# 결과를 JSON으로 출력
yara -C malware_rule.yar suspicious_file.exe

# 여러 규칙 파일 사용
yara rules/ malware_rule.yar suspicious_file.exe

5. 핵심 설정 레퍼런스

설정/옵션 기본값 설명 실무 활용 예시
filesize - 현재 파일의 크기 (바이트) filesize < 500KB and filesize > 1KB
entrypoint - PE 파일의 진입점 오프셋 entrypoint == 0x1000
imports - PE 파일의 임포트 함수 확인 imports contains ("CreateRemoteThread", "VirtualAlloc")
exports - PE 파일의 익스포트 함수 확인 exports contains "malicious_function"
strings - 문자열 카운트/인덱스 접근 #str1 > 3 or @str1[0] < 100
pe 모듈 비활성 PE 파일 구조 분석 모듈 pe.characteristics & 0x2000
elf 모듈 비활성 ELF 파일 분석 모듈 elf.type == elf.ET_EXEC
math 모듈 비활성 수학 연산 함수 제공 math.entropy(0, filesize) > 7.5
time 모듈 비활성 시간 관련 함수 제공 time.now() > 2026

모듈 활성화 방법

YARA 모듈은 기본적으로 비활성화되어 있습니다. 사용하려면 Python 코드에서 명시적으로 활성화하거나 CLI에서 모듈을 로드해야 합니다.

# Python에서 모듈 활성화
rules = yara.compile(
    source='rule test { condition: true }',
    modules=['pe', 'math', 'time']
)

6. 실무 팁 & 트러블슈팅

자주 겪는 문제와 해결책

문제: 성능 저하

증상: 대용량 파일 스캔 시 매우 느림

원인: 너무 많은 정규식, 광범위한 와일드카드

해결책:

  • 정규식보다는 문자열 리터럴 우선 사용
  • 와일드카드 범위 제한: ?? 대신 [A-F][0-9]
  • 파일 크기 조건으로 조기 필터링

문제: 오탐률 높음

증상: 정상 파일까지 악성코드로 탐지

원인: 너무 일반적인 패턴 사용

해결책:

  • 패턴 조합 사용: 단일 패턴이 아닌 AND 조건
  • 컨텍스트 추가: 특정 오프셋에서만 매칭
  • 거짓 양성 데이터셋으로 테스트

문제: 규칙 관리 어려움

증상: 규칙 수가 많아지고 충돌 발생

원인: 체계적인 규칙 관리 부재

해결책:

  • 태그 체계 도입: malware, ransomware, info_stealer
  • 네임스페이스 분리: rules/{vendor}/{malware_family}
  • 규칙 버전 관리: meta section에 version 필드 추가

고급 실무 팁

팁 1: 엔트로피 기반 탐지

패킹된 악성코드는 높은 엔트로피를 가집니다. math 모듈의 entropy() 함수를 활용:

rule High_Entropy_Packed {
    meta:
        description = "높은 엔트로피를 가진 패킹 파일 탐지"
    
    condition:
        math.entropy(0, filesize) > 7.8
        and
        filesize > 50KB and filesize < 2MB
}

팁 2: AI와 연동한 동적 규칙 생성

AI 모델이 발견한 패턴을 YARA 규칙으로 자동 생성:

import yara
import json

def create_rule_from_ai_findings(finding_id, patterns, metadata):
    """AI 분석 결과로부터 YARA 규칙 생성"""
    rule_template = """
rule AI_Detected_{rule_id} {{
    meta:
        ai_finding_id = "{finding_id}"
        confidence = {confidence}
        generated_at = "{timestamp}"
    
    strings:
{strings}
    
    condition:
{condition}
}}
"""
    
    strings_section = ""
    for i, pattern in enumerate(patterns.get('strings', [])):
        strings_section += f'        $ai_str{i} = "{pattern}"\n'
    
    for i, hex_pattern in enumerate(patterns.get('hex_patterns', [])):
        strings_section += f'        $ai_hex{i} = {{ {hex_pattern} }}\n'
    
    # 조건 생성 (모든 패턴이 존재해야 함)
    condition = "        "
    all_patterns = [f'$ai_str{i}' for i in range(len(patterns.get('strings', [])))]
    all_patterns += [f'$ai_hex{i}' for i in range(len(patterns.get('hex_patterns', [])))]
    
    if all_patterns:
        condition += " and ".join(all_patterns)
    else:
        condition += "false"
    
    return rule_template.format(
        rule_id=finding_id.replace("-", "_"),
        finding_id=finding_id,
        confidence=metadata.get('confidence', 0.8),
        timestamp=metadata.get('timestamp', '2026-03-01'),
        strings=strings_section,
        condition=condition
    )

# 사용 예제
ai_findings = {
    'finding_id': 'malware-xyz-123',
    'patterns': {
        'strings': ['evil.com', 'malicious payload'],
        'hex_patterns': ['E8 ?? ?? ?? ?? 83 C4 04']
    },
    'metadata': {
        'confidence': 0.92,
        'timestamp': '2026-03-01T12:00:00Z'
    }
}

yara_rule = create_rule_from_ai_findings(**ai_findings)
print(yara_rule)

주의: 메모리 사용량

YARA는 규칙 컴파일 시 메모리에 로드합니다. 수만 개의 규칙을 동시에 사용하면 메모리 사용량이 급증할 수 있습니다. 규칙을 논리적 그룹으로 나누고 필요할 때만 로드하는 전략을 고려하세요.
