1. MLflow란?

MLflow는 머신러닝 라이프사이클을 관리하기 위한 오픈소스 플랫폼입니다. 실험 추적, 모델 패키징, 배포, 레지스트리 관리를 단일 인터페이스로 제공하여 재현성과 협업을 강화합니다.

📊 실험 추적 (Tracking)

코드, 데이터, 설정, 메트릭을 기록하여 실험의 재현성을 보장합니다. 로컬 파일, 데이터베이스, 원격 서버에 저장 가능합니다.

📦 프로젝트 (Projects)

재현 가능한 ML 코드 패키징 표준을 제공합니다. Conda, Docker를 활용해 환경을 정의하고, 프로젝트를 실행 가능한 단위로 묶습니다.

🏷️ 모델 관리 (Models)

모델을 표준 포맷으로 패키징하여 다양한 배포 환경(Docker, REST API, Apache Spark 등)에서 사용할 수 있게 합니다.

🗂️ 모델 레지스트리 (Registry)

모델의 버전 관리, 스테이징(Staging/Production/Archived), 협업 및 승인 워크플로우를 중앙에서 관리합니다.

2. 왜 써야 하는가?

MLflow는 기존의 자체 구축 솔루션 또는 다른 도구들에 비해 다음과 같은 장점을 제공합니다.

대안 한계점 MLflow의 해결책
스프레드시트/노트 수동 기록, 실수 가능성 높음, 재현성 낮음 자동화된 실험 추적, 코드와 결과의 직접 연결
자체 구축 DB + 대시보드 유지보수 부담, 확장성 문제, 기능 제한 오픈소스 생태계, 활발한 커뮤니티, 지속적 업데이트
상용 플랫폼 (e.g., Weights & Biases) 비용, Vendor Lock-in, 데이터 외부 유출 우려 오픈소스/자체 호스팅 가능, 데이터 주권 보장
Git만 사용 대용량 모델/데이터 관리 불가, 메트릭 비교 어려움 대용량 아티팩트 저장소, 시각화된 메트릭 비교 UI
💡 핵심 가치: MLflow는 팀의 ML 개발 속도를 높이고, 모델 관리의 복잡성을 줄이며, 개발부터 운영까지의 간극(Gap)을 해소하는 데 초점을 맞춥니다.

3. 핵심 개념

MLflow의 아키텍처는 백엔드 저장소, 아티팩트 저장소, UI 서버로 구성됩니다.

+-----------------------------------------------------------------------+ | MLflow Tracking Server | | +-------------------+ +-------------------+ +-------------------+ | | | Backend Store | | Artifact Store | | UI/HTTP | | | | (SQLite, MySQL, | | (Local, S3, ADLS, | | (Port 5000) | | | | PostgreSQL) | | GCS, NFS) | | | | | +-------------------+ +-------------------+ +-------------------+ | +-----------------------------------|-----------------------------------+ | (HTTP API) +-----------------------------------|-----------------------------------+ | Client Code (Python, R, Java, etc.) | | with mlflow.start_run(): | | mlflow.log_param("lr", 0.01) | | mlflow.log_metric("accuracy", 0.95) | | mlflow.log_artifact("model.pkl") | +-----------------------------------------------------------------------+

주요 컴포넌트

  • Run: 한 번의 실험 실행 단위. 파라미터, 메트릭, 태그, 아티팩트를 포함합니다.
  • Experiment: 관련된 Run들의 집합 (예: "iris-classification").
  • Backend Store: Run의 메타데이터(파라미터, 메트릭)를 저장하는 DB.
  • Artifact Store: 모델 파일, 이미지, 데이터 등 대용량 파일을 저장하는 저장소.
  • Model Registry: 모델의 버전, 스테이지, 설명을 관리하는 중앙 저장소.

4.1 설치

CLI / Python

pip install mlflow

# 추가 기능 포함 설치 (예: scikit-learn, PyTorch 지원)

# 또는 특정 라이브러리만
pip install mlflow-skinny  # 최소 의존성 버전

4.2 로컬 서버 실행

CLI
# 기본 설정 (SQLite 백엔드, 로컬 아티팩트 저장소)
mlflow ui --backend-store-uri sqlite:///mlflow.db --default-artifact-root ./mlruns

# 브라우저에서 http://localhost:5000 접속

4.3 첫 번째 실험 코드

Python Script (train.py)
import mlflow
import mlflow.sklearn
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

# 실험 설정 (없으면 생성)
mlflow.set_experiment("Iris-Classification-Quickstart")

with mlflow.start_run(run_name="RF-Baseline"):
    # 1. 데이터 로드 및 분할
    iris = load_iris()
    X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, test_size=0.2)

    # 2. 파라미터 정의 및 로깅
    params = {"n_estimators": 100, "max_depth": 5, "random_state": 42}
    mlflow.log_params(params)

    # 3. 모델 학습
    model = RandomForestClassifier(**params)
    model.fit(X_train, y_train)

    # 4. 평가 및 메트릭 로깅
    y_pred = model.predict(X_test)
    acc = accuracy_score(y_test, y_pred)
    mlflow.log_metric("accuracy", acc)

    # 5. 모델 아티팩트 로깅 (MLflow 형식)
    mlflow.sklearn.log_model(model, "model")

    # 6. 추가 아티팩트 (예: 평가 리포트 텍스트 파일)
    with open("eval_report.txt", "w") as f:
        f.write(f"Accuracy: {acc:.4f}")
    mlflow.log_artifact("eval_report.txt")

    print(f"Run saved. Accuracy: {acc:.4f}")
    print(f"UI: http://localhost:5000")
실행: 터미널에서 python train.py 실행 후, 브라우저로 http://localhost:5000 접속하여 결과를 확인하세요.

5. 핵심 설정 레퍼런스

환경 변수 / 설정 기본값 설명 실무 예시
MLFLOW_TRACKING_URI file:///./mlruns Tracking Server 주소 http://10.0.0.100:5000, databricks
MLFLOW_EXPERIMENT_NAME - 기본 실험 이름 prod-user-churn-v1
MLFLOW_ARTIFACT_ROOT 로컬 경로 아티팩트 저장소 위치 s3://my-mlflow-bucket/artifacts
MLFLOW_REGISTRY_URI Tracking URI와 동일 모델 레지스트리 URI postgresql://user:pass@host/db
--backend-store-uri sqlite:///mlflow.db 서버 실행 시 메타데이터 DB mysql://user:pass@host/mlflow
--host 127.0.0.1 서버 바인딩 호스트 0.0.0.0 (외부 접근 허용)
⚠️ 보안 주의: 프로덕션 환경에서는 반드시 인증/권한 설정(mlflow[server]의 기본 인증 또는 프록시)과 안전한 백엔드 DB(PostgreSQL 등)를 사용하세요.

6. 실무 팁 & 트러블슈팅

자주 겪는 문제

🚫 아티팩트 업로드 실패 (S3/GCS)

증상: PermissionError 또는 Access Denied.

해결: AWS CLI/GCloud SDK의 자격 증명이 설정되었는지 확인. 또는 boto3 세션을 직접 전달.
mlflow.set_tracking_uri("http://...")
mlflow.log_artifact(..., boto_session=my_session)

🐢 UI 로딩이 매우 느림

증상: Run 목록을 불러오는 데 시간이 오래 걸림.

해결: SQLite 대신 PostgreSQL/MySQL 사용. 많은 Run이 있는 경우 백엔드 DB 인덱싱 확인. --gunicorn-opts로 워커 수 증가.

🔗 모델 레지스트리 동기화 오류

증상: 레지스트리에 모델이 보이지 않거나 스테이지 변경이 안 됨.

해결: MLFLOW_REGISTRY_URI가 올바르게 설정되었는지 확인. 레지스트리용 별도 DB 스키마 사용 권장.

💾 대용량 파일 로깅 시 메모리 부족

증상: MemoryError 발생.

해결: mlflow.log_artifact()를 사용해 파일 시스템의 파일을 직접 업로드. 모델은 mlflow.*.log_model() 사용.

프로덕션을 위한 팁

  • 서버 배포: 단일 mlflow ui 명령어 대신, Gunicorn(Nginx 뒤에서) 또는 Docker 컨테이너로 배포.
  • 스케일 아웃: 아티팩트 저장소로 S3/GCS/ADLS 사용, 백엔드 DB는 관리형 서비스(RDS 등) 활용.
  • 모니터링: 서버 로그, DB 연결 수, 아티팩트 저장소 사용량을 모니터링하세요.
  • CI/CD 통합: 학습 파이프라인 끝에서 mlflow.register_model()을 호출하여 자동으로 모델 레지스트리에 등록.
