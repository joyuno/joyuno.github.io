1. Ghidra란?

Ghidra는 미국 국가안보국(NSA)에서 개발한 오픈소스 소프트웨어 리버스 엔지니어링(SRE) 도구 모음입니다. 바이너리 분석, 디스어셈블리, 디컴파일, 디버깅을 위한 통합 환경을 제공하며, 확장성 높은 플러그인 아키텍처가 특징입니다.

정적 분석

실행 없이 바이너리의 코드, 데이터 구조, 제어 흐름을 분석. 심볼릭 실행과 데이터 흐름 분석을 지원.

디컴파일

기계어를 고수준 언어(C/Pseudo-code)로 변환. 다중 아키텍처(x86/x64, ARM, MIPS, PPC 등) 지원.

협업 분석

여러 분석가가 동일한 프로젝트에서 작업할 수 있는 공유 저장소 기능. 변경 이력 추적 및 병합.

2. 왜 써야 하는가?

기존 도구 대비 Ghidra의 장점은 오픈소스, 확장성, NSA급 분석 알고리즘에 있습니다.

도구 라이선스 주요 강점 Ghidra 대비
IDA Pro 상용 업계 표준, 풍부한 플러그인 비용 부담, 폐쇄적
Binary Ninja 상용/제한적 무료 현대적 UI, API 우선 설계 중소규모 커뮤니티
Radare2/Cutter 오픈소스 CLI 중심, 경량, 스크립팅 학습 곡선 가파름
Ghidra Apache 2.0 무료/오픈소스, 고급 디컴파일러, 협업 기능 자체 JRE 필요, 초기 설정 복잡
AI 엔지니어 관점: Ghidra의 Java 기반 API와 헤드리스 모드는 분석 파이프라인 자동화 및 머신러닝 모델 학습 데이터 생성에 이상적입니다.

3. 핵심 개념

Ghidra는 클라이언트-서버 아키텍처와 프로젝트 기반 작업 흐름을 따릅니다.

+-------------------+ +----------------------+ +------------------------+ | Ghidra Server |<-->| Shared Repository |<-->| Analysis Database | | (Headless) | | (Versioned) | | (Program/Listing) | +-------------------+ +----------------------+ +------------------------+ ^ ^ ^ | | | +-------------------+ +----------------------+ +------------------------+ | Ghidra Client | | Tool/Plugin | | Script/Extension | | (GUI/Headless) |------| (Disassembler, |------| (Python/Java) | | | | Decompiler, etc.) | | | +-------------------+ +----------------------+ +------------------------+
  • Program: 분석 대상 바이너리의 내부 표현. 메모리 맵, 함수, 데이터를 포함.
  • Listing: 디스어셈블리/디컴파일된 코드의 뷰.
  • Project: 하나 이상의 Program과 관련 메타데이터의 컨테이너.
  • Plugin: 기능 확장 모듈 (Java/Python).
  • Headless Mode: GUI 없이 스크립트/배치 분석 실행.

4. 퀵스타트

4.1 설치 (Ubuntu 예제)

CLI: Ghidra 다운로드 및 실행 준비
# 의존성 설치
sudo apt update && sudo apt install -y openjdk-17-jdk unzip

# Ghidra 다운로드 (가정: 11.0 버전)
wget https://github.com/NationalSecurityAgency/ghidra/releases/download/Ghidra_11.0_build/ghidra_11.0_PUBLIC_20241015.zip
unzip ghidra_*.zip
cd ghidra_11.0_PUBLIC

# 실행 스크립트 권한 부여
chmod +x ghidraRun

# GUI 모드 실행
./ghidraRun
# 또는 헤드리스 모드 실행
# ./support/analyzeHeadless <project_path> <project_name> -import <binary_path> -postScript <script_name>

4.2 첫 프로젝트 및 분석

Python: 헤드리스 모드로 간단한 분석 자동화
# analyze_sample.py
# Ghidra의 Headless 분석 스크립트 예제 (Jython)
# 사용법: ./support/analyzeHeadless /tmp/ghidra_projects MyProject -import /path/to/binary -postScript analyze_sample.py

import sys
from ghidra.app.decompiler import DecompInterface
from ghidra.util.task import ConsoleTaskMonitor

def analyze_program():
    program = getCurrentProgram()
    name = program.getName()
    print(f"[*] Analyzing: {name}")

    # 기본 정보 수집
    listing = program.getListing()
    func_manager = program.getFunctionManager()
    functions = func_manager.getFunctions(True)  # True for forward iteration

    func_count = 0
    for func in functions:
        func_count += 1
        entry = func.getEntryPoint()
        print(f"  Function: {func.getName()} at {entry}")

    print(f"[+] Total Functions: {func_count}")

    # 디컴파일러 초기화 (예시)
    decompiler = DecompInterface()
    decompiler.openProgram(program)
    monitor = ConsoleTaskMonitor()

    # 첫 번째 함수 디컴파일 시도
    first_func = func_manager.getFunctions(True).next()
    if first_func:
        decomp_result = decompiler.decompileFunction(first_func, 60, monitor)
        if decomp_result.decompileCompleted():
            hcode = decomp_result.getDecompiledFunction().getC()
            print(f"[+] Decompiled snippet from {first_func.getName()}:\n{hcode[:500]}...")

if __name__ == "__main__":
    analyze_program()
주의: Ghidra 스크립팅은 Jython(Python 2.7 호환) 또는 Java로 작성됩니다. 최신 Python 3 기능을 사용하려면 외부에서 API를 호출하는 방식을 고려해야 합니다.

6. 실무 팁 & 트러블슈팅

자주 겪는 문제

  • 문제: "Out of memory" 오류 발생.
    해결: `ghidraRun` 스크립트 내 `MAXMEM=` 값을 증가 (예: `MAXMEM=4G`).
  • 문제: 디컴파일 결과가 비어 있거나 불완전.
    해결: 분석 옵션에서 `Decompiler Parameter ID` 분석을 실행하고, 함수 프로토타입을 수동으로 정의.
  • 문제: 헤드리스 모드에서 Python 스크립트 임포트 오류.
    해결: 스크립트 경로를 절대 경로로 지정하고, `-scriptPath` 인자로 디렉토리 추가.
  • 문제: 공유 프로젝트 동기화 충돌.
    해결: `File → Restore`로 버전 복원, 변경 사항을 병합할 때는 충돌하는 주소 범위를 수동 검토.

AI/자동화 팁

  • 함수 임베딩: Ghidra API로 추출한 CFG(제어 흐름 그래프)를 네트워크 임베딩 알고리즘(Node2Vec)에 입력하여 유사 함수 클러스터링.
  • 취약점 패턴 학습: 디컴파일된 코드를 NLP 토큰화하여 BERT 계열 모델로 취약한 코드 패턴 분류.
  • 대규모 코퍼스 구축: 헤드리스 모드로 수천 개의 바이너리를 배치 처리, 함수와 디컴파일 코드를 JSONL 형식으로 추출.
