1. Trivy란?

Trivy는 Aqua Security가 개발한 간단하고 포괄적인 취약점 스캐너입니다. 컨테이너 이미지, 파일 시스템, Git 리포지토리, IaC 구성 파일 등을 스캔하여 OS 패키지, 언어 라이브러리, 구성 오류의 취약점을 식별합니다.

멀티타겟 스캔

단일 도구로 이미지, 파일시스템, Git, IaC(테라폼, AWS CloudFormation 등)를 스캔합니다. CI/CD 파이프라인에 쉽게 통합.

정확한 SBOM 생성

스캔 결과와 함께 Software Bill of Materials(SBOM)를 생성하여 의존성 트리를 명확히 파악할 수 있습니다.

실시간 DB 업데이트

취약점 DB를 자동으로 업데이트하여 최신 CVE 정보를 제공합니다. 별도 DB 서버 관리가 필요 없습니다.

2. 왜 써야 하는가?

기존 취약점 스캐너와 비교한 Trivy의 강점입니다.

도구 주요 타겟 설치/운영 복잡도 스캔 속도 SBOM 지원 IaC 스캔
Trivy 멀티타겟 (이미지, FS, Git, IaC) 낮음 (단일 바이너리) 빠름 네이티브 지원 테라폼, CloudFormation 등
Clair 컨테이너 이미지 중간 (서비스 운영 필요) 중간 간접 지원 없음
Anchore 컨테이너 이미지 높음 (전체 플랫폼) 중간 지원 없음
OpenSCAP 시스템/컨테이너 높음 (복잡한 프로파일) 느림 제한적 없음
핵심 차별점: Trivy는 "단일 바이너리, 모든 스캔" 철학으로 설계되어 AI/데이터 엔지니어의 빠른 개발-배포 주기와 CI/CD 자동화에 최적화되었습니다.

3. 핵심 개념

Trivy의 내부 아키텍처와 데이터 흐름을 ASCII 다이어그램으로 이해합니다.

+-------------------+ +-------------------+ +-------------------+ | | | | | | | Scan Target |----->| Vulnerability |----->| Output Format | | (Image/FS/Git) | | Database | | (JSON/SARIF) | | | | | | | +-------------------+ +-------------------+ +-------------------+ | | | v v v +-------------------+ +-------------------+ +-------------------+ | | | | | | | SBOM Generator |<-----| Scanner Engine |<-----| Report Filter | | | | (Pkg/Lang/IaC) | | (by severity) | | | | | | | +-------------------+ +-------------------+ +-------------------+

주요 컴포넌트 설명

  • Scanner Engine: 타겟을 분석하여 패키지, 라이브러리, IaC 리소스를 식별하는 핵심 엔진.
  • Vulnerability Database: 정기적으로 업데이트되는 취약점 정보 저장소 (GitHub 저장소에서 동기화).
  • SBOM Generator: 식별된 모든 컴포넌트 목록을 표준 형식 (CycloneDX, SPDX)으로 생성.
  • Report Filter: 결과를 severity, type 등으로 필터링하여 출력 조정.

4. 퀵스타트

4.1 설치 (Linux/macOS)

curl -sfL https://github.com/aquasecurity/trivy/releases/latest/download/trivy_0.50.1_Linux-64bit.tar.gz | tar -xzf -
sudo mv trivy /usr/local/bin/
trivy version

4.2 첫 번째 컨테이너 이미지 스캔

# 공식 nginx 이미지 스캔
trivy image nginx:latest

# JSON 출력으로 결과 저장 (CI/CD 통합용)
trivy image --format json nginx:latest > scan_result.json

4.3 Python 프로젝트 파일 시스템 스캔

# 현재 디렉토리의 Python 패키지 취약점 검사
trivy fs .

# 특정 severity 이상만 출력
trivy fs --severity HIGH,CRITICAL .

4.4 IaC 구성 파일 스캔 (테라폼)

# 테라폼 파일 취약점 및 misconfiguration 검사
trivy config terraform/
주의: 처음 실행 시 취약점 DB를 다운로드하므로 네트워크 환경과 시간이 필요합니다. 이후 실행은 자동 업데이트됩니다.

5. 핵심 설정 레퍼런스

옵션/설정 설명 실무 사용 예
--format 출력 형식 지정 (table, json, sarif) CI/CD 통합 시 --format json 사용
--severity 특정 심각도 이상만 출력 (UNKNOWN,LOW,MEDIUM,HIGH,CRITICAL) --severity HIGH,CRITICAL으로 핵심 취약점만 필터링
--ignore-unfixed 수정 패치가 없는 취약점을 결과에서 제외 현재 조치 불가능한 항목을 보고서에서 제거
--exit-code 취약점 발견 시 지정한 코드로 종료 (기본: 0) CI 실패 트리거용 --exit-code 1
--cache-dir DB 캐시 디렉토리 지정 (기본: ~/.cache/trivy) 공유 스토리지에 캐시 위치 지정하여 여러 실행자 공유
--skip-db-update DB 업데이트 스킵 (오프라인 환경) 네트워크 제한 환경에서 캐시된 DB 사용
--timeout 스캔 작업 타임아웃 (기본: 5m) 대형 이미지 스캔 시 --timeout 10m

6. 실무 팁 & 트러블슈팅

자주 겪는 문제 + 해결책

문제 1: 스캔 속도가 느림 (대형 이미지)

해결책: --timeout 값 증가, 필요 시 --skip-db-update 사용 (캐시된 DB 활용), 또는 부분 스캔 (--severity HIGH,CRITICAL)으로 필터링.

문제 2: CI/CD에서 JSON 출력 파싱 오류

해결책: Python 스크립트 예제로 안정적 파싱 구현.

import json
with open('scan_result.json') as f:
    data = json.load(f)
for result in data.get('Results', []):
    for vuln in result.get('Vulnerabilities', []):
        if vuln['Severity'] in ['HIGH', 'CRITICAL']:
            print(f"{vuln['VulnerabilityID']} - {vuln['PkgName']}")
문제 3: DB 업데이트 네트워크 오류 (프록시/방화벽)

해결책: 환경변수 TRIVY_DB_REPOSITORY를 내부 미러로 설정 또는 오프라인 DB 미리 다운로드.

export TRIVY_DB_REPOSITORY="https://internal-mirror/trivy-db"
trivy image --skip-db-update nginx:latest  # 최초 한번 업데이트 후
팁 1: Git 리포지토리 직접 스캔

컨테이너화 전 소스 코드 취약점 검사.

trivy repo https://github.com/your-org/your-repo.git
팁 2: SBOM 생성 및 활용

의존성 관리와 규제 준수 문서화.

trivy image --format cyclonedx nginx:latest > sbom.xml
