1. DVC란?

DVC(Data Version Control)는 Git을 확장하여 대규모 데이터 파일, 머신러닝 모델, 실험 환경을 버전 관리하는 오픈소스 도구입니다. 코드와 데이터를 함께 관리하여 ML 프로젝트의 재현성과 협업을 가능하게 합니다.

데이터 버전 관리

대용량 데이터셋을 Git과 별도의 저장소(로컬, S3, GCS 등)에 저장하고, 메타파일(.dvc)을 통해 Git에서 포인터로 관리합니다.

실험 추적

모델 학습의 매개변수, 데이터 버전, 평가 메트릭을 자동 기록하여 실험 비교와 재현을 용이하게 합니다.

ML 워크플로우 관리

파이프라인을 정의하여 데이터 준비, 학습, 평가 단계를 자동화하고 각 단계의 출력을 캐싱합니다.

2. 왜 써야 하는가?

기존 방식과의 비교를 통해 DVC의 장점을 확인합니다.

대안 / 접근법 문제점 DVC 해결 방식
Git에 데이터 직접 저장 대용량 파일로 레포지토리 크기 폭발, 병합/클론 느림 데이터는 외부 저장소, Git에는 메타파일만 저장
수동 데이터 관리 (폴더/파일명) 데이터 버전 혼란, 재현 불가, 협업 어려움 명시적 버전 태그 및 커밋과 연결
실험 기록 스크립트/노트 분산된 기록, 비교 분석 어려움 통합 실험 추적 (metrics, params)
직접 파이프라인 스크립팅 단계 캐싱 없음, 재실행 비효율 자동 캐싱 및 의존성 그래프 관리

핵심 가치: DVC는 Git 중심의 ML 프로젝트 관리 패러다임을 제공합니다. 데이터와 모델을 코드와 동일한 버전 컨트롤 시스템 내에서 관리하여 엔드투엔드 재현성을 보장합니다.

3. 핵심 개념

DVC의 핵심 구성 요소와 아키텍처를 이해합니다.

+-------------------+ +-------------------+ +-------------------+ | Git Repository | | DVC Meta Files | | Remote Storage | | | | (.dvc, .dvc/) | | (S3, GCS, SSH) | | code, .dvc files|------| pointers, hash |------| actual data | | | | dependencies | | files | +-------------------+ +-------------------+ +-------------------+ | | | | | | +-------------------+-----------------------------------+-------------------+ | DVC Commands | | DVC Cache | | (add, run, push)| | (local cache) | +-------------------+ +-------------------+ | | | | +-------------------------------------------------------------------------------+ | ML Pipeline & Experiments | | (dvc.yaml, params.yaml, metrics) | +-------------------------------------------------------------------------------+

주요 구성 요소

  • .dvc 파일: 데이터 파일의 메타데이터(해시, 경로)를 저장하는 포인터 파일.
  • DVC 캐시: 로컬에 저장된 데이터 객체의 내용물. 기본적으로 `.dvc/cache`에 위치.
  • 원격 저장소: 데이터가 실제 저장되는 곳 (AWS S3, Google Cloud Storage, Azure Blob, SSH, 등).
  • dvc.yaml: 파이프라인 단계(stages), 의존성(deps), 출력(outputs)을 정의.
  • params.yaml / metrics.yaml: 매개변수와 평가 지표를 관리하는 파일.

4. 퀵스타트

실습 환경을 빠르게 구성하고 첫 데이터 버전 관리와 파이프라인 실행을 진행합니다.

4.1 설치

# pip를 통해 설치 (Python >=3.8)
pip install dvc

# 원격 저장소에 따라 추가 옵션 설치 (예: S3)
pip install 'dvc[s3]'

# 설치 확인
dvc version

4.2 초기화 및 설정

# Git 프로젝트 내에서 DVC 초기화
git init my-ml-project
cd my-ml-project
dvc init

# 원격 저장소 설정 (예: 로컬 디렉토리 - 실제는 S3/GCS 사용)
dvc remote add -d myremote /path/to/remote-storage

# 설정 파일 확인
cat .dvc/config

4.3 첫 데이터 버전 관리

# 데이터 파일 추가 (예: dataset.csv)
dvc add data/dataset.csv

# .dvc 파일이 생성됨 (Git에 추가)
git add data/dataset.csv.dvc
git commit -m "Add dataset v1"

# 데이터를 원격 저장소에 푸시
dvc push

4.4 기본 파이프라인 예제

# dvc.yaml 생성 (간단한 학습 파이프라인)
cat > dvc.yaml << EOF
stages:
  prepare:
    cmd: python scripts/prepare.py
    deps:
      - data/dataset.csv
    outs:
      - data/prepared.csv
  train:
    cmd: python scripts/train.py
    deps:
      - data/prepared.csv
      - params.yaml
    params:
      - train.learning_rate
      - train.epochs
    outs:
      - model/model.pkl
  evaluate:
    cmd: python scripts/evaluate.py
    deps:
      - model/model.pkl
    metrics:
      - metrics.json
EOF

# 파이프라인 실행
dvc repro

주의: 첫 실행 시 원격 저장소가 구성되지 않으면 데이터는 로컬 캐시만 존재합니다. 협업 환경에서는 원격 저장소 설정이 필수입니다.

5. 핵심 설정 레퍼런스

주요 DVC 구성 파일과 명령어 레퍼런스입니다.

파일/명령 목적 예제/구문
.dvc/config DVC 전역 및 원격 저장소 설정 [core] remote = myremote
.dvcignore DVC 관리 제외 패턴 (like .gitignore) *.tmp
logs/
dvc.yaml 파이프라인 단계 정의 stages:
train:
cmd: python train.py
params.yaml 모델 매개변수 중앙 관리 train:

dvc add 데이터 파일 버전 관리 시작 dvc add data/raw.zip

dvc exp show

6. 실무 팁 & 트러블슈팅

실무에서 자주 발생하는 문제와 해결 방법입니다.

자주 겪는 문제

  • 데이터 파일이 너무 커서 dvc add가 느림
    • 해결: 대용량 파일은 압축 또는 분할 후 추가. 또는 `.dvcignore`로 임시 파일 제외.
  • dvc pull 시 특정 버전 데이터 불러오기
    • 해결: 해당 데이터 버전의 Git 커밋으로 이동 후 dvc pull 실행. 또는 dvc checkout 사용.
  • 원격 저장소 변경 시 기존 데이터 연결 끊김
    • 해결: 새 원격 저장소 설정 후 dvc push --all로 모든 데이터 마이그레이션.
  • 파이프라인 단계 캐싱이 예상대로 동작하지 않음
    • 해결: dvc dag로 의존성 그래프 확인. dvc run --no-run-cache로 캐시 무시 실행.
  • 팀원이 Git만 클론하고 데이터가 없음
    • 해결: dvc pull 명령어 교육 및 프로젝트 README에 명시. CI/CD에 dvc pull 단계 추가.

실무 팁

저장소 선택

S3/GCS 등 객체 저장소를 원격으로 사용할 경우, 저비용 스토리지 클래스와 라이프사이클 정책을 설정하여 비용 관리.

CI/CD 통합

GitHub Actions 등 CI에서 dvc pull 후 파이프라인 실행하여 모델 재학습/평가 자동화.

실험 관리

dvc exp run --set-param으로 하이퍼파라미터 변경 실험을 쉽게 생성하고 dvc exp show로 비교.

성능 팁: 대규모 데이터셋은 `.dvc/cache`를 빠른 SSD 디스크에 위치시키고, 원격 저장소는 지역적 리전을 선택하여 전송 속도 향상.
