1. Weights & Biases란?

Weights & Biases(W&B)는 머신러닝 실험의 추적, 시각화, 관리를 위한 플랫폼입니다. 연구 및 프로덕션 환경에서 모델 개발 라이프사이클 전반을 가속화하고 재현성을 보장합니다.

📊 실험 추적 (Experiment Tracking)

하이퍼파라미터, 메트릭, 코드 상태, 시스템 리소스를 자동으로 기록하고 비교할 수 있는 대시보드를 제공합니다.

📈 모델 시각화 (Model Visualization)

학습 곡선, 예측 결과, 모델 그래프, 임베딩, 컨퓨전 매트릭스 등을 인터랙티브하게 시각화합니다.

🤝 협업 & 버전 관리 (Collaboration)

팀원과 실험 결과, 모델 아티팩트, 노트북을 공유하고 피드백을 주고받을 수 있는 공동 작업 공간을 제공합니다.

💡 핵심 가치

W&B는 "MLOps의 GitHub"으로 불리며, 실험의 재현성과 협업 효율성을 극대화하는 것이 목표입니다. 로컬 파일이나 스프레드시트에 의존하는 방식을 벗어나 체계적인 ML 개발을 가능하게 합니다.

2. 왜 써야 하는가?

ML 실험 관리의 기존 대안과 W&B를 비교합니다.

도구 / 방법 장점 단점 W&B의 차별점
텐서보드 (TensorBoard) 무료, TensorFlow와 긴밀한 통합 협업 기능 부족, 실험 비교 기능 약함, 지속적 관리 어려움 강력한 실험 비교, 팀 대시보드, 아티팩트 버전 관리
MLflow 오픈소스, 모델 패키징 및 배포 기능 UI/UX가 단순, 시각화 기능 제한적, 호스팅 필요 뛰어난 사용자 경험, 리치한 인터랙티브 시각화, 클라우드 서비스
사내 구축 솔루션 커스터마이징 가능, 데이터 유출 위험 감소 개발 및 유지보수 비용 큼, 기능 발전 속도 느림 즉시 사용 가능한 완성도 높은 플랫폼, 지속적인 기능 업데이트
스프레드시트 + 로그 파일 간단함, 추가 도구 불필요 확장 불가능, 실수 많음, 재현성 낮음, 비교 분석 어려움 자동화된 추적, 중앙 집중식 관리, 재현성 보장

⚠️ 선택 기준

소규모 개인 프로젝트라면 MLflow나 TensorBoard로 시작해도 좋습니다. 그러나 팀 단위 협업, 수백 수천 번의 실험, 프로덕션 모델 관리가 필요하다면 W&B의 생산성 향상 효과가 매우 큽니다.

3. 핵심 개념

W&B의 아키텍처와 주요 컴포넌트를 이해합니다.

+-------------------+ logs +----------------------+ | Your Training |----------------->| W&B Local Agent | | Script (Python) | | (Optional) | +-------------------+ +-----------+----------+ | syncs +-------------------+ +-----------v----------+ | Config (params) | | W&B Cloud Service | | Metrics (loss) | | (Dashboard, API) | | Artifacts (model)|<-----------------+ | | System (GPU) | queries | - Projects | +-------------------+ & reports | - Experiments (Runs) | | - Artifacts | | - Sweeps | +----------------------+ | +-----------v----------+ | Team Collaboration | | (Compare, Share) | +----------------------+

주요 용어

  • Run: 한 번의 학습 실험 단위. 하이퍼파라미터, 메트릭, 로그가 기록됩니다.
  • Project: 관련된 Run들을 그룹화하는 컨테이너입니다.
  • Artifact: 모델 가중치, 데이터셋, 평가 결과 등 버전 관리가 필요한 파일이나 디렉토리입니다.
  • Sweep: 하이퍼파라미터 최적화를 자동화하는 기능입니다.
  • Dashboard: 프로젝트 내 Run들을 비교하고 시각화하는 웹 인터페이스입니다.

4. 퀵스타트

5분 안에 첫 실험을 W&B에 기록해보세요.

단계 1: 설치 및 로그인

Terminal
# pip를 통한 설치
pip install wandb

# CLI 로그인 (API 키 입력)
wandb login
# 또는 환경변수 설정
# export WANDB_API_KEY='your-api-key-here'

API 키는 https://wandb.ai/authorize에서 얻을 수 있습니다.

단계 2: 기본 Python 스크립트 작성

train.py
import wandb
import random

# 1. 실험 초기화
wandb.init(
    project="my-first-wandb-project", # 프로젝트명
    config={                          # 하이퍼파라미터 기록
        "learning_rate": 0.01,
        "architecture": "CNN",
        "dataset": "CIFAR-10",
        "epochs": 10,
    }
)

config = wandb.config

# 2. 모의 학습 루프
for epoch in range(config.epochs):
    # 모의 메트릭 생성
    loss = random.random() / (epoch + 1)
    accuracy = 1 - loss + random.random() / 10

    # 3. 메트릭 로깅
    wandb.log({
        "epoch": epoch,
        "loss": loss,
        "accuracy": accuracy,
        "val_loss": loss * 0.9,
        "val_accuracy": accuracy * 0.95
    })

    # 모의 모델 저장 (Artifact 예시)
    if epoch % 5 == 0:
        model_name = f"model_epoch_{epoch}.pth"
        # 실제로는 torch.save(model.state_dict(), model_name) 등
        with open(model_name, 'w') as f:
            f.write(f"mock model at epoch {epoch}")

        # 4. 아티팩트로 모델 업로드
        artifact = wandb.Artifact(name=f"run-model-{wandb.run.id}", type="model")
        artifact.add_file(model_name)
        wandb.log_artifact(artifact)

# 5. 실험 종료
wandb.finish()
print("실험이 W&B 대시보드에 동기화되었습니다!")

단계 3: 실행 및 결과 확인

python train.py

스크립트 실행 후 터미널에 출력된 URL(예: https://wandb.ai/your-username/my-first-wandb-project/runs/...)을 클릭하여 대시보드에서 실시간으로 메트릭과 아티팩트를 확인하세요.

5. 핵심 설정 레퍼런스

wandb.init() 및 주요 함수의 필수/자주 쓰는 설정 옵션입니다.

매개변수 기본값 설명 실무 예시
project None 실험을 그룹화할 프로젝트 이름. 필수. project="sentiment-analysis-2026"
name 랜덤 생성 Run의 고유 이름. 식별용. name="bert-base-0.001-lr"
config {} 하이퍼파라미터 딕셔너리. 자동 기록/비교. config={"lr":0.01, "batch":64}
entity 개인 계정 팀 또는 조직 이름. 협업 시 설정. entity="my-company-ml-team"
tags [] Run을 필터링하기 위한 태그 리스트. tags=["baseline", "transformer"]
notes "" Run에 대한 설명 메모. notes="첫 번째 베이스라인 실험"
resume False 중단된 Run 재개 시 "allow", "must", True 등. resume="allow"
mode "online" "online", "offline", "disabled". mode="offline" (인터넷 없을 때)

🔧 환경변수 설정 (고급)

Docker 또는 클라우드 환경에서 자주 사용합니다.

export WANDB_API_KEY=your_key
export WANDB_PROJECT=production-models
export WANDB_ENTITY=team-name
export WANDB_SILENT=true  # 로그 최소화
export WANDB_DIR=/path/to/logs  # 로컬 저장 경로

6. 실무 팁 & 트러블슈팅

🚀 실무 팁

오프라인 실행 후 동기화

인터넷이 안 되는 환경(클러스터)에서는 오프라인 모드로 실행한 후, 결과를 동기화하세요.

wandb.init(mode="offline")
# ... 학습 코드 ...
wandb.finish()
# 이후 동기화: wandb sync ./wandb/offline-run-...

대용량 파일은 Artifact로

모델 체크포인트, 데이터셋 등 대용량 파일은 wandb.log_artifact()를 사용하세요. wandb.log()에 파일을 직접 넣지 마십시오.

Sweep으로 HPO 자동화

하이퍼파라미터 최적화는 Sweep 기능을 사용하면 편리합니다. Bayesian 최적화, 그리드 서치 등을 지원합니다.

🔧 트러블슈팅

문제 원인 해결책
"wandb: Network error..." 연결 실패 프록시, 방화벽, 인터넷 문제. 1. mode="offline" 후 동기화.
2. export WANDB_BASE_URL=http://내부-서버 (온프레미스 설치 시)
3. 프록시 설정: export HTTPS_PROXY=...
실행이 느리다 너무 빈번한 wandb.log() 호출. 매 스텝마다 로깅하지 말고, N 스텝마다 또는 epoch 종료 시 한 번에 로깅하세요.
디스크 공간 부족 로컬 wandb/ 디렉토리에 캐시가 쌓임. 정기적으로 wandb artifact cache cleanup 실행. 또는 export WANDB_CACHE_DIR=/큰_디스크 설정.
PyTorch Lightning 통합 시 중복 실행 Trainer에서 logger=WandbLogger()를 설정하고, 스크립트에서도 wandb.init()을 호출하는 경우. PyTorch Lightning 사용 시 WandbLogger만 사용하고, 직접 wandb.init()을 호출하지 마세요.
동일한 Run 이름 충돌 고정된 이름으로 여러 작업을 병렬 실행. name 파라미터를 설정하지 말거나, uuid 등을 추가해 고유하게 만드세요. 또는 wandb.util.generate_id() 사용.
