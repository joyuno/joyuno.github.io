1. CrowdStrike Falcon란?

CrowdStrike Falcon은 클라우드 네이티브 아키텍처 기반의 차세대 엔드포인트 보안 플랫폼입니다. 단일 가벼운 에이전트(Falcon Sensor)를 통해 EDR(Endpoint Detection and Response), AV, Threat Intelligence, IT Hygiene 등을 통합 제공하는 XDR(eXtended Detection and Response) 솔루션입니다.

1. AI/ML 기반 위협 탐지

Falcon 플랫폼의 핵심은 클라우드 기반의 인공지능 엔진입니다. 1초에 1조 개 이상의 이벤트를 처리하며, 실시간으로 위협 인디케이터(IoCs)와 행위 기반 이상 징후를 분석합니다.

2. 단일 에이전트 아키텍처

기존 보안 제품군이 여러 에이전트를 필요로 했던 것과 달리, Falcon Sensor 하나로 모든 기능(방어, 탐지, 대응, 가시성)을 수행합니다. 리소스 사용률이 극도로 낮습니다.

3. 통합 XDR 플랫폼

엔드포인트, 클라우드 워크로드, ID, 이메일, 네트워크 데이터를 하나의 대시보드에서 통합하여 조사 및 대응할 수 있습니다. 데이터 실효화(Silo) 문제를 해결합니다.

2. 왜 써야 하는가? (기존 대안과 비교)

전통적인 시그니처 기반 AV와 EDR, 그리고 현대적인 XDR 플랫폼 간의 차이점을 이해하는 것이 중요합니다.

기능/특징 전통적 AV (예: McAfee) 기존 EDR (예: Carbon Black) CrowdStrike Falcon (XDR)
아키텍처 온프레미스 중앙 서버 하이브리드 (온프레미스+클라우드) 완전 클라우드 네이티브 (SaaS)
탐지 방식 시그니처, 휴리스틱 행위 기반 + 시그니처 AI/ML 기반 실시간 분석 + Threat Graph
에이전트 부하 높음 (주기적 전체 검사) 중간 매우 낮음 (스트리밍 이벤트)
대응 자동화 제한적 (격리/삭제) 플레이북 기반 반자동 완전 자동화 (SOAR 통합)
데이터 통합 엔드포인트만 엔드포인트 중심 통합 XDR (엔드포인트, 클라우드, ID, 네트워크)

핵심 차별점

Threat Graph: 모든 엔드포인트, 사용자, 프로세스 간의 관계를 실시간 그래프 데이터베이스로 모델링하여 공격자의 TTP(Tactics, Techniques, Procedures)를 시각적으로 추적할 수 있습니다. 이는 기존 제품이 단일 이벤트 로그만 제공하는 것과 근본적으로 다릅니다.

3. 핵심 개념 & 아키텍처

Falcon의 동작 원리를 이해하기 위한 간소화된 아키텍처 다이어그램입니다.

+-----------------------------------------------------------------------+ | CrowdStrike Falcon Cloud | | +----------------+ +----------------+ +----------------+ | | | AI/ML 엔진 | | Threat Intel | | Threat Graph | | | | (실시간 분석) | | (격리/공격지) | | (관계 매핑) | | | +----------------+ +----------------+ +----------------+ | | ^ ^ ^ | | | | | | +------------|-------------------|-------------------|----------------+ | | | | (REST API / Stream) | | | +------------|---------------------------------------|----------------+ | | | | | v v | | +----------------+ +----------------+ | | | Falcon Sensor | | Falcon Console| | | | (에이전트) |<------------------->| (관리 콘솔) | | | +----------------+ 이벤트 스트리밍 +----------------+ | | ^ | | | (커널/유저 모드 후크) | +-----------|--------------------------------------------------------+ | v +------------------------------------------------------------------------+ | 호스트 (엔드포인트) | | 프로세스 생성, 파일 쓰기, 네트워크 연결, 레지스트리 변경 등 모든 활동 | +------------------------------------------------------------------------+

주요 컴포넌트

  • Falcon Sensor: 커널 레벨에서 시스템 활동을 모니터링하고 이벤트를 클라우드로 스트리밍하는 가벼운 에이전트.
  • Falcon Cloud: 모든 분석, 상관관계, 위협 인텔리전스가 이루어지는 백엔드 플랫폼.
  • Threat Graph: 실시간 그래프 데이터베이스로, 모든 엔드포인트와 이벤트 간 관계를 저장/분석.
  • Falcon Console: 웹 기반 관리 인터페이스. 대시보드, 인시던트 관리, 조사 도구 제공.

4. 퀵스타트: 설치 → 설정 → 첫 예제

Falcon API를 이용한 기본적인 자동화 예제를 통해 플랫폼과의 상호작용을 시작해보세요.

  1. 사전 준비: API 클라이언트 생성

    Falcon Console에서 Support & Resources → API Clients and Keys로 이동하여 새 클라이언트를 생성합니다. Detections Read, Hosts Read 권한을 부여하세요.

    CLI: 환경변수 설정
    # Bash (Linux/macOS)
export FALCON_CLIENT_ID="your_client_id_here"
export FALCON_CLIENT_SECRET="your_client_secret_here"
export FALCON_CLOUD="us-1"  # 또는 eu-1, us-2 등 리전 코드

# PowerShell (Windows)
$env:FALCON_CLIENT_ID="your_client_id_here"
$env:FALCON_CLIENT_SECRET="your_client_secret_here"
$env:FALCON_CLOUD="us-1"

  2. Python SDK 설치 및 인증

    Python: 패키지 설치 및 기본 인증
    # crowdstrike-falconpy 패키지 설치
pip install crowdstrike-falconpy

# 기본 인증 및 서비스 클래스 생성
from falconpy import OAuth2, Hosts, Detects

# 인증 객체 생성
auth = OAuth2(client_id=os.getenv('FALCON_CLIENT_ID'),
              client_secret=os.getenv('FALCON_CLIENT_SECRET'),
              base_url=os.getenv('FALCON_CLOUD'))

# Hosts 및 Detects 서비스 클라이언트 생성
hosts = Hosts(auth_object=auth)
detects = Detects(auth_object=auth)

print("인증 성공!")

  3. 첫 번째 쿼리: 최근 위협 탐지 조회

    Python: 최근 10개 탐지 이벤트 가져오기
    def get_recent_detections(limit=10):
        """최근 탐지 이벤트를 조회합니다."""
        # 쿼리 파라미터 설정
        parameters = {
            'limit': limit,
            'sort': 'first_behavior|desc',  # 최신 순 정렬
            'filter': 'status:"new"'  # 새로 발생한 탐지만
        }
        
        try:
            response = detects.query_detects(parameters=parameters)
            if response['status_code'] == 200:
                detection_ids = response['body']['resources']
                if detection_ids:
                    details = detects.get_detect_summaries(
                        body={'ids': detection_ids}
                    )
                    return details['body']['resources']
                else:
                    print("최근 탐지 이벤트가 없습니다.")
                    return []
            else:
                print(f"쿼리 실패: {response['status_code']}")
                return []
        except Exception as e:
            print(f"오류 발생: {e}")
            return []

# 실행 예제
recent_detects = get_recent_detections(5)
for detect in recent_detects:
    print(f"[{detect.get('first_behavior', '')}] {detect.get('technique', '')}")
    print(f"  호스트: {detect.get('device', {}).get('hostname', 'N/A')}")
    print(f"  심각도: {detect.get('max_severity', 'N/A')}")
    print()

  4. 호스트 정보 일괄 조회

    Python: 특정 OS의 모든 호스트 정보 가져오기
    def get_hosts_by_os(os_platform="Windows"):
        """특정 OS 플랫폼의 호스트 목록을 조회합니다."""
        filter_query = f"platform_name:'{os_platform}'"
        
        response = hosts.query_devices_by_filter(filter=filter_query, limit=100)
        
        if response['status_code'] == 200:
            host_ids = response['body']['resources']
            if host_ids:
                host_details = hosts.get_device_details(ids=host_ids)
                return host_details['body']['resources']
        return []

# 실행 예제
windows_hosts = get_hosts_by_os("Windows")
print(f"총 {len(windows_hosts)}개의 Windows 호스트 발견")
for host in windows_hosts[:3]:  # 처음 3개만 출력
    print(f"- {host.get('hostname')} | {host.get('local_ip')} | "
          f"Sensor: {host.get('agent_version')}")

5. 핵심 설정 레퍼런스

Falcon Console에서 자주 설정하는 주요 정책과 그 의미입니다.

정책 카테고리 키 설정 권장값 설명
예방 정책 Cloud Machine Learning Block (High Threshold) 클라우드 ML이 악성으로 판단하면 실행 차단
On-Sensor Machine Learning Detect (Medium Threshold) 오프라인 환경을 위한 온디바이스 ML 탐지
탐지 정책 IOA (Indicator of Attack) Rules Enabled (All) 공격 지표 기반 실시간 탐지 활성화
Behavioral Threat Detection Enabled (Aggressive) 이상 행위 기반 탐지 민감도
데이터 수집 Sensor Data Collection Enhanced (or Full) 조사를 위한 상세 이벤트 수집 레벨
대응 정책 Automated Response Contain (Critical Severity) 심각도 Critical 시 자동 격리
관리 Sensor Update Policy Maintenance Window 설정 에이전트 자동 업데이트 시간대

주의: 데이터 수집 레벨 트레이드오프

Enhanced/Full 모드는 포렌식 조사에 필수적이지만, 네트워크 대역폭과 클라우드 스토리지 사용량을 증가시킵니다. 조직의 네트워크 용량과 보안 요구사항을 고려하여 균형을 잡아 설정하세요.

6. 실무 팁 & 트러블슈팅

자주 겪는 문제

문제 1: Sensor 상태 "Offline"

증상: 호스트가 Falcon Console에서 오프라인으로 표시됨.

원인: 네트워크 연결 문제, Sensor 프로세스 중단, 호스트 방화벽 정책.

해결: 1. 호스트에서 sc query csfalconservice (Windows) 또는 systemctl status falcon-sensor (Linux)로 서비스 상태 확인. 2. Sensor 로그 확인: C:\ProgramData\CrowdStrike\Logs 또는 /var/log/crowdstrike/. 3. 아웃바운드 방화벽이 *.crowdstrike.com:443 허용하는지 확인.

문제 2: API 호출 Rate Limit 초과

증상: 429 Too Many Requests 에러 반환.

원인: Falcon API는 분당/일별 호출 제한이 있음.

해결: 1. 응답 헤더의 X-Ratelimit-Remaining 확인. 2. 대량 작업 시 time.sleep()으로 호출 간 지연 추가. 3. 가능하면 배치 API (예: /devices/entities/devices/v2) 사용.

문제 3: False Positive 탐지

증상: 합법적인 내부 도구가 악성으로 탐지됨.

원인: ML 모델이 특정 행위 패턴을 악성으로 오판별.

해결: 1. 탐지 이벤트에서 "Exclude" 버튼으로 일시적 제외. 2. IOC Management에서 해당 파일 해시를 허용 목록에 추가. 3. 필요한 경우 Prevention Policy의 ML 민감도 조정.

성능 최적화 팁

  • API 사용: 반복적인 호출은 캐시 구현 (예: 호스트 정보는 5분 캐시).
  • 대량 조회: limit 파라미터와 offset을 사용한 페이지네이션 구현.
  • 실시간 모니터링: Detection Streams API를 사용한 이벤트 스트리밍 (WebSocket).
  • 자동화: Critical 심각도 탐지 시 Slack/Teams 알림 자동화 구축.

팁: Detection Streams API로 실시간 모니터링
