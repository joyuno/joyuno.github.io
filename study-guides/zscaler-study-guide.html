1. Zscaler란?

Zscaler는 클라우드 기반의 제로 트러스트(Zero Trust) 보안 플랫폼입니다. 기존 VPN/VPN 어플라이언스를 대체하며, 사용자와 애플리케이션을 직접 연결하지 않고 클라우드 브로커를 통해 안전하게 접속합니다. AI 보안 엔지니어에게는 데이터 유출 방지, 위협 인텔리전스, SASE 아키텍처의 핵심 컴포넌트로 이해됩니다.

3대 핵심 능력

🔹 ZTNA (Zero Trust Network Access)

사용자 신원과 디바이스 상태를 기반으로 앱별 접근 제어. 내부 네트워크에 노출 없음.

🔹 SWG (Secure Web Gateway)

모든 웹 트래픽을 검사, 위협 차단 및 데이터 손실 방지(DLP) 정책 적용.

🔹 CASB (Cloud Access Security Broker)

SaaS 앱 사용 모니터링, 섀도 IT 탐지, 데이터 보호 정책 중앙 관리.

💡 AI 보안 관점: Zscaler의 트래픽 로그와 위협 인텔리전스는 머신러닝 기반 이상 탐지, 사용자 행동 분석(UEBA)에 바로 활용 가능.

2. 왜 써야 하는가?

기존 VPN, 방화벽 어플라이언스 대비 Zscaler의 차별점:

항목기존 VPN / 방화벽Zscaler (ZTNA/SASE)
아키텍처하드웨어 중앙 집중형클라우드 에지 분산 (150+ PoP)
접근 모델네트워크 기반 (IP/포트)앱 기반 제로 트러스트
데이터 검사제한적 (성능 이슈)전 트래픽 SSL 검사 + DLP
AI/ML 활용제한적 시그니처위협 인텔리전스 + 행동 분석
운영 복잡도높음 (패치, 용량)낮음 (완전 관리형)
⚠️ 주의: Zscaler는 인터넷 트래픽이 항상 Zscaler 클라우드를 경유하므로 대역폭 및 지연에 대한 설계가 필요합니다.

3. 핵심 개념 & 아키텍처

Zscaler의 동작 방식: 사용자 → Zscaler Client Connector (ZCC) → Zscaler Cloud (PoP) → 앱

┌─────────────────────────────────────────────────────────────┐ │ Zscaler Cloud (SASE PoP) │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ │ │ SWG │ │ ZTNA │ │ CASB │ │ │ │ (웹 필터)│ │ (앱 터널)│ │ (SaaS) │ │ │ └────┬─────┘ └────┬─────┘ └────┬─────┘ │ │ │ │ │ │ │ ┌────▼──────────────▼──────────────▼─────┐ │ │ │ 정책 엔진 & 위협 인텔리전스 │ │ │ └─────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────────┘ ▲ │ │ GRE / IPSec / ZCC │ HTTPS ┌───────────┴──────────┐ ┌──────────▼──────────┐ │ 사용자 / 디바이스 │ │ 애플리케이션 │ │ (ZCC 에이전트) │ │ (SaaS, on-prem) │ └───────────────────────┘ └─────────────────────┘

핵심 컴포넌트: ZCC (Zscaler Client Connector), ZPA (Zscaler Private Access), ZIA (Zscaler Internet Access), 정책 엔진, 로그/분석 파이프라인.

4. 퀵스타트

🔧 설치 → 설정 → 첫 예제 (Python + CLI)

1) ZCC 에이전트 설치 (macOS 예시)

# CLI를 통한 설치 (관리자 권한)
sudo installer -pkg Zscaler.pkg -target /
# 또는 MDM 배포 (JAMF, Intune)

2) Zscaler API 인증 (Python)

import requests, json

# Zscaler API 키와 비밀 (관리자 콘솔에서 생성)
API_KEY = "your_api_key"
SECRET  = "your_secret"
BASE_URL = "https://api.zscaler.com"

# OAuth2 토큰 발급
payload = {"apiKey": API_KEY, "secret": SECRET}
resp = requests.post(f"{BASE_URL}/auth/v1/login", json=payload)
token = resp.json()["token"]
headers = {"Authorization": f"Bearer {token}"}

# 정책 목록 조회 (예시)
policies = requests.get(f"{BASE_URL}/policy/v1/urlCategories", headers=headers)
print(policies.json()[:2])

3) CLI를 통한 로그 확인 (zdiagnostic)

# Zscaler 진단 도구 (ZCC 번들)
zdiagnostic --collect-logs
# 실시간 트래픽 확인 (터널 상태)
zcc status --verbose
✅ 첫 예제 완료: Zscaler API로 URL 카테고리를 조회하고, 에이전트 상태를 CLI로 확인.

5. 핵심 설정 레퍼런스

설정 항목설명권장 값 / 예시
ZCC 터널 모드터널링 프로토콜 (Z-Tunnel 2.0)Z-Tunnel 2.0 (UDP/443)
SSL 검사트래픽 복호화 정책모든 트래픽 검사 (예외: 금융, 헬스)
DLP 규칙데이터 유출 방지 패턴PCI, PII, 소스코드 정규식
로그 전송SIEM / 데이터 레이크 연동NATS / S3 / Splunk HEC
AI 위협 탐지머신러닝 기반 이상 징후Advanced Threat Protection ON
⚠️ 실무: SSL 검사 시 인증서 체인을 사내 CA로 배포해야 사용자 경고가 발생하지 않습니다.
