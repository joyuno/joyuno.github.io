1. Cloudflare WAF란?

Cloudflare WAF(Web Application Firewall)는 클라우드 기반의 웹 애플리케이션 방화벽으로, 악성 트래픽을 필터링하고 웹 애플리케이션을 OWASP Top 10을 포함한 다양한 공격으로부터 보호합니다. 전역 Anycast 네트워크를 통해 초저지연 보안 서비스를 제공합니다.

1. 규칙 기반 필터링

사용자 정의 규칙(Cloudflare Rules) 및 관리형 규칙셋(Managed Rulesets)을 통해 세밀한 트래픽 제어가 가능합니다. SQLi, XSS, 패스 트래버설 등 다양한 공격 시그니처를 차단합니다.

2. 머신러닝 기반 보호

Cloudflare의 글로벌 위협 인텔리전스와 AI/ML 모델을 활용하여 제로데이 공격 및 변종 공격을 탐지합니다. WAF Attack Score를 통해 의심스러운 요청을 점수화합니다.

3. API-First 보안

RESTful API와 Terraform Provider를 제공하여 보안 정책을 코드(Infrastructure as Code)로 관리할 수 있습니다. CI/CD 파이프라인과 통합이 용이합니다.

2. 왜 써야 하는가?

기존 온프레미스 WAF 및 다른 클라우드 WAF 대비 Cloudflare WAF의 차별화된 장점을 비교합니다.

비교 항목 Cloudflare WAF 전통적 온프레미스 WAF 타 클라우드 WAF
배포 모델 글로벌 Anycast 네트워크 (Reverse Proxy) 인라인 어플라이언스 리전별 배포
성능 영향 극히 낮음 (Edge 처리) 높음 (지연 증가) 중간 (리전 내 처리)
규칙 업데이트 실시간 자동 업데이트 수동 업데이트 필요 주기적 업데이트
DDoS 보호 통합 네이티브 통합 (무제한 계층) 별도 솔루션 필요 제한적 통합
API 관리 완전한 API 지원 (Terraform, SDK) 제한적 부분적
비용 효율성 사용량 기반, 예측 가능 고정 CAPEX + 유지보수 비용 트래픽 양에 따라 급증 가능

3. 핵심 개념

Cloudflare WAF의 아키텍처와 핵심 구성 요소를 ASCII 다이어그램으로 시각화합니다.

┌─────────────────────────────────────────────────────────────┐ │ Client Request │ └───────────────────────────┬─────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────┐ │ Cloudflare Global Network │ │ ┌────────────┐ ┌────────────┐ ┌────────────┐ │ │ │ Edge │ │ Edge │ │ Edge │ │ │ │ PoP 1 │ │ PoP 2 │ │ PoP N │ │ │ └─────┬──────┘ └─────┬──────┘ └─────┬──────┘ │ │ │ │ │ │ │ ┌─────▼──────┐ ┌─────▼──────┐ ┌─────▼──────┐ │ │ │ WAF │ │ WAF │ │ WAF │ │ │ │ Engine │ │ Engine │ │ Engine │ │ │ └─────┬──────┘ └─────┬──────┘ └─────┬──────┘ │ │ │ │ │ │ │ ┌─────▼──────┐ ┌─────▼──────┐ ┌─────▼──────┐ │ │ │ DDoS │ │ Rate │ │ Managed │ │ │ │ Protection│ │ Limiting │ │ Rulesets │ │ │ └─────┬──────┘ └─────┬──────┘ └─────┬──────┘ │ │ │ │ │ │ └────────┼───────────────┼───────────────┼──────────────────┘ │ │ │ ▼ ▼ ▼ ┌────────┼───────────────┼───────────────┼──────────────────┐ │ │ │ │ │ │ ┌─────▼──────┐ ┌─────▼──────┐ ┌─────▼──────┐ │ │ │ Custom │ │ ML-Based │ │ Logging │ │ │ │ Rules │ │ Filtering │ & Analytics │ │ │ └─────┬──────┘ └─────┬──────┘ └─────┬──────┘ │ │ │ │ │ │ └────────┼───────────────┼───────────────┼──────────────────┘ │ │ │ └───────────────┼───────────────┘ ▼ ┌─────────────────────────────────────────────────────────────┐ │ Origin Server │ └─────────────────────────────────────────────────────────────┘

핵심 용어 설명

Managed Rulesets: Cloudflare에서 관리하는 사전 정의된 규칙 세트 (OWASP Core, Cloudflare Managed, WordPress 등).

Custom Rules: 사용자가 작성하는 조건-동작 기반의 규칙 (필드, 연산자, 값 구성).

WAF Attack Score: 요청의 악성도를 0-100 점수로 평가, 동적 차단에 활용.

Rate Limiting: IP, 세션, 국가별 요청 수 제한으로 Brute Force 방지.

4. 퀵스타트

Cloudflare WAF를 빠르게 설정하고 첫 번째 보안 규칙을 배포하는 단계별 가이드입니다.

    단계 1: 도메인 추가 및 DNS 설정

    Cloudflare 대시보드에서 도메인을 추가하고 네임서버를 변경합니다.

    CLI
    # Cloudflare CLI (flarectl)을 사용한 도메인 추가 예시
flarectl zone create --zone=example.com --jump-start=false

    단계 2: WAF 기본 설정 활성화

    Security > WAF 메뉴에서 관리형 규칙셋을 활성화합니다.

    Python
    import requests

CLOUDFLARE_API_TOKEN = "your-api-token"
ZONE_ID = "your-zone-id"
headers = {
    "Authorization": f"Bearer {CLOUDFLARE_API_TOKEN}",
    "Content-Type": "application/json"
}

# OWASP Core 규칙셋 활성화
payload = {
    "rulesets": [
        {
            "id": "481f4f5f6f1c4a0a8f1c4a0a8f1c4a0a8",  # OWASP Core Ruleset ID
            "action": "block",
            "enabled": True
        }
    ]
}

response = requests.patch(
    f"https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/rulesets",
    headers=headers,
    json=payload
)
print(response.json())

    단계 3: 첫 번째 커스텀 규칙 생성

    SQL Injection 시도를 차단하는 커스텀 규칙을 생성합니다.

    Python
    import requests

CLOUDFLARE_API_TOKEN = "your-api-token"
ZONE_ID = "your-zone-id"
headers = {
    "Authorization": f"Bearer {CLOUDFLARE_API_TOKEN}",
    "Content-Type": "application/json"
}

# SQL Injection 패턴 차단 규칙
rule_payload = {
    "name": "Block SQLi Attempts",
    "description": "차단 SQL Injection 패턴",
    "expression": """
        http.request.uri.query contains "union" or 
        http.request.uri.query contains "select" or
        http.request.uri.query contains "insert" or
        http.request.uri.query contains "drop" or
        http.request.uri.query contains "delete"
    """,
    "action": "block",
    "enabled": True
}

response = requests.post(
    f"https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/rulesets/phases/http_request_firewall_custom/entrypoint",
    headers=headers,
    json=rule_payload
)
print(response.json())

    단계 4: Rate Limiting 설정

    /admin 경로에 대한 Brute Force 공격 방지를 위한 Rate Limiting 설정

    CLI
    # Cloudflare API를 사용한 Rate Limiting 규칙 생성
curl -X POST "https://api.cloudflare.com/client/v4/zones/{zone_id}/rate_limits" \
-H "Authorization: Bearer $CLOUDFLARE_API_TOKEN" \
-H "Content-Type: application/json" \
-d '{
  "description": "Protect admin login from brute force",
  "match": {
    "request": {
      "methods": ["POST"],
      "schemes": ["HTTP", "HTTPS"],
      "url_pattern": "*/admin/login"
    }
  },
  "period": 60,
  "threshold": 10,
  "action": {
    "mode": "simulate",
    "timeout": 600,
    "response": {
      "content_type": "application/json",
      "body": "{\"error\":\"Rate limit exceeded\"}"
    }
  }
}'

5. 핵심 설정 레퍼런스

Cloudflare WAF의 주요 설정 옵션과 권장값을 정리한 레퍼런스 테이블입니다.

설정 항목 권장값 설명 API 필드
Security Level Medium 익명 IP 및 신뢰할 수 없는 소스에 대한 도전 수준 security_level
Bot Fight Mode On 알려진 봇 트래픽 자동 차단 bot_fight_mode
OWASP Core Ruleset Block OWASP Top 10 공격에 대한 기본 보호 rulesets[].action
Managed Ruleset Sensitivity Medium 관리형 규칙의 민감도 (Low/Medium/High) sensitivity
WAF Attack Score Threshold 30 차단을 트리거할 최소 공격 점수 (0-100) score_threshold
Rate Limiting (일반 API) 1000 req/10min API 엔드포인트 기본 Rate Limit threshold/period
Logging Retention 30 days WAF 이벤트 로그 보관 기간 retention
Anomaly Detection On 비정상 트래픽 패턴 자동 탐지 anomaly_detection

6. 실무 팁 & 트러블슈팅

팁 1: 스테이징 환경에서 먼저 테스트

모든 WAF 규칙은 스테이징 환경에서 "Log" 모드로 테스트한 후 프로덕션에 배포하세요. False Positive로 인한 서비스 장애를 방지할 수 있습니다.

팁 2: 규칙 우선순위 관리

Cloudflare WAF는 규칙을 순차적으로 평가합니다. 특정 규칙이 다른 규칙보다 먼저 평가되도록 우선순위를 설정하세요. 글로벌 규칙보다 로컬 규칙이 우선 적용됩니다.

문제 1: False Positive 차단

증상: 정상 사용자가 차단됨
원인: 너무 광범위한 규칙 패턴, 높은 민감도 설정
해결책:
1. 차단 이벤트 로그에서 트리거된 규칙 ID 확인
2. 규칙 패턴을 구체화 (예: 특정 경로만 적용)
3. "skip" 동작으로 예외 IP 추가
4. Attack Score 임계값 조정

문제 2: 성능 저하

증상: 응답 시간 증가
원인: 복잡한 정규식, 과도한 규칙 수
해결책:
1. 규칙 수 최소화 (필요한 규칙만 활성화)
2. 정규식 대신 단순 문자열 매칭 사용
3. Rate Limiting 규칙 최적화
4. 캐싱 계층 활용 증가

팁 3: 모니터링과 알림 설정

Cloudflare Analytics와 웹훅을 연동하여 WAF 이벤트에 대한 실시간 알림을 설정하세요. 주요 메트릭:
- 총 차단 요청 수
- 상위 차단 규칙
- 공격 소스 국가
- Attack Score 분포

문제 3: API 통합 실패

증상: Terraform/API로 규칙 배포 실패
원인: API 토큰 권한 부족, 잘못된 Zone ID
해결책:
1. API 토큰에 필요한 권한 확인 (Zone.WAF:Edit)
2. Zone ID가 올바른지 확인
3. API 요청 제한 초과 여부 확인 (1200 req/5min)
4. Terraform 상태 파일 동기화
