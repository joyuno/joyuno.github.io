1. Weaviate란?

Weaviate는 오픈소스 벡터 데이터베이스로, 객체 + 벡터를 함께 저장하고, 벡터 유사도 검색과 GraphQL을 통한 풍부한 쿼리가 가능한 하이브리드 데이터베이스입니다. ML 모델을 통한 자동 벡터화(모듈)와 멀티모달 데이터 지원이 핵심 강점입니다.

🔍 하이브리드 검색

BM25(키워드) + 벡터 유사도를 결합한 검색으로, 정확도와 리콜을 동시에 높입니다.

🤖 모듈 시스템

text2vec, img2vec, multi2vec 등 모듈을 연결해 데이터 수집 시점에 자동 벡터화를 수행합니다.

🔗 GraphQL First

단일 쿼리로 객체 속성, 벡터, 참조 관계, 집계 등을 모두 조회할 수 있습니다.

2. 왜 써야 하는가? (vs. 기존 대안)

솔루션장점단점 / Weaviate 차별점
Elasticsearch + 플러그인성숙한 생태계, 풀텍스트 검색벡터 검색은 부가기능, 운영 복잡도 증가
Pinecone / Chroma완전관리형, 사용 간편Lock-in, 객체-관계 모델링 제한적
PostgreSQL (pgvector)ACID, SQL 친숙함하이브리드 검색, 모듈화, 성능 최적화 부족
Weaviate하이브리드 검색, 모듈 자동화, GraphQL, 오픈소스운영 부담 존재 (Self-hosted 기준)
✅ Weaviate 선택 포인트: RAG, 멀티모달 검색, 실시간 하이브리드 검색이 필요하고, 오픈소스 기반의 유연한 데이터 모델링을 원할 때.

3. 핵심 개념 & 아키텍처

핵심 용어

  • Class: RDBMS의 Table, MongoDB의 Collection에 해당하는 데이터 스키마.
  • Property: Class의 필드 (텍스트, 숫자, 벡터, 참조 등).
  • Vectorizer Module: 데이터를 벡터로 변환하는 모듈 (e.g., text2vec-transformers).
  • GraphQL: 데이터 조회/검색의 주요 API 인터페이스.

아키텍처 다이어그램

+-------------------+ HTTP / gRPC +----------------------+ | Client App | -------------------> | Weaviate | | (Python, JS, ...)| GraphQL / REST | Server Node | +-------------------+ +-----------+----------+ | +---------------v--------------+ | Storage & Compute Layers | | +----------------------+ | | | Vector Index | | | | (HNSW) | | | +----------------------+ | | +----------------------+ | | | Inverted Index | | | | (BM25) | | | +----------------------+ | | +----------------------+ | | | Object Store | | | | (on-disk/ S3) | | | +----------------------+ | +---------------+--------------+ | +---------------v--------------+ | Modules (Plugins) | | +----------------------+ | | | text2vec-* | | | | img2vec-* | | | | qna-transformers | | | +----------------------+ | +-------------------------------+

4. 퀵스타트: 설치부터 첫 벡터 검색까지

Step 1: Docker로 로컬 실행

CLI
# 최신 버전 (모듈 포함) 실행
docker run -d \
  -p 8080:8080 \
  -p 50051:50051 \
  --name weaviate \
  -e QUERY_DEFAULTS_LIMIT=25 \
  -e AUTHENTICATION_ANONYMOUS_ACCESS_ENABLED=true \
  -e PERSISTENCE_DATA_PATH='/var/lib/weaviate' \
  cr.weaviate.io/semitechnologies/weaviate:latest

Step 2: Python 클라이언트 설치 및 스키마 생성

Python
pip install weaviate-client
import weaviate
import json

client = weaviate.Client("http://localhost:8080")

# 클래스(테이블) 스키마 정의
schema = {
    "class": "Document",
    "vectorizer": "text2vec-transformers", # 벡터화 모듈 지정
    "properties": [
        {
            "name": "title",
            "dataType": ["text"]
        },
        {
            "name": "content",
            "dataType": ["text"]
        }
    ]
}

client.schema.create_class(schema)
print("✅ 클래스 생성 완료")

Step 3: 데이터 임포트 & 벡터 검색

# 데이터 객체 추가 (벡터는 모듈이 자동 생성)
client.data_object.create(
    data_object={
        "title": "벡터 데이터베이스 개요",
        "content": "Weaviate는 객체와 벡터를 함께 저장하는 하이브리드 데이터베이스입니다."
    },
    class_name="Document"
)

# GraphQL을 이용한 하이브리드(벡터+키워드) 검색
query = """
{
  Get {
    Document(
      hybrid: {
        query: "하이브리드 데이터베이스",
        alpha: 0.5 # 0: 키워드 검색, 1: 벡터 검색
      }
      limit: 5
    ) {
      title
      content
      _additional {
        score
      }
    }
  }
}
"""

result = client.query.raw(query)
print(json.dumps(result, indent=2, ensure_ascii=False))

5. 핵심 설정 레퍼런스

설정기본값설명실무 권장
vectorizernone벡터화 모듈text2vec-transformers (로컬), text2vec-openai (클라우드)
vectorIndexTypehnsw벡터 인덱스 알고리즘hnsw 유지. ef/efConstruction 조정으로 정확도/속도 트레이드오프
efConstruction128인덱스 빌드 시 정확도값 ↑ → 정확도 ↑, 빌드 시간 ↑ (256~512)
ef-1검색 시 탐색 노드 수값 ↑ → 정확도 ↑, 쿼리 속도 ↓ (동적 조정 권장)
flatSearch.cutoff5000전수검색으로 전환하는 임계치소규모 데이터셋(5천개 이하)에서 정확도 최대화

6. 실무 팁 & 트러블슈팅

💡 팁 1: 배치 임포트 사용
대량 데이터 삽입 시 단일 요청보다 배치 임포트를 사용하면 성능이 10배 이상 향상됩니다. 
client.batch.configure(batch_size=100, dynamic=True)
with client.batch as batch:
    for item in data:
        batch.add_data_object(item, "Document")
⚠️ 주의 1: 벡터 차원 불일치
모듈 변경 시 벡터 차원이 달라져 기존 데이터 검색이 불가능할 수 있습니다. 모듈 전환 시 재색인 필요.
🔧 트러블슈팅: 쿼리 속도 저하
  1. ef 값을 낮춰보기 (기본값 -1은 동적 할당).
  2. 필요한 속성만 쿼리 (_additional { vector } 제외).
  3. 벡터 인덱스와 객체 저장소가 동일 디스크에 있어 I/O 경합 발생하지 않는지 확인.
