1. SentinelOne Singularity란?

SentinelOne Singularity는 AI 기반의 통합 확장 탐지 및 대응(XDR) 플랫폼으로, 엔드포인트, 클라우드, ID, 네트워크 등 모든 공격 표면을 하나의 콘솔에서 자율적으로 보호합니다. 정적 시그니처에 의존하지 않고 행위 기반 AI 엔진(Storyline)을 통해 위협을 실시간으로 예방, 탐지, 대응합니다.

자율적 운영 (Autonomous Operations)

AI 에이전트가 엔드포인트에서 직접 위협을 분석하고, 필요한 경우 자동으로 격리/치료를 수행합니다. 운영자 개입을 최소화하는 '핸즈오프' 보안을 실현합니다.

통합 가시성 (Unified Visibility)

Singularity Data Lake에 모든 보안 데이터(엔드포인트, 클라우드, 네트워크 로그)를 수집·상관관계 분석하여 공격 스토리(Storyline)를 재구성하고 단일 창에서 관리합니다.

예측적 대응 (Predictive Response)

Purple AI(내장형 생성형 AI)를 통해 자연어로 위협을 조회하고, 공격 TTP를 분석하며, 자동화된 플레이북을 실행하여 평균 대응 시간(MTTR)을 분 단위로 단축합니다.

2. 왜 써야 하는가? (기존 대안과 비교)

기존 EDR/AV 솔루션과의 핵심 차별점은 AI의 자율성과 데이터 통합에 있습니다.

기능/특징 SentinelOne Singularity 전통적 EDR (CrowdStrike 등) 기존 AV
탐지 엔진 행위 기반 AI (정적 시그니처 무관) 시그니처 + IOC + 행위 일부 시그니처 의존적
대응 자동화 완전 자율 치료 (에이전트 수준) 반자동 (콘솔에서 수동 승인) 격리만 가능 (치료 없음)
데이터 통합 Singularity Data Lake (모든 로그 통합) 주로 엔드포인트 로그만 해당 없음
AI 조수 Purple AI (내장형 GenAI) 타사 SOAR/챗봇 연동 필요 없음
클라우드 네이티브 완전 SaaS, 에이전트 무중단 업데이트 하이브리드/온프레미스 구성 복잡 주로 온프레미스

핵심 선택 기준

복잡한 멀티클라우드 환경, 보안 운영 인력 부족, 실시간 자동 대응이 필요할 경우 Singularity가 최적입니다. 기존 EDR에서 '알람 피로'와 수동 대응 지연을 겪고 있다면 전환 고려가 필요합니다.

3. 핵심 개념 & 아키텍처

Singularity 플랫폼은 크게 에이전트(컬렉터), 클라우드 콘솔(분석 엔진), Data Lake(저장소)로 구성됩니다.

+-----------------------------------------------------------------------+ | SentinelOne Singularity Cloud | | +-------------------+ +-------------------+ +-------------------+ | | | Purple AI | | Management | | Singularity | | | | (GenAI 조수) | | Console | | Data Lake | | | | | | | | (통합 로그) | | | +-------------------+ +-------------------+ +-------------------+ | | | | | | | | +-------+-------+ | | | | | AI 엔진 | | | | | | (Storyline) | | | | | +-------+-------+ | | +-------------------------------|---------------------------------------+ | API (REST/GraphQL) +-------------------------------|---------------------------------------+ | 엔드포인트 (Windows, Linux, macOS, 모바일) | | +------------------------------------------------------------+ | | | SentinelOne Agent | | | | +-----------+ +-----------+ +-----------+ +-----------+ | | | | | EPP 모듈 | | EDR 모듈 | | CWPP 모듈 | | 행위 AI | | | | | | (방지) | | (탐지) | | (워크로드)| | (실시간) | | | | | +-----------+ +-----------+ +-----------+ +-----------+ | | | +------------------------------------------------------------+ | | | | | | | 프로세스 제어 메모리 분석 파일 시스템 모니터링 | +-------------------------------------------------------------------+

주요 컴포넌트 설명

  • SentinelOne Agent: 경량 통합 에이전트. EPP, EDR, CWPP 기능을 모두 포함하며, 로컬 AI 엔진으로 실시간 위협 판단 및 자동 치료 수행.
  • Storyline Technology: 관련 이벤트를 하나의 공격 스토리로 자동 연결하여 조사 부담을 99% 줄임.
  • Singularity Data Lake: 모든 원본 로그를 무제한 보관하며, OpenSearch 호환 API로 직접 쿼리 가능.
  • Purple AI: 플랫폼 내장 생성형 AI. 자연어로 조사 쿼리 생성, 리포트 작성, 플레이북 자동화 지원.

4. 퀵스타트: 설치 → 설정 → 첫 예제

4.1 관리 콘솔 접속 및 API 키 발급

Singleton 콘솔(https://[tenant].sentinelone.net)에 로그인 후, 설정 → API 토큰에서 토큰 생성.

# 환경 변수 설정 (Linux/macOS)
export S1_API_TOKEN="your_api_token_here"
export S1_API_URL="https://[tenant].sentinelone.net"

# Windows (PowerShell)
$env:S1_API_TOKEN="your_api_token_here"
$env:S1_API_URL="https://[tenant].sentinelone.net"

4.2 Python SDK를 이용한 첫 연결

# sentinelone_sdk.py
import requests
import json
import os

class SentinelOneClient:
    def __init__(self):
        self.api_token = os.getenv('S1_API_TOKEN')
        self.base_url = os.getenv('S1_API_URL')
        self.headers = {
            'Authorization': f'ApiToken {self.api_token}',
            'Content-Type': 'application/json'
        }
    
    def get_agents(self, limit=10):
        """활성 에이전트 목록 조회"""
        endpoint = f"{self.base_url}/web/api/v2.1/agents"
        params = {
            'limit': limit,
            'sortBy': 'lastSeen',
            'sortOrder': 'desc'
        }
        response = requests.get(endpoint, headers=self.headers, params=params)
        response.raise_for_status()
        return response.json()
    
    def run_deep_visibility_query(self, query):
        """Deep Visibility (Data Lake) 쿼리 실행"""
        endpoint = f"{self.base_url}/web/api/v2.1/dv/init-query"
        payload = {
            'query': query,
            'fromDate': '2026-03-01T00:00:00.000Z',
            'toDate': '2026-03-15T23:59:59.999Z'
        }
        response = requests.post(endpoint, json=payload, headers=self.headers)
        response.raise_for_status()
        query_id = response.json()['data']['queryId']
        
        # 결과 가져오기
        result_endpoint = f"{self.base_url}/web/api/v2.1/dv/query-results"
        result_params = {'queryId': query_id, 'limit': 50}
        result_response = requests.get(result_endpoint, headers=self.headers, params=result_params)
        return result_response.json()

if __name__ == "__main__":
    client = SentinelOneClient()
    
    # 에이전트 상태 확인
    agents = client.get_agents(5)
    print(f"총 에이전트 수: {agents['pagination']['totalItems']}")
    for agent in agents['data']:
        print(f"- {agent['computerName']} ({agent['osType']}) - 상태: {agent['threatHealthStatus']}")
    
    # Deep Visibility 예제 쿼리: PowerShell 실행 이벤트
    query = """
    {
        "queryType": "events",
        "fields": ["timestamp", "agentName", "processName", "cmdline"],
        "filters": {
            "operator": "AND",
            "operands": [
                {"fieldName": "processName", "operator": "EQUALS", "value": "powershell.exe"},
                {"fieldName": "eventType", "operator": "EQUALS", "value": "Process Creation"}
            ]
        }
    }
    """
    print("\nDeep Visibility 쿼리 실행 중...")
    dv_results = client.run_deep_visibility_query(query)
    print(f"조회된 이벤트: {len(dv_results.get('data', []))}개")

5. 핵심 설정 레퍼런스

설정 영역 키 설정값 권장값/설명 CLI/API 경로
에이전트 정책 autoMitigationAction "kill" (위협 프로세스 즉시 종료) 또는 "quarantine" (격리) /web/api/v2.1/policies
네트워크 격리 networkIsolationEnabled true (중요 호스트 위협 시 자동 격리) /web/api/v2.1/policies/{id}
스캔 예약 scanSchedule "0 2 * * *" (매일 오전 2시 전체 스캔) 콘솔 정책 편집기
로그 수준 agentLogLevel "info" (운영), "debug" (트러블슈팅 시) sentinelctl config set
Data Lake 보존 dataRetentionDays 365 (기본), 필요시 무제한 확장 가능 콘솔 설정 → Data Lake
Purple AI 접근 purpleAIEnabled true (모든 분석가에게 활성화 권장) /web/api/v2.1/settings/purple-ai
외부 연동 (SIEM) siemForwarding Splunk, QRadar, Azure Sentinel 엔드포인트 콘솔 설정 → 연동

정책 설정 팁

처음 도입 시 autoMitigationAction을 "quarantine"으로 설정하여 오탐(false positive) 시 영향을 최소화하세요. 운영에 익숙해진 후 "kill"로 변경하는 것이 안전합니다. 중요 서버 그룹에는 별도의 정책을 적용하여 네트워크 격리 기능을 필수로 활성화하세요.

6. 실무 팁 & 트러블슈팅

자주 겪는 문제와 해결책

문제 1: 에이전트가 "Disconnected" 상태

증상: 콘솔에서 에이전트 연결이 끊어지고, 최근 활동 시간이 오래됨.

해결 단계:

  1. 호스트에서 에이전트 서비스 상태 확인: sudo systemctl status sentinelone-agent
  2. 네트워크 연결 확인 (아웃바운드 443 포트): curl -v https://[tenant].sentinelone.net
  3. 에이전트 로그 확인: sudo journalctl -u sentinelone-agent -f
  4. 에이전트 재등록: sudo sentinelctl reregister

문제 2: 오탐(False Positive)으로 정상 프로세스 차단

증상: 내부 개발 도구나 특정 업무 앱이 위협으로 탐지되어 차단됨.

해결 단계:

  1. 콘솔에서 해당 위협 이벤트로 이동 → "Actions" → "Mark as Resolved" (False Positive 선택)
  2. 동일한 파일/프로세스를 글로벌 예외로 추가: 정책 → 예외 → 파일/폴더 예외 추가
  3. 해시(SHA256) 기반 예외가 가장 안전하며, 서명된 인증서 예외도 활용 가능

문제 3: Deep Visibility 쿼리 성능 저하

증상: 대용량 쿼리 실행 시 타임아웃 또는 매우 느린 응답.

해결 단계:

  1. 쿼리 시간 범위를 좁히기 (7일 이내 권장)
  2. 필드 선택을 명시적으로 제한: "fields": ["timestamp", "agentName", "processName"]
  3. 집계 쿼리 사용: "queryType": "aggregation"으로 그룹화
  4. Purple AI를 이용해 최적화된 쿼리 자동 생성 활용

프로덕션 운영 팁

  • 스테이징 배포: 먼저 테스트/개발 그룹에 배포하여 1-2주간 모니터링 후 전체로 확장.
  • 태깅 전략: 에이전트에 env:production, department:finance 같은 태그를 부여하여 정책 적용과 조회를 효율화.
  • 자동화 플레이북: 반복적인 대응 작업(예: 랜섬웨어 감염 시)은 콘솔 내 자동화 빌더로 플레이북을 만들어 MTTR을 단축.
  • 정기 리포트: Purple AI로 "지난주 주요 위협 활동 요약 리포트 생성해줘" 명령어로 주간 리포트 자동화.
