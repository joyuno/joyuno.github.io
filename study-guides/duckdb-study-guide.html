1. DuckDB란?

DuckDB는 임베디드 분석형 데이터베이스 관리 시스템(Embedded OLAP DBMS)입니다. 단일 파일로 존재하며, 서버 설치나 관리가 필요 없이 애플리케이션 내에 직접 통합되어 사용됩니다. SQL을 통해 대용량 데이터를 빠르게 분석하는 것이 핵심 목표입니다.

임베디드 (Embedded)

별도의 서버 프로세스가 필요 없습니다. Python, R, Java 등 애플리케이션에 라이브러리 형태로 직접 포함됩니다. 데이터베이스 파일(.db)은 단일 파일로 관리됩니다.

분석형 (OLAP)

트랜잭션 처리(OLTP)보다는 복잡한 분석 쿼리(집계, 조인, 윈도우 함수)에 최적화되어 있습니다. 컬럼형 저장 방식과 벡터화 실행 엔진을 통해 높은 성능을 제공합니다.

SQL 중심

표준 SQL(ANSI SQL)을 완벽 지원하며, PostgreSQL과 유사한 문법을 사용합니다. Parquet, CSV, JSON 등 외부 파일을 직접 SQL로 쿼리할 수 있습니다.

핵심 철학: "단순함과 성능". 복잡한 인프라 없이 데이터 분석 작업을 로컬에서 빠르게 수행할 수 있도록 설계되었습니다.

2. 왜 써야 하는가?

기존 대안과 비교하여 DuckDB의 강점을 확인합니다.

대안 주요 특징 DuckDB 비교점
PostgreSQL / MySQL 서버형 OLTP DBMS, 트랜잭션 관리 강점 설치/관리 부담 없음, 분석 쿼리 성능 우수, 임베디드 통합 쉬움
Apache Spark 분산 처리 프레임워크, 대규모 클러스터 필요 로컬 단일 머신에서 빠른 분석, 설정 복잡도 낮음, 실시간 반응성 높음
SQLite 임베디드 OLTP DBMS, 범용 트랜잭션 컬럼형 저장, 분석 쿼리(조인/집계) 성능 월등, 벡터화 실행 엔진
Pandas (in-memory) 메모리 기반 데이터프레임, Python 생태계 SQL 표준 사용, 메모리 효율성 높음(외부 파일 직접 쿼리), 대용량 처리 가능
선택 기준: 서버 설정이 어렵거나 불필요한 경우, 로컬/단일 머신에서 대용량 데이터를 SQL로 빠르게 분석해야 하는 경우, 애플리케이션 내에 DB를 직접 통합해야 하는 경우 DuckDB가 이상적입니다.

3. 핵심 개념

아키텍처 다이어그램

+-----------------------------+ | 애플리케이션 | | (Python, R, Java, C++ 등) | +-----------------------------+ | | (라이브러리 통합) v +-----------------------------+ | DuckDB 엔진 | | (임베디드 인메모리/파일) | +-----------------------------+ | | (벡터화 실행 엔진) v +-----------------------------+ | 컬럼형 저장 포맷 | | (테이블 데이터) | +-----------------------------+ | | (직접 접근) v +-----------------------------+ | 외부 데이터 소스 | | (Parquet, CSV, JSON 등) | +-----------------------------+

주요 개념 설명

  • 임베디드 엔진: 전체 DB가 하나의 라이브러리로 제공됩니다. import duckdb 후 바로 사용 가능.
  • 컬럼형 저장: 데이터를 컬럼별로 저장하여 분석 쿼리(특정 컬럼 집계) 시 I/O와 메모리 효율을 극대화합니다.
  • 벡터화 실행: 쿼리를 처리할 때 데이터를 벡터(배열) 단위로 처리하여 CPU 효율을 높입니다.
  • 외부 파일 쿼리: 별도 임포트 없이 Parquet, CSV 등을 직접 SQL로 조회할 수 있습니다. (예: SELECT * FROM 'data.parquet')
  • 메모리 관리: 메모리 내 처리와 파일 기반 처리를 혼용하며, 대용량 데이터도 효율적으로 처리합니다.

4. 퀵스타트

4.1 설치

# Python (pip)
pip install duckdb

# 또는 특정 버전 (확장 기능 포함)
pip install "duckdb==0.10.0"

# CLI 도구 (macOS/Linux 예시)
curl -LO https://github.com/duckdb/duckdb/releases/download/v0.10.0/duckdb_cli-linux-amd64.zip
unzip duckdb_cli-linux-amd64.zip
./duckdb

4.2 기본 설정 및 연결

import duckdb

# 메모리 DB 연결 (임시)
con = duckdb.connect(':memory:')
# 또는 파일 DB 연결 (영구 저장)
con = duckdb.connect('my_database.db')

# 기본 설정 (예: 메모리 제한 설정)
con.execute("SET memory_limit='4GB';")
con.execute("SET threads TO 4;")

4.3 첫 예제: CSV 파일 직접 분석

# CSV 파일을 직접 테이블처럼 쿼리
result = con.execute("""
    SELECT
        category,
        COUNT(*) as count,
        AVG(price) as avg_price
    FROM 'sales_2026.csv'
    WHERE date > '2026-01-01'
    GROUP BY category
    ORDER BY avg_price DESC
""").fetchall()

print(result)

# 결과를 DataFrame으로 바로 받기 (Python)
df = con.execute("SELECT * FROM 'sales_2026.csv' LIMIT 10").df()
print(df.head())

# Parquet 파일로 저장 (결과 내보내기)
con.execute("COPY result TO 'summary.parquet' (FORMAT PARQUET);")
DuckDB는 CSV/Parquet 파일을 직접 읽을 수 있으므로, 별도의 테이블 생성/임포트 과정이 필요 없습니다. 분석을 바로 시작할 수 있습니다.

5. 핵심 설정 레퍼런스

성능 및 동작을 제어하는 주요 설정입니다. SET 설정명 TO 값; 형태로 사용합니다.

설정 기본값 설명 실무 권장값
memory_limit 무제한 DB가 사용할 최대 메모리 '4GB' 또는 시스템 메모리 50%~80%
threads CPU 코어 수 쿼리 실행에 사용할 스레드 수 4~8 (병렬 쿼리 시)
temp_directory 시스템 임시 폴더 임시 파일 저장 경로 빠른 SSD 경로 지정 (대용량 작업 시)
enable_object_cache true Parquet 등 외부 파일 메모리 캐시 true (반복 쿼리 시 성능 향상)
timezone UTC 시간대 설정 'Asia/Seoul' (지역 분석 시)
preserve_insert_order false INSERT 순서 보존 true (순서 중요한 데이터 시)
주의: memory_limit을 너무 작게 설정하면 대용량 쿼리 시 메모리 부족 오류가 발생할 수 있습니다. 시스템 전체 메모리를 고려하여 설정하세요.

6. 실무 팁 & 트러블슈팅

자주 겪는 문제와 해결책

  • 문제: 대용량 CSV 읽기 시 느림
    해결: CSV 대신 Parquet 형식으로 변환 후 사용. Parquet는 컬럼형이며 DuckDB에서 최적 성능을 제공합니다. 또는 SET enable_object_cache TO true;로 캐시 활성화.
  • 문제: 메모리 부족 오류
    해결: SET memory_limit='6GB';로 제한 증가. 또는 SET temp_directory='/fast_ssd/tmp';로 임시 파일 저장 경로를 빠른 디스크로 변경하여 메모리 부담 줄임.
  • 문제: Python에서 DataFrame 변환 시 느림
    해결: .fetchall() 대신 .df()를 바로 사용. 또는 Arrow 형식(.arrow())으로 받아서 변환하면 더 빠릅니다.
  • 문제: 다중 조인 쿼리 성능 저하
    해결: 조인 순서를 최적화 (작은 테이블 먼저). 인덱스 생성은 DuckDB에서 자동 관리되지만, CREATE INDEX idx ON table(col);로 명시적 생성 가능.
  • 문제: CLI에서 한글 출력 문제
    해결: CLI 실행 시 .duckdb 파일에 SET utf8_output TO true;를 설정 파일에 추가.

성능 팁

  • 반복 쿼리 시 외부 파일 캐시 활성화(enable_object_cache)
  • 대용량 작업 시 메모리 제한과 임시 디스크 경로를 동시 설정
  • 여러 파일 분석 시 UNION 대신 FROM 'folder/*.parquet' 패턴 사용
  • Python에서 대량 INSERT 시 con.execute("BEGIN TRANSACTION;") 후 한번에 커밋
