1. Splunk Enterprise Security란?

Splunk Enterprise Security (ES)는 Splunk 플랫폼 위에서 동작하는 프리미어 SIEM 솔루션입니다. 실시간 위협 탐지, 사고 대응, 위험 기반 분석을 통합하며, 머신러닝 기반 이상 징후 탐지와 자동화된 플레이북을 지원합니다.

🔍 위협 탐지

상관관계 규칙, 위협 인텔리전스, 이상 징후 탐지로 알려진/미지의 위협을 실시간 식별.

⚡ 사고 대응

케이스 관리, 위험 점수, 자동화된 대응 (어댑티브 플레이북) 으로 대응 시간 단축.

📊 위험 기반 분석

에셋, 인증, 위협 데이터를 결합한 위험 점수로 우선순위 기반 알림.

2. 왜 써야 하는가?

기존 SIEM 대비 Splunk ES의 차별점은 데이터 온보딩 유연성, 고급 분석, 확장성입니다. 아래 비교표를 참고하세요.

기능Splunk ESELK (Elastic Security)QRadar
데이터 소스500+ 애드온, 모든 형식Beats, 제한적주로 로그/플로우
ML 기반 탐지내장 (MLTK, 이상탐지)플러그인 필요제한적
위험 점수기본 내장커스텀 필요부분 지원
자동화 (SOAR)네이티브 통합별도 구성IBM Resilient
쿼리 언어SPL (강력, 직관적)EQL / KQLAQL
✅ AI/데이터 엔지니어 포인트: Splunk ES는 Python 기반 머신러닝 툴킷(MLTK)과 연동되며, 대규모 데이터 파이프라인에서 스트리밍 분석을 지원합니다.

3. 핵심 개념

Splunk ES는 데이터 수집 → 인덱싱 → 상관관계 → 위험 점수 → 대시보드 흐름으로 동작합니다. 아래 아키텍처 다이어그램을 참고하세요.

+------------------+ +-------------------+ +------------------+ | 데이터 소스 | | Splunk ES | | 대시보드 & 알림 | | (로그, API, | ----> | 상관관계 엔진 | ----> | (위험 점수, | | 네트워크, 클라우드)| | 위협 인텔리전스 | | 케이스 관리) | +------------------+ +-------------------+ +------------------+ | | | v v v +------------------+ +-------------------+ +------------------+ | Heavy Forwarder | | 인덱서 / 검색 헤드 | | 알림 (Slack, | | (데이터 전처리) | | (MLTK, 위험 분석) | | 이메일, SOAR) | +------------------+ +-------------------+ +------------------+
  • 상관관계 규칙: SPL 기반, 위험 점수 매핑
  • 위험 점수: 에셋 중요도 + 위협 심각도 + 이상 징후
  • Adaptive Response: 자동화된 플레이북 (Python, 스크립트)

4. 퀵스타트

📦 설치 (Linux / Splunk Enterprise 9.x)

# 1. Splunk Enterprise 설치 (예: /opt/splunk)
wget -O splunk-9.2.1.tar.gz 'https://download.splunk.com/...'
tar -xzf splunk-9.2.1.tar.gz -C /opt/
/opt/splunk/bin/splunk start --accept-license

# 2. ES 앱 설치 (Passport 또는 크레딧 필요)
/opt/splunk/bin/splunk install app splunk-enterprise-security_910.spl -auth admin:changeme

⚙️ 기본 설정 (데이터 입력 및 위협 인텔리전스)

# CLI로 데이터 입력 추가 (예: syslog)
/opt/splunk/bin/splunk add udp 514 -sourcetype syslog -index main

# 위협 인텔리전스 피드 활성화 (ES 설정 UI 또는 REST)
curl -k -u admin:changeme https://localhost:8089/servicesNS/admin/splunk_essentials/...
    -d '{"threat_intel_feed":"https://feeds.example.com/taxtii"}'

🧪 첫 예제: SPL 상관관계 규칙 (위험 점수 80 이상 알림)

| tstats `summariesonly` count from datamodel=Risk.All_Risk 
  where nodename=All_Risk risk_score>=80 
  by _time, risk_object, risk_score, source 
| `drop_dm_object_name("Risk")` 
| sort - _time 
| table _time, risk_object, risk_score, source
⚠️ 실무 팁: 위험 점수 기준은 환경에 맞게 조정하세요. 기본 80은 높은 편이며, PoC에서는 40~50부터 시작하는 것을 권장합니다.

5. 핵심 설정 레퍼런스

설정 항목위치 (UI / 파일)설명
상관관계 규칙설정 > 상관관계 검색SPL 기반, 위험 점수, 라벨, 알림 액션
위험 점수 할당설정 > 위험 점수에셋/인증 기반 기본값 0~100

import requests; requests.post('https://hooks.slack.com/...', json={'text': f'위험 점수 {risk_score}'})

6. 실무 팁 & 트러블슈팅

📌 문제: 상관관계 규칙 미탐지

해결: 데이터 모델 가속 상태 확인 (| datamodel), 요약 인덱싱 활성화. SPL에 summariesonly 사용.

📌 문제: 위험 점수 과다 알림

해결: 위험 점수 임계값 조정, 에셋 중요도 재평가, 중복 제거 로직 추가.

📌 문제: ES 앱 속도 저하

해결: 인덱서 클러스터링, 데이터 모델 가속 스케줄, dispatch 디렉토리 정리.

🔧 트러블슈팅 CLI:
splunk show datamodel-risk-acceleration · splunk search "index=_internal sourcetype=splunkd" · tail -f $SPLUNK_HOME/var/log/splunk/splunkd.log
