1. BigQuery란?

Google Cloud Platform(GCP)의 완전 관리형 서버리스 데이터 웨어하우스 및 분석 플랫폼입니다. 페타바이트 규모의 데이터에 대해 초고속 SQL 쿼리를 실행할 수 있으며, 인프라 관리 없이 분석에 집중할 수 있습니다.

⚡ 초고속 쿼리

컬럼형 저장소와 Dremel 쿼리 엔진을 활용하여 대규모 데이터셋에서도 초고속 SQL 쿼리 실행. 분리된 스토리지와 컴퓨팅 아키텍처.

🔧 완전 관리형

서버리스 아키텍처로 인프라 프로비저닝, 패치, 백업, 클러스터 관리가 필요 없습니다. 사용한 만큼만 비용 지불.

📈 머신러닝 통합

BigQuery ML을 통해 SQL만으로 모델 훈련 및 배포 가능. TensorFlow, Vertex AI와의 원활한 통합.

2. 왜 써야 하는가?

기존 온프레미스 및 클라우드 대안과의 비교를 통해 BigQuery의 강점을 확인합니다.

솔루션 장점 단점 BigQuery 대비
온프레미스 (Oracle, Teradata) 완전한 제어권, 레거시 시스템 통합 높은 유지보수 비용, 확장성 제한, 전문 인력 필요 관리 오버헤드 90% 감소, 탄력적 확장
AWS Redshift AWS 생태계 통합, 성숙한 도구 클러스터 관리 필요, 용량 계획 필요 완전 서버리스, 자동 스케일링
Snowflake 멀티 클라우드, 시장점유율 높음 비용이 높을 수 있음, 벤더 종속성 GCP 통합 최적화, BigQuery ML 고유 기능
Azure Synapse Azure 서비스 통합, 다양한 컴퓨팅 옵션 복잡한 아키텍처, 학습 곡선 가파름 단순한 서버리스 모델, 빠른 시작
핵심 선택 기준: 완전 관리형 서버리스가 필요하고, GCP 생태계를 사용하며, ML 통합이 중요한 경우 BigQuery가 최적의 선택입니다.

3. 핵심 개념

BigQuery의 아키텍처와 핵심 구성 요소를 이해합니다.

+-------------------+ +-------------------+ +-------------------+ | 데이터 소스 | | BigQuery | | 분석/시각화 | | (Cloud Storage, |---->| Storage |---->| (Looker, | | Streaming, etc.)| | (Colossus) | | Data Studio, | +-------------------+ +-------------------+ | BI Tools) | | +-------------------+ | 쿼리 요청 v +---------------------------------------------------------------+ | BigQuery 서비스 계층 | | +-------------------+ +---------------------------+ | | | Dremel 엔진 | | 슬롯 & 리소스 관리 | | | | (쿼리 실행) |<----->| (자동 스케일링) | | | +-------------------+ +---------------------------+ | +---------------------------------------------------------------+ | | 결과 반환 v +-------------------+ +-------------------+ +-------------------+ | 머신러닝 | | 데이터 마트 | | 애플리케이션 | | (BigQuery ML) | | (Materialized | | (API 호출) | | | | Views) | | | +-------------------+ +-------------------+ +-------------------+

주요 개념 설명

  • 프로젝트: BigQuery의 최상위 컨테이너. 모든 리소스는 프로젝트에 속함.
  • 데이터셋: 테이블, 뷰, 모델을 포함하는 논리적 컨테이너 (데이터베이스 개념).
  • 테이블: 행과 열로 구성된 실제 데이터. 네이티브 테이블과 외부 테이블로 구분.
  • 슬롯: 쿼리 실행을 위한 컴퓨팅 용량 단위. 자동으로 할당 및 관리됨.
  • 파티셔닝 & 클러스터링: 쿼리 성능 최적화 및 비용 절감을 위한 데이터 조직화 기술.

4. 퀵스타트

Python을 사용하여 BigQuery를 시작하는 단계별 가이드입니다.

단계 1: 환경 설정

GCP 프로젝트 생성 및 서비스 계정 키 준비

# CLI에서 GCP 인증 및 프로젝트 설정
gcloud auth login
gcloud config set project YOUR_PROJECT_ID
gcloud auth application-default login

단계 2: Python 클라이언트 설치

# pip를 사용한 클라이언트 라이브러리 설치
pip install google-cloud-bigquery pandas pyarrow

단계 3: 첫 번째 쿼리 실행

from google.cloud import bigquery

# 클라이언트 초기화 (환경변수 GOOGLE_APPLICATION_CREDENTIALS 설정 필요)
client = bigquery.Client(project='your-project-id')

# 공개 데이터셋에서 샘플 쿼리 실행
query = """
    SELECT name, SUM(number) as total
    FROM `bigquery-public-data.usa_names.usa_1910_current`
    WHERE state = 'TX'
    GROUP BY name
    ORDER BY total DESC
    LIMIT 10
"""

# 쿼리 실행 및 결과를 DataFrame으로 변환
query_job = client.query(query)
df = query_job.to_dataframe()

print("텍사스에서 가장 인기 있는 이름 Top 10:")
print(df.head())

단계 4: 데이터 로드 및 테이블 생성

# CSV 파일에서 데이터 로드
dataset_id = "your_dataset"
table_id = "sample_sales"

# 데이터셋 참조
dataset_ref = client.dataset(dataset_id)
table_ref = dataset_ref.table(table_id)

# 작업 구성
job_config = bigquery.LoadJobConfig(
    source_format=bigquery.SourceFormat.CSV,
    skip_leading_rows=1,
    autodetect=True,  # 스키마 자동 감지
    write_disposition=bigquery.WriteDisposition.WRITE_TRUNCATE  # 기존 테이블 덮어쓰기
)

# Cloud Storage 또는 로컬 파일에서 로드
with open("sales_data.csv", "rb") as source_file:
    load_job = client.load_table_from_file(
        source_file, table_ref, job_config=job_config
    )

load_job.result()  # 작업 완료 대기
print(f"테이블 {table_id}에 {load_job.output_rows} 행 로드 완료")

5. 핵심 설정 레퍼런스

실무에서 자주 사용하는 중요한 설정 옵션들입니다.

카테고리 설정/옵션 기본값 권장값/설명
쿼리 최적화 maximum_bytes_billed 무제한 쿼리당 비용 제한 설정 (예: 1000000000 = 1GB)
use_legacy_sql False Standard SQL 사용 권장 (성능 및 기능 우위)
priority INTERACTIVE BATCH: 비용 50% 절감 but 큐 대기 가능
테이블 설계 파티셔닝 없음 시간 단위(일별) 또는 정수 범위 파티셔닝 권장
클러스터링 없음 자주 필터링하는 컬럼 4개까지 지정 가능
만료 시간 없음 임시 데이터 자동 정리를 위해 설정 (초 단위)
작업 관리 job_timeout_ms 무제한 장시간 실행 작업 타임아웃 설정 (예: 300000 = 5분)
labels 없음 작업에 라벨 추가하여 비용 추적 및 관리
ML 모델 model_type LINEAR_REG linear_reg, logistic_reg, kmeans, tensorflow 등

6. 실무 팁 & 트러블슈팅

주의: 아래 문제들은 실무에서 자주 발생하는 사례들입니다. 미리 대비하면 비용과 시간을 절약할 수 있습니다.

자주 겪는 문제와 해결책

  • 문제: 쿼리 비용이 예상보다 많이 발생
    해결: 1. maximum_bytes_billed 설정으로 비용 상한 제한
    2. 파티셔닝/클러스터링으로 스캔 데이터 양 최소화
    3. 쿼리 실행 전 dry run으로 처리 데이터량 확인
    job_config = bigquery.QueryJobConfig(dry_run=True)
  • 문제: "Resources exceeded during query execution" 에러
    해결: 1. 쿼리를 여러 개의 작은 쿼리로 분할
    2. 임시 테이블 사용하여 중간 결과 저장
    3. 슬롯 예약(reservation)으로 컴퓨팅 용량 보장
  • 문제: 데이터 로드 속도가 너무 느림
    해결: 1. AVRO 또는 Parquet 포맷 사용 (CSV 대비 2-5배 빠름)
    2. 여러 파일을 병렬로 로드
    3. 스트리밍 삽입 API 사용 (실시간 데이터)
  • 문제: 동시 쿼리 실행 제한에 걸림
    해결: 1. 쿼리 큐잉 구현 (exponential backoff)
    2. 슬롯 예약 풀 크기 증가 요청
    3. BATCH 우선순위 쿼리 사용
  • 문제: BigQuery ML 모델 성능이 낮음
    해결: 1. 특징 공학 적용 (데이터 전처리)
    2. 하이퍼파라미터 튜닝
    3. 더 큰 데이터셋으로 훈련
