1. Burp Suite란?

Burp Suite는 PortSwigger에서 개발된 통합 웹 애플리케이션 보안 테스팅 플랫폼입니다. 프록시, 스캐너, 인트루더, 리피터 등 다양한 도구를 하나의 통합 인터페이스에서 제공하여, 수동 및 자동화된 보안 테스트를 지원합니다.

1. 가로채기 & 조작(Intercept & Manipulate)

브라우저와 대상 애플리케이션 사이의 모든 HTTP/HTTPS 트래픽을 중간에서 가로채고 실시간으로 수정할 수 있습니다. 요청/응답의 모든 부분(헤더, 파라미터, 쿠키, 바디)을 테스터의 의도에 맞게 변조하는 것이 핵심입니다.

2. 자동화된 탐지(Automated Discovery)

정적 분석(SAST)과 달리, 실행 중인 애플리케이션에 실제 요청을 보내 취약점을 탐지하는 동적 분석(DAST)을 수행합니다. OWASP Top 10에 대응하는 SQLi, XSS, CSRF, SSRF 등 다양한 취약점을 자동으로 스캔합니다.

3. 확장성 & 통합(Extensibility & Integration)

Java 기반의 확장 API(Burp Extender)를 제공하여, 사용자 정의 스캔 로직, 자동화 워크플로우, 외부 도구(예: Nuclei, sqlmap)와의 연동을 위한 플러그인 개발이 가능합니다. 이는 AI 엔지니어링 워크플로우에 통합하는 데 핵심 가치를 제공합니다.

2. 왜 써야 하는가? (기존 대안과 비교)

다른 도구나 프레임워크 대비 Burp Suite를 선택해야 하는 이유는 통합성, 정확성, 그리고 생태계에 있습니다.

도구/접근법 장점 단점 Burp Suite의 차별점
수동 cURL/스크립트 완전한 제어, 가벼움 반복 작업 비효율, 시각적 피드백 부재, 세션 관리 복잡 GUI 기반의 직관적인 트래픽 조작과 자동화된 세션/쿠키 관리
OWASP ZAP 무료 오픈소스, 기본 기능 충실 스캔 정확도와 성능, 고급 기능 및 상용 지원 부족 상용 스캐너(Professional)의 높은 정확도와 탐지율, Active Scan 최적화
네트워크 스캐너 (Nmap, Nessus) 인프라 레벨 취약점 탐지에 강점 웹 애플리케이션 레이어의 비즈니스 로직 취약점 탐지 불가 애플리케이션 레벨(레이어7)에 특화된 테스트 도구 세트 제공
자체 Python 스크립트 맞춤형 자동화 가능, AI 파이프라인 통합 용이 기본 인프라(프록시, 스파이더, 디코더)부터 구축해야 하는 부담 견고한 플랫폼 위에 확장 API를 통해 자체 로직만 추가하면 됨 (하이브리드 접근)
AI 엔지니어링 관점: Burp는 단순한 스캐너가 아닌, Burp Extender API를 통해 AI 모델(취약점 예측, 페이로드 생성, 결과 분류)을 통합할 수 있는 실행 플랫폼입니다. 수집된 트래픽 데이터는 머신러닝 모델 학습용 고품질 데이터셋으로도 사용 가능합니다.

3. 핵심 개념 & 아키텍처

Burp Suite는 모듈식 아키텍처로 설계되어 있으며, 각 도구는 독립적으로 동작하면서도 프로젝트 파일과 메모리를 공유합니다.

+-----------------------------------------------------------------------+ | Burp Suite Dashboard | +-----------------------------------------------------------------------+ | | | +-------------+ +-------------+ +-------------+ +-------------+ | | | Proxy | | Intruder | | Repeater | | Scanner | | | | (Intercept)| | (Automated | | (Manual | | (Automated | | | | | | Attacks) | | Testing) | | Scanning) | | | +------+------+ +------+------+ +------+------+ +------+------+ | | | | | | | +---------+----------------+----------------+----------------+----------+ | | | | v v v v +---------+----------------+----------------+----------------+----------+ | | | Shared Project File & Session Data | | (요청/응답 히스토리, 설정, 맵) | | | +-----------------------------+------------------------------------------+ | v +---------------------------+ | Target Web App | | (테스트 대상) | +---------------------------+

주요 컴포넌트 설명

  • Proxy: 모든 트래픽의 관문. 인터셉트 on/off, 인증 처리, TLS 설정.
  • Target: 테스트 범위(Scope) 정의. 사이트맵 자동 생성.
  • Intruder: 파라미터 퍼징(Fuzzing) 자동화. 클러스터 폭탄, 피치포크 등 공격 유형.
  • Repeater: 단일 요청을 반복적으로 수정하고 재전송.
  • Scanner (Professional): 자동 취약점 스캔 엔진.
  • Extender: Python, Ruby, Java로 작성된 플러그인 로드 및 관리.

4. 퀵스타트: 설치 → 설정 → 첫 예제

4.1 설치 (Community Edition 기준)

CLI / 터미널
# Java 확인 (필수)
java --version  # 11 이상 권장

# 1. 공식 사이트에서 JAR 파일 다운로드 (수동)
# https://portswigger.net/burp/releases

# 2. 실행 (가장 일반적인 방법)
java -jar -Xmx4G downloads/burpsuite_community.jar

# 3. 또는, 커맨드라인 헤드리스 모드 (자동화용)
java -jar burpsuite_community.jar --unpause-spider-and-scanner --project-file=project.burp

4.2 초기 프록시 설정

  1. Burp 실행 후 Proxy → Options 탭 확인.
  2. 기본 프록시 리스너는 127.0.0.1:8080에서 실행 중.
  3. 브라우저(또는 시스템) 프록시를 위 주소로 설정.
  4. http://burpsuite 접속하여 CA 인증서 다운로드 및 설치 (HTTPS 트래픽 디코딩 필수).

4.3 첫 번째 테스트: Python으로 Burp 연동

requests 라이브러리를 사용해 Burp 프록시를 통해 요청을 보내고, 응답을 검사하는 기본 예제입니다.

Python: proxy_test.py
import requests
import json

# Burp Suite 프록시 설정 (Community Edition 기본)
PROXIES = {
    'http': 'http://127.0.0.1:8080',
    'https': 'http://127.0.0.1:8080'
}

# 대상 URL (테스트용 안전한 사이트)
TARGET_URL = "https://httpbin.org/get"

# 1. Burp를 통해 요청 보내기
try:
    response = requests.get(TARGET_URL, proxies=PROXIES, verify=False)
    print(f"[*] Status Code: {response.status_code}")
    print(f"[*] Response Headers:\n{json.dumps(dict(response.headers), indent=2)}")
    print(f"[*] First 500 chars of Body:\n{response.text[:500]}")

except requests.exceptions.ProxyError as e:
    print(f"[!] 프록시 연결 실패. Burp Suite가 실행 중이고 8080 포트에서 수신 대기 중인지 확인하세요.")
    print(f"    오류: {e}")
except Exception as e:
    print(f"[!] 오류 발생: {e}")

# 2. Burp의 'Repeater' 기능을 모방한 간단한 파라미터 퍼징
print("\n" + "="*50)
print("[*] 간단한 파라미터 퍼징 시뮬레이션 (Intruder 개념)")

test_params = ['admin', "' OR '1'='1", '', '../../etc/passwd']
base_url = "https://httpbin.org/get"

for payload in test_params:
    # 파라미터를 추가한 요청
    params = {'input': payload}
    try:
        resp = requests.get(base_url, params=params, proxies=PROXIES, verify=False)
        # 실제 테스트에서는 응답 내용을 분석해 취약점 지표를 찾음
        print(f"  [+] Payload: {payload:30} -> Length: {len(resp.text)}")
    except:
        pass
주의: verify=False는 Burp의 CA 인증서를 신뢰하지 않을 때 사용하는 것으로, 실제 프로덕션 환경에서는 보안상 권장되지 않습니다. 적절한 CA 인증서 설치를 우선하세요.

5. 핵심 설정 레퍼런스

실무 효율성을 높이는 필수 설정들을 표로 정리했습니다.

설정 영역 경로/키 권장값/설명 목적
프록시 리스너 Proxy → Options → Proxy Listeners 127.0.0.1:8080 (Loopback only)
Binding: All interfaces 필요시		 트래픽 가로채기 기본 포트. 보안을 위해 로컬호스트로 제한.
인터셉트 제외 Proxy → Options → Intercept Client Requests 파일 확장자 추가: ^.*\.(css|js|png|jpg|gif|ico|svg|woff2?)$ 정적 리소스 트래픽으로 인한 인터셉트 방해 방지.
타겟 스코프 Target → Scope 정규식으로 도메인/IP 범위 명시적 정의. 예: ^https?://target\.com.* 테스트 범위 제한으로 부수적 피해 방지 및 효율성 향상.
세션 처리 Project options → Sessions Session Handling Rules에서 매크로 기록 및 자동 로그인 설정 인증이 필요한 애플리케이션에서 스캔/테스트 지속성 보장.
스캔 최적화 Dashboard → New Scan → Configuration 크롤링 한도 설정, Insertion Point 제한(헤더 제외), 최대 요청 빈도 조절 무한 루프 방지 및 서비스 장애(DOS) 예방.
Extender API Extender → APIs Python 환경 설정, burp 모듈 임포트 가능 경로 추가 사용자 정의 플러그인 개발 및 외부 도구 통합.

6. 실무 팁 & 트러블슈팅

문제: HTTPS 트래픽이 디코드되지 않음(깨진 문자)

원인: 브라우저/클라이언트가 Burp의 CA 인증서를 신뢰하지 않음.

해결: http://burpsuite에서 인증서 다운로드 후 시스템/브라우저 신뢰할 수 있는 루트 인증 기관에 설치. 또는 Python 코드에서 REQUESTS_CA_BUNDLE 환경변수 설정.

문제: 스캔이 매우 느리거나 멈춤

원인: 무한 루프(로그아웃-로그인), 대용량 응답, 제한 없는 크롤링.

해결: 스코프 정확히 설정, 크롤링 한도(링크 깊이, 요청 수) 적용, Session Handling Rules로 로그인 상태 유지.

문제: 외부 도구(sqlmap)와 연동이 안됨

원인: sqlmap이 Burp의 로그 파일 형식을 인식하지 못하거나 프록시 설정 오류.

해결: Burp에서 Proxy → HTTP history → Save 시 XML 형식 선택. sqlmap 실행 시: sqlmap -l burp_log.xml --batch --proxy=http://127.0.0.1:8080
