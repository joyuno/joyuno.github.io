1. CyberArk란?

CyberArk는 특권 계정 관리(Privileged Access Management, PAM) 분야의 선두주자로, 조직 내 가장 중요한 자산에 대한 접근을 보호하는 솔루션 스위트입니다. 핵심은 Vault라는 중앙 집중식 보안 저장소에 모든 특권 자격증명(패스워드, SSH 키, API 토큰 등)을 암호화하여 저장하고, 접근을 엄격하게 제어·감사·관리하는 것입니다.

🔐 자격증명 관리

수명주기 전반(발견, 등록, 순환, 폐기)에 대한 자동화된 관리. 정책 기반의 자동 패스워드 변경으로 공격 표면을 최소화합니다.

🚪 세션 격리 및 모니터링

PSM(Privileged Session Manager)을 통해 직접적인 대상 시스템 접속을 차단하고, 중계 세션을 기록·감사합니다. 모든 키스트로크와 화면을 저장.

🤖 자동화된 위협 분석

이상적인 접근 패턴을 AI/ML로 학습, 위험한 특권 세션(비정상 시간/위치/명령어)을 실시간 탐지하고 대응합니다.

2. 왜 써야 하는가? (기존 대안과 비교)

많은 조직이 스크립트, 일반 패스워드 관리자, 또는 클라우드 네이티브 도구로 특권 계정을 관리하려 시도하지만, 다음과 같은 한계가 있습니다.

솔루션장점단점 (CyberArk 대비)적합 시나리오
CyberArk 통합 PAM, 강력한 감사·규정 준수, 세션 격리, 위협 분석, 높은 확장성 복잡성, 높은 라이선스 비용, 학습 곡선 대규모 엔터프라이즈, 금융/공공, 높은 규제 환경
Hashicorp Vault API 중심, 동적 비밀, 클라우드 친화적, 오픈소스 코어 네이티브 세션 모니터링 부재, PAM 특화 기능 미비, 운영 부담 클라우드/DevOps, 동적 인프라, 서비스간 비밀
1Password / LastPass Enterprise 사용자 친화적, 쉬운 배포, 강력한 개인용 기능 세션 격리 없음, 제한된 특권 계정 관리, 기본적인 감사 로그 내부 직원 비밀번호 공유, 2FA 관리
수동 관리 (스프레드시트, 로테이션 스크립트) 초기 비용 없음, 간단한 구조 보안 취약점 극대화, 감사 불가, human error, 규모 확장 불가 소규모 PoC, 비권장
핵심 차별점: CyberArk는 "관리"를 넘어 "세션 격리와 행위 분석"까지 포함한 종합적인 특권 접근 보안 생태계를 제공합니다. 규정 준수(SOX, PCI-DSS, GDPR) 요구사항을 충족시키는 데 필수적입니다.

3. 핵심 개념과 아키텍처

CyberArk 솔루션은 여러 컴포넌트로 구성됩니다. 가장 일반적인 Enterprise Architecture Vault(온프레미스) 기준의 상호작용을 ASCII로 표현합니다.

    +----------------+     안전한 채널     +---------------+
    |  관리자/사용자 | ------------------> |   PVWA        |
    |  (Web/CLI)     |    (HTTPS/443)     |(Password Vault|
    +----------------+                    |  Web Access)  |
                                          +-------+-------+
                                                  | REST API
                                          +-------+-------+
                                          |  Central Vault|
                                          |  (암호화 저장) |
                                          +-------+-------+
                                                  | 내부 통신
    +----------------+                    +-------+-------+    +---------------+
    |  대상 시스템   | <----------------- |     PSM       | -> |  감사 저장소  |
    |  (SSH, RDP, DB)|   격리된 세션     |(Privileged    |    |(세션 녹화/로그)|
    +----------------+   (SSH/3389 터널)  | Session Manager)|   +---------------+
                                          +----------------+
            ↑                                    ↑
            |                                    | CPM 통신
    +----------------+                    +----------------+
    |  자격증명 변경 | <----------------- |      CPM       |
    |  (Target)      |    정기/이벤트 기반|(Credential     |
    +----------------+                    | Password Manager)|
                                          +----------------+

주요 컴포넌트 설명

  • Vault: 암호화된 자격증명 저장소. 모든 보안의 핵심.
  • PVWA: 사용자/관리자의 웹 인터페이스. 접근 요청, 승인, 자격증명 검색.
  • PSM: 세션을 중계하고 모든 활동을 기록하는 게이트웨이. 직접 접근 차단.
  • CPM: 대상 시스템의 패스워드를 정책에 따라 자동으로 변경하는 엔진.
  • PSM: 세션을 중계하고 모든 활동을 기록하는 게이트웨이. 직접 접근 차단.

4. 퀵스타트: 설치부터 첫 실행까지

개발/테스트 환경을 위한 CyberArk CLI 및 Python SDK를 이용한 기본 플로우입니다. (가상 시나리오)

CLI 도구 설치 및 초기 설정

CyberArk PAS(Privileged Access Security) CLI를 다운로드하고 구성합니다.

# 1. CyberArk 패키지 저장소 추가 (예시: Ubuntu/Debian)
echo "deb [arch=amd64] https://repo.cyberark.cloud/pas/stable/deb/ stable main" | sudo tee /etc/apt/sources.list.d/cyberark.list
wget -qO - https://repo.cyberark.cloud/pas/stable/gpgkey | sudo apt-key add -
sudo apt update

# 2. PAS CLI 설치
sudo apt install cyberark-pas-cli

# 3. PVWA 서버에 대한 CLI 구성
cybr configure -s https://pvwa.example.com -u admin_user --insecure
# 인증 후 토큰이 로컬에 저장됩니다.

Python SDK를 이용한 첫 번째 자격증명 조회

CyberArk의 공식 Python 라이브러리(pycyberark)를 사용하여 안전한 자격증명 검색을 자동화합니다.

# pip install pycyberark
import requests
import json

# PVWA REST API를 통한 인증 (예시 - 실제로는 더 안전한 방법 사용)
def get_cyberark_token(pvwa_url, username, password):
    auth_url = f"{pvwa_url}/PasswordVault/API/auth/CyberArk/Logon"
    headers = {"Content-Type": "application/json"}
    payload = {"username": username, "password": password}
    response = requests.post(auth_url, json=payload, headers=headers, verify=False)
    response.raise_for_status()
    return response.json()["CyberArkLogonResult"]

def retrieve_account_credentials(pvwa_url, token, safe, account_name):
    search_url = f"{pvwa_url}/PasswordVault/API/Accounts"
    headers = {"Authorization": token, "Content-Type": "application/json"}
    params = {"filter": f"safeName eq {safe} and username eq {account_name}"}
    response = requests.get(search_url, headers=headers, params=params, verify=False)
    accounts = response.json()["value"]
    if accounts:
        account_id = accounts[0]["id"]
        # 실제 패스워드 검색 요청 (권한 필요)
        retrieve_url = f"{pvwa_url}/PasswordVault/API/Accounts/{account_id}/Password/Retrieve"
        pwd_response = requests.post(retrieve_url, headers=headers, verify=False)
        return pwd_response.json()["value"]
    return None

6. 실무 팁 & 트러블슈팅

🚨 CPM이 패스워드 변경에 실패합니다.

증상: "Password change failed" 알림, 대상 시스템에 로그인 불가.

원인 및 해결:

  • 네트워크/방화벽: CPM 서버 → 대상 시스템 포트(SSH 22, WinRM 5985/6) 접근성 확인.
  • 권한: CPM이 사용하는 계정에 대상 시스템의 패스워드 변경 권한이 있는지 확인.
  • 플러그인: 대상 플랫폼(예: Oracle DB, Cisco IOS)에 맞는 CPM 플러그인이 설치·구성되었는지 확인.
  • 복구: 미리 설정된 '복구 계정'으로 수동 접근 후 문제 해결.

🐢 PSM을 통한 연결이 매우 느립니다.

증상: SSH/RDP 세션 지연, 반응 속도 저하.

원인 및 해결:

  • PSM 부하: 동시 세션 수 모니터링, 필요 시 PSM 서버 추가 (Scale-out).
  • 녹화 설정: '고해상도 비디오 녹화'가 기본일 경우, '텍스트 로깅만'으로 변경 테스트.
  • 네트워크 경로: PSM 서버와 대상 시스템 간의 네트워크 대역폭/지연 확인.

🤖 AI/자동화 파이프라인과의 통합

과제: CI/CD 툴, IaC, AI 모델 학습 스크립트가 안전하게 자격증명을 가져와야 함.

모범 사례:

  • AAM 사용: 애플리케이션 ID를 활용한 OAuth 2.0 기반 인증. 하드코딩된 비밀번호 제거.
  • REST API + 토큰: 짧은 수명의 토큰 발급, IP 제한, 정기 순환.
  • Conjur 통합: CyberArk의 오소스 비밀 관리 솔루션으로 Kubernetes, Docker 환경에 최적화.

🔐 Vault 장애 시 비상 접근 절차

시나리오: Vault 클러스터 다운, 모든 자격증명 접근 불가.

대비책:

  • DR(재해복구) 사이트: 동기화된 보조 Vault 구성 및 정기 테스트.
  • 비상 계정 (Break Glass): Vault 외부에 암호화되어 저장된 제한된 수의 최고 권한 계정. 물리적 안전 장치와 결합.
  • 문서화: 장애 조치 절차를 명확히 문서화하고 정기 훈련 실시.
