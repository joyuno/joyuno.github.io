1. PyTorch란?

PyTorch는 Python 기반의 오픈소스 딥러닝 프레임워크로, 동적 계산 그래프(Dynamic Computational Graph)와 직관적인 API를 특징으로 합니다. 연구와 프로덕션 환경 모두에서 널리 사용됩니다.

1. 동적 계산 그래프

연산이 실행되는 순간 그래프가 정의되며, 조건문/반복문을 자연스럽게 포함할 수 있습니다. 이는 모델 구조가 실행 중에 변경될 수 있는 연구 및 복잡한 모델에 유리합니다.

2. 직관적인 Pythonic API

NumPy와 유사한 인터페이스를 제공하며, Python 프로그래머가 쉽게 접근할 수 있습니다. `torch.Tensor`는 NumPy 배열과 호환성이 높습니다.

3. 강력한 GPU 지원 및 분산 학습

CUDA를 통해 GPU 연산을 완벽 지원하며, `torch.distributed` 모듈으로 다중 GPU/다중 노드 분산 학습을 효율적으로 구현할 수 있습니다.

2. 왜 써야 하는가?

다양한 딥러닝 프레임워크가 존재하지만, PyTorch는 특정 장점으로 실무에서 선택됩니다.

대안 특징 PyTorch 비교
TensorFlow/Keras 정적 그래프 (TF1), 프로덕션 강점, Keras의 간결함 PyTorch는 동적 그래프로 연구/실험 유연성 높음. 코드가 더 Pythonic.
JAX 함수형, 자동 미분, 고성능 (XLA) PyTorch는 더 넓은 생태계와 프로덕션 도구(TorchServe)를 갖춤.
ONNX Runtime 최적화된 추론 엔진, 멀티 플랫폼 PyTorch는 학습부터 추론까지의 전체 파이프라인 통합 제공.
Custom C++/CUDA 극대화된 성능, 완전한 제어 PyTorch는 C++ 확장 가능하며, 대부분의 경우 성능과 편의성 균형 우수.
결론: 연구의 빠른 프로토타이핑과 프로덕션의 안정성을 모두 필요로 하는 조직에서 PyTorch는 균형 잡힌 선택입니다. 특히 MLOps 파이프라인과의 통합(예: TorchServe, MLflow)이 잘 지원됩니다.

3. 핵심 개념

PyTorch의 아키텍처는 여러 핵심 모듈로 구성됩니다.

+-------------------------------+ | PyTorch Core Architecture | +-------------------------------+ | | +------------------------+ | | torch.Tensor | -- 기본 데이터 구조 (CPU/GPU) | +------------------------+ | | +------------------------+ | | torch.autograd | -- 자동 미분 엔진 (동적 그래프) | +------------------------+ | | +------------------------+ | | torch.nn | -- 신경망 모델 구성 (Layer, Module) | +------------------------+ | | +------------------------+ | | torch.optim | -- 옵티마이저 (SGD, Adam 등) | +------------------------+ | | +------------------------+ | | torch.utils.data | -- 데이터 로딩 (Dataset, DataLoader) | +------------------------+ | | +------------------------+ | | torch.distributed | -- 분산 학습 (DDP, RPC) | +------------------------+ | | +------------------------+ | | torch.jit | -- 추론 최적화 (TorchScript) | +------------------------+ | +-------------------------------+

핵심 용어

  • Tensor: 다차원 배열, PyTorch의 기본 데이터 단위. `.to(device)`로 GPU 이동.
  • Module (nn.Module): 모든 네트워크 층과 모델의 기본 클래스. 상태(파라미터)와 연산을 정의.
  • Autograd: 연산에 대한 자동 미분 기록. `requires_grad=True` 텐서의 연산을 추적.
  • DataLoader: 데이터셋을 배치 형태로 로드하며, 멀티프로세싱 지원.
  • DistributedDataParallel (DDP): 다중 GPU에서 모델을 복제하고 데이터를 분배하여 병렬 학습.

4. 퀵스타트

Step 1: 설치

CUDA 지원 GPU 환경을 기준으로 설치합니다.

# 기본 설치 (최신 버전)
pip install torch torchvision torchaudio

# 특정 CUDA 버전 지정 (예: CUDA 12.1)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# CPU-only 설치
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu

Step 2: 환경 확인

import torch

print(f"PyTorch 버전: {torch.__version__}")
print(f"CUDA 가능: {torch.cuda.is_available()}")
print(f"CUDA 버전: {torch.version.cuda}")
print(f"GPU 개수: {torch.cuda.device_count()}")
print(f"현재 GPU 이름: {torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'None'}")

Step 3: 첫 예제 - 간단한 선형 회귀

import torch
import torch.nn as nn
import torch.optim as optim

# 1. 데이터 준비
X = torch.randn(100, 1)  # 100개의 샘플, 1개 특징
y = 3 * X + 2 + 0.1 * torch.randn(100, 1)  # 실제 관계 + 노이즈

# 2. 모델 정의
model = nn.Linear(1, 1)  # 입력 1, 출력 1

# 3. 손실 함수와 옵티마이저
criterion = nn.MSELoss()
optimizer = optim.SGD(model.parameters(), lr=0.01)

# 4. 학습 루프
for epoch in range(100):
    # 순전파
    predictions = model(X)
    loss = criterion(predictions, y)

    # 역전파
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if epoch % 10 == 0:
        print(f'Epoch {epoch}, Loss: {loss.item()}')

# 5. 결과 확인
print(f'학습된 가중치: {model.weight.data.item()}')
print(f'학습된 편향: {model.bias.data.item()}')
참고: 실제 프로젝트에서는 `Dataset`과 `DataLoader`를 사용하여 데이터를 관리하고, `nn.Module`을 상속하여 복잡한 모델을 구성합니다.

6. 실무 팁 & 트러블슈팅

자주 겪는 문제와 해결책

문제 1: GPU 메모리 부족 (CUDA out of memory)

해결책:

  • 배치 크기 줄이기.
  • `torch.cuda.empty_cache()` 호출.
  • Mixed Precision (`autocast`) 사용.
  • 모델의 불필요한 중간 출력 저장 방지 (`torch.no_grad()` 활용).
문제 2: DataLoader가 너무 느림

해결책:

  • `num_workers` 증가 (시스템 CPU 코어 수 고려).
  • `pin_memory=True` 설정 (GPU 전송 속도 향상).
  • 데이터 전처리를 최대한 간소화 또는 GPU에서 수행.
문제 3: 학습 중 Loss가 NaN으로 변함

해결책:

  • 입력 데이터 정규화/스케일링 확인.
  • 학습률(learning rate)을 낮춰보기.
  • `torch.autograd.set_detect_anomaly(True)`로 문제 위치 찾기.
  • 손실 함수 또는 모델 연산에 불안정한 수학 연산(예: log(0)) 확인.
문제 4: 다중 GPU 학습 시 속도 향상 미비

해결책:

  • DDP 사용 시 `torch.distributed.init_process_group` 백엔드 확인 (NCCL 권장).
  • 각 GPU의 배치 크기 균등 분배 확인.
  • 네트워크 병목 (특히 multi-node) 확인.

일반 팁

  • 버전 관리: PyTorch와 CUDA 드라이버 버전 호환성을 항상 확인하세요. 공식 PyTorch 웹사이트의 설치 명령어를 사용.
  • 프로파일링: `torch.profiler` 또는 `cProfile`을 사용하여 연산 시간과 메모리 사용을 분석.
  • 모델 배포: 추론 단계에서는 `torch.jit.trace` 또는 `torch.jit.script`로 TorchScript 모델을 생성하여 성능 향상.
  • MLOps 통합: 모델 버전 관리 및 배포에는 TorchServe 또는 MLflow와 PyTorch 통합을 고려.
