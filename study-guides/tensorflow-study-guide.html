1. TensorFlow란?

TensorFlow는 Google Brain 팀이 개발한 오픈소스 머신러닝/딥러닝 프레임워크입니다. 데이터 흐름 그래프(Data Flow Graph)를 사용하여 수치 계산을 표현하고, 대규모 분산 학습 및 프로덕션 배포를 위한 포괄적인 도구 생태계를 제공합니다.

🔧 생산성 (Productivity)

Keras 고수준 API를 통한 빠른 모델 개발, 즉시 실행(Eager Execution) 모드로 Pythonic 디버깅, 재사용 가능한 컴포넌트.

🚀 성능 (Performance)

XLA 컴파일러, GPU/TPU 자동 분산 학습, 그래프 최적화, TF Serving을 통한 고성능 추론.

🔄 배포 (Deployment)

TensorFlow Lite(모바일/엣지), TensorFlow.js(웹), TensorFlow Serving(REST/gRPC), SavedModel 표준 포맷.

2. 왜 써야 하는가? (기존 대안과 비교)

프로덕션 환경에서의 안정성, 도구 완성도, 커뮤니티/엔터프라이즈 지원을 중시한다면 TensorFlow가 강력한 선택지입니다.

프레임워크 강점 약점 적합한 경우
TensorFlow 생산 배포 도구 완성도(TFX, Serving), 엔터프라이즈 지원, TPU 지원, 그래프 최적화 상대적 복잡성, PyTorch 대비 연구 커뮤니티 비중 감소 대규모 프로덕션 시스템, 클라우드(Google Cloud) 환경, MLOps 파이프라인
PyTorch Pythonic & 직관적, 동적 그래프(연구 편의), 활발한 연구 커뮤니티 프로덕션 배포 도구 상대적 미성숙 (TorchServe 등 발전 중) 연구/실험, 빠른 프로토타이핑, academia
JAX 함수형 프로그래밍, 자동 미분/벡터화, NumPy 호환, 컴파일 성능 저수준, 학습 곡선 가파름, 공식 배포 도구 부재 수치 계산 연구, 새 알고리즘 구현, 하드웨어 가속 실험

3. 핵심 개념 & 아키텍처

TensorFlow의 핵심은 텐서(Tensor)(다차원 배열)가 그래프(Graph)를 따라 흐르며 연산(Operation)이 수행된다는 개념입니다. v2에서는 즉시 실행(Eager Execution)이 기본이지만, 성능을 위해 @tf.function 데코레이터로 그래프 모드로 변환합니다.

┌─────────────────────────────────────────────────────────────┐ │ High-Level APIs (Keras) │ │ Sequential, Functional API, Model Subclassing, Prebuilt │ └───────────────┬─────────────────────────────────────────────┘ │ ┌───────────────▼─────────────────────────────────────────────┐ │ Mid-Level APIs │ │ tf.data (입력 파이프라인), tf.distribute (분산 학습) │ │ tf.layers, tf.losses, tf.metrics │ └───────────────┬─────────────────────────────────────────────┘ │ ┌───────────────▼─────────────────────────────────────────────┐ │ Low-Level APIs (Core TensorFlow) │ │ Tensors, Operations, Graphs, Automatic Differentiation │ └───────────────┬─────────────────────────────────────────────┘ │ ┌───────────────▼─────────────────────────────────────────────┐ │ 배포 & 런타임 │ │ TensorFlow Serving (gRPC/REST) TensorFlow Lite │ │ TensorFlow.js SavedModel Format │ └─────────────────────────────────────────────────────────────┘

SavedModel: 모델 아키텍처, 가중치, 서명, 자산을 포함하는 표준 배포 포맷. Serving, Lite, JS에서 공통 사용.

import tensorflow as tf

# 사용 가능한 디바이스 확인
print("Devices:", tf.config.list_physical_devices())

# 메모리 증가 설정 (GPU)
gpus = tf.config.list_physical_devices('GPU')

Step 3: 첫 모델 - MNIST 분류

Python: first_model.py
import tensorflow as tf

# 1. 데이터 로드 및 전처리
(x_train, y_train), (x_test, y_test) = tf.keras.datasets.mnist.load_data()
x_train, x_test = x_train / 255.0, x_test / 255.0  # 정규화
x_train = x_train[..., tf.newaxis]  # 채널 차원 추가 (28,28,1)
x_test = x_test[..., tf.newaxis]

# 2. tf.data.Dataset으로 파이프라인 구성 (성능 향상)
train_ds = tf.data.Dataset.from_tensor_slices((x_train, y_train))
train_ds = train_ds.shuffle(10000).batch(32).prefetch(tf.data.AUTOTUNE)

# 3. 모델 정의 (Keras Functional API)
inputs = tf.keras.Input(shape=(28, 28, 1))
x = tf.keras.layers.Conv2D(32, 3, activation='relu')(inputs)
x = tf.keras.layers.MaxPooling2D()(x)
x = tf.keras.layers.Conv2D(64, 3, activation='relu')(x)
x = tf.keras.layers.GlobalAveragePooling2D()(x)
outputs = tf.keras.layers.Dense(10, activation='softmax')(x)
model = tf.keras.Model(inputs=inputs, outputs=outputs)

# 4. 컴파일
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

# 5. 학습 (콜백 포함)
callbacks = [
    tf.keras.callbacks.EarlyStopping(patience=2, monitor='val_loss'),
    tf.keras.callbacks.ModelCheckpoint('best_model.keras', save_best_only=True)
]
history = model.fit(
    train_ds,
    validation_data=(x_test, y_test),
    epochs=5,
    callbacks=callbacks,
    verbose=1
)

# 6. SavedModel으로 저장 (배포용)
model.save('mnist_model', save_format='tf')
print("Model saved as SavedModel format.")

5. 핵심 설정 레퍼런스

설정 항목 권장값 / 옵션 설명
메모리 관리 tf.config.experimental.set_memory_growth(gpu, True) GPU 메모리를 필요시 점진적으로 증가시킴. OOM 방지.
분산 전략 MirroredStrategy, MultiWorkerMirroredStrategy, TPUStrategy 단일 머신 다중 GPU, 다중 워커, TPU 클러스터 학습.
데이터 파이프라인 .shuffle().batch().prefetch(tf.data.AUTOTUNE) tf.data 최적화 패턴. AUTOTUNE은 런타임에 최적 버퍼 크기 결정.
혼합 정밀도 tf.keras.mixed_precision.set_global_policy('mixed_float16') GPU에서 메모리 감소, 처리량 증가 (Volta 아키텍처 이상).
XLA 컴파일 @tf.function(jit_compile=True) 또는 TF_XLA_FLAGS 환경 변수 Just-In-Time 컴파일로 연산 퓨전, 성능 향상 (특정 워크로드).
로그 레벨 os.environ['TF_CPP_MIN_LOG_LEVEL'] = '2' 0(모든 로그) ~ 3(에러만). 정보성 로그 억제.

6. 실무 팁 & 트러블슈팅

⚠️ 문제: GPU 메모리 부족(OOM) 에러

증상: ResourceExhaustedError: OOM when allocating tensor

해결책:

  1. 배치 크기 줄이기 (batch_size).
  2. 모델의 FP16 혼합 정밀도 사용: tf.keras.mixed_precision.set_global_policy('mixed_float16')
  3. GPU 메모리 증가 설정 활성화 (위 설정 참조).
  4. tf.data 파이프라인에 .cache() 또는 .prefetch() 사용으로 CPU-GPU 병목 해소.

💡 팁: 프로덕션을 위한 그래프 모드 최적화

즉시 실행 모드는 디버깅에 좋지만, 학습/추론 성능을 위해 @tf.function으로 그래프로 변환하세요.

@tf.function
def train_step(x_batch, y_batch):
    with tf.GradientTape() as tape:
        predictions = model(x_batch, training=True)
        loss = loss_fn(y_batch, predictions)
    gradients = tape.gradient(loss, model.trainable_variables)
    optimizer.apply_gradients(zip(gradients, model.trainable_variables))
    return loss
# 첫 호출 시 그래프 트레이싱 발생, 이후 호출은 컴파일된 그래프 실행.

🔍 문제: SavedModel 불러오기 실패 (서명 불일치)

증상: KeyError: '[serving_default] not found in signature def'

해결책: 모델 저장 시 서명을 명시적으로 정의하거나, 로드 시 서명 이름을 확인.

# 저장 시 서명 지정 (Custom Model)
@tf.function(input_signature=[tf.TensorSpec(shape=[None, 28, 28, 1], dtype=tf.float32)])
def serve(inputs):
    return {'predictions': model(inputs)}
tf.saved_model.save(model, 'path', signatures={'serving_default': serve})

# 로드 시 서명 확인
loaded = tf.saved_model.load('path')
print(list(loaded.signatures.keys()))  # ['serving_default']

🚀 팁: TensorFlow Serving으로 배포

# 1. Docker로 Serving 실행 (가장 일반적)
docker run -p 8501:8501 \
  --mount type=bind,source=/path/to/mnist_model,target=/models/mnist_model \
  -e MODEL_NAME=mnist_model -t tensorflow/serving

# 2. gRPC (포트 8500) 또는 REST (포트 8501) API로 추론 요청
# REST API 예시 (Python)
import requests
import json
data = json.dumps({"instances": x_test[:3].tolist()})
headers = {"content-type": "application/json"}
response = requests.post('http://localhost:8501/v1/models/mnist_model:predict',
                         data=data, headers=headers)
print(response.json())
