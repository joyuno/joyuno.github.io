1. Garak란?

Garak은 LLM(Large Language Model) 시스템을 대상으로 자동화된 보안 취약점 탐지 및 스캐닝을 수행하는 오픈소스 도구입니다. 생성형 AI 시스템에 존재할 수 있는 프롬프트 인젝션, 데이터 유출, 지시 무시, 편향 등 다양한 위험을 체계적으로 진단합니다.

탐지 능력

70개 이상의 프로브(Probe)를 통해 LLM 응답을 분석하여 취약점 패턴을 식별합니다. (예: DAN, 지시 무시, 맥락 확장)

자동화 스캔

CLI 또는 Python API로 손쉽게 스캔 작업을 구성하고 실행할 수 있으며, JSON, CSV 등 다양한 형식으로 리포트를 생성합니다.

확장성

새로운 프로브, LLM 백엔드(OpenAI, Anthropic, vLLM 등), 리포트 포맷을 플러그인 형태로 추가할 수 있습니다.

2. 왜 써야 하는가?

LLM 시스템 보안 검증을 위한 기존 수동 테스트나 일반화된 보안 도구의 한계를 극복합니다.

방법장점단점Garak 비교
수동 테스트도메인 지식 활용 가능시간 소모적, 비체계적, 재현성 낮음자동화, 체계적, 재현성 높음
정적 코드 분석초기 취약점 발견런타임 동작(LLM 응답) 검사 불가런타임 LLM 응답 직접 분석
기존 웹 취약점 스캐너인프라 레벨 검사LLM 특화 공격(프롬프트 인젝션 등) 탐지 불가LLM 레이어 특화 공격 탐지
베이스라인 벤치마크표준화된 비교실제 운영 환경의 복잡한 상호작용 반영 부족실제 API/엔드포인트에 대한 운영 테스트

3. 핵심 개념

Garak은 모듈식 아키텍처로 구성되어 있으며, 주요 컴포넌트는 다음과 같습니다.

+----------------+ +----------------+ +----------------+ | | | | | | | LLM Probe |---->| Detector |---->| Reporter | | (70+ 종류) | | (분석기) | | (보고서) | | | | | | | +----------------+ +----------------+ +----------------+ | | | v v v +----------------+ +----------------+ +----------------+ | | | | | | | LLM Backend | | Harness | | Output | | (OpenAI, | | (실행 환경) | | (JSON/CSV) | | Anthropic) | | | | | +----------------+ +----------------+ +----------------+
  • Probe: 특정 취약점(예: "지시 무시", "PII 유출")을 테스트하는 질문 또는 프롬프트 세트.
  • Detector: LLM 응답을 분석하여 취약점 존재 여부를 판단하는 로직(키워드 매칭, ML 분류 등).
  • Harness: 대상 LLM 시스템(로컬 모델, API 엔드포인트)과의 연결 및 테스트 실행을 관리.
  • Reporter: 스캔 결과를 정리하여 보고서(JSON, CSV, HTML)로 출력.

4. 퀵스타트

4.1 설치

CLI
pip install garak
Python (가상환경 권장)
python -m venv garak-env
source garak-env/bin/activate  # Linux/macOS
garak-env\Scripts\activate     # Windows
pip install --upgrade garak

4.2 기본 설정 (환경 변수)

# OpenAI API 사용 시
export OPENAI_API_KEY="sk-..."

# 또는 Anthropic, Cohere 등
export ANTHROPIC_API_KEY="..."

4.3 첫 번째 스캔 실행

CLI로 기본 스캔 (OpenAI GPT-4)
garak --model_type openai --model_name gpt-4 --probes all
Python API로 상세 설정
from garak import Garak
from garak.probes import PromptInjection, DAN

scanner = Garak(model_type="openai",
                model_name="gpt-3.5-turbo",
                probes=[PromptInjection, DAN],
                output_dir="./scan_results")

report = scanner.run()
print(f"스캔 완료. 발견된 취약점: {report['total_findings']}")
참고: --probes all은 모든 프로브를 실행하므로 시간과 토큰 비용이 많이 소요될 수 있습니다. 초기 테스트는 특정 프로브(--probes promptinject.Dan)로 시작하세요.

6. 실무 팁 & 트러블슈팅

주의: 프로덕션 LLM에 대한 대량 스캔은 비용 폭발(API 토큰) 및 서비스 부하를 유발할 수 있습니다. 스테이징 환경에서 제한된 프로브로 먼저 테스트하세요.

자주 겪는 문제

  • 문제: "ModuleNotFoundError: No module named 'garak'"
    해결: 가상환경이 활성화되었는지 확인. pip install garak 재실행.
  • 문제: API 오류 (429 Too Many Requests)
    해결: --parallel_requests 값을 줄이거나(예: 2), --request_timeout을 증가.
  • 문제: 특정 프로브에서 항상 실패(Fail)만 보고됨
    해결: 해당 프로브의 Detector가 너무 엄격할 수 있음. --detectors 옵션으로 다른 탐지기(예: similarity) 시도.
  • 문제: 커스텀 엔드포인트(로컬 vLLM 서버) 테스트 방법
    해결: --model_type huggingface와 --model_name http://localhost:8000/v1 사용. API 키는 --api_key ""로 설정 가능.

성능 최적화 팁

  • 프로브 필터링: garak --list-probes로 목록 확인 후, 필요한 카테고리(예: --probes injection, leakage)만 지정.
  • 샘플링: --max_probe_count와 --probe_sample로 테스트 수를 제한하여 빠른 피드백.
  • 로컬 캐싱: 동일한 모델/프로브 조합 재실행 시 --cache 옵션으로 이전 응답 재사용 가능.
