1. Darktrace란?

Darktrace는 Enterprise Immune System 개념을 기반으로 한 AI 기반 네트워크 감지 및 대응(NDR) 플랫폼입니다. 자가 학습(Self-Learning AI)을 통해 조직의 정상적인 '디지털 생활 패턴'을 이해하고, 이로부터 벗어나는 이상 행위를 실시간으로 탐지합니다.

자가 학습 AI

사전 정의된 시그니처 없이, 각 환경의 고유한 네트워크 패턴을 학습하여 정상 기준선(Baseline)을 형성합니다. 지도 학습이 아닌 비지도/자기지도 학습 방식을 사용합니다.

실시간 위협 탐지

네트워크 트래픽, 클라우드 활동, 엔드포인트 데이터, 이메일 등을 분석하여 랜섬웨어, 내부자 위협, 제로데이 공격, 무단 데이터 반출 등을 탐지합니다.

자동화된 대응(Antigena)

AI가 탐지한 위협에 대해 자동으로 대응 조치를 취할 수 있습니다. 예: 의심스러운 연결 차단, 파일 암호화 속도 제한, 명령 제어(C2) 통신 차단 등.

2. 왜 써야 하는가? (기존 대안과 비교)

솔루션 접근 방식 탐지 유형 AI 활용도 Darktrace 차별점
전통적 방화벽/IPS 시그니처 기반, 정책 기반 차단 알려진 위협 낮음 (규칙 엔진) 미지의 위협(Zero-day) 및 이상 행위 탐지 가능
SIEM (Splunk, QRadar) 로그 수집, 상관관계 분석 사후 분석, 규칙 기반 알림 중간 (머신러닝 추가 기능) 실시간 자가 학습, 네트워크 트래픽 직접 분석, 자동 대응
EDR (CrowdStrike, SentinelOne) 엔드포인트 행위 분석 엔드포인트 중심 위협 높음 (행위 AI) 네트워크 레벨의 상호작용 및 통신 패턴까지 포괄적 분석
Darktrace 자가 학습 AI, 정상 기준선 이상 탐지 알려지지 않은 위협, 미묘한 이상 행위 매우 높음 (핵심 엔진) 조직별 맞춤형 '면역 체계', 실시간 자동 대응(Antigena)

3. 핵심 개념

Darktrace의 핵심은 Cyber AI Loop입니다: 학습(Learn) → 탐지(Detect) → 조사(Investigate) → 대응(Respond) → 발전(Evolve)의 순환 구조입니다.

+---------------------+ +---------------------+ +---------------------+ | | | | | | | 네트워크/클라우드 |---->| Darktrace AI |---->| Antigena (응답) | | 데이터 수집 | | (자가 학습 엔진) | | (자동화된 조치) | | | | | | | +---------------------+ +---------------------+ +---------------------+ | | | v v v +---------------------+ +---------------------+ +---------------------+ | Raw Data | | 이상 점수 | | 조치 로그 | | (NetFlow, 패킷, | | (Threat Score) | | (차단, 경고, 속도 | | 로그, 엔드포인트) | | 모델 위반 | | 제한 등) | +---------------------+ +---------------------+ +---------------------+ | | +---------> 조사 콘솔 <------+ (사용자 인터페이스)

주요 모델 유형

  • 디바이스 모델: 각 디바이스(서버, 사용자 PC, IoT)의 행동 프로파일.
  • 사용자 모델: 사용자의 로그인, 리소스 접근, 데이터 이동 패턴.
  • 네트워크 연결 모델: 디바이스 간 통신 패턴 및 프로토콜 사용.
  • 클라우드 모델: AWS, Azure, GCP 내 리소스 구성 및 활동 패턴.
핵심 원리: "정상(Normal)"을 정의하는 것이 아니라, "새로움(New)"과 "희귀성(Rare)"을 기반으로 이상을 탐지합니다. 통계적 확률과 베이지안 추론을 활용합니다.

4. 퀵스타트 (PoC 환경 구축)

개발/테스트 환경에서 Darktrace API를 활용한 기본 통합 예제입니다.

# 기본 API 클라이언트 클래스
import requests
import json
import hashlib
import hmac
import time
import pandas as pd

class DarktraceClient:
    def __init__(self, base_url, public_key, private_key):
        self.base_url = base_url.rstrip('/')
        self.public_key = public_key
        self.private_key = private_key

    def _generate_signature(self, endpoint, params=None, date=None, method='GET'):
        if date is None:
            date = time.strftime('%a, %d %b %Y %H:%M:%S GMT', time.gmtime())
        string_to_sign = f"{method}\n{endpoint}"
        if params:
            # 간소화: 실제로는 정렬된 쿼리 문자열 필요
            pass
        signature = hmac.new(
            self.private_key.encode('utf-8'),
            string_to_sign.encode('utf-8'),
            hashlib.sha1
        ).hexdigest()
        return date, signature

    def get(self, endpoint, params=None):
        date, signature = self._generate_signature(endpoint, params)
        headers = {
            'DTAPI-Token': self.public_key,
            'DTAPI-Date': date,
            'DTAPI-Signature': signature
        }
        response = requests.get(f"{self.base_url}{endpoint}", headers=headers, params=params)
        response.raise_for_status()
        return response.json()

# 클라이언트 초기화
client = DarktraceClient(
    base_url=os.environ['DARKTRACE_BASE_URL'],
    public_key=os.environ['DARKTRACE_PUBLIC_KEY'],
    private_key=os.environ['DARKTRACE_PRIVATE_KEY']
)

Step 3: 첫 번째 탐지 데이터 가져오기

# 최근 1시간 내의 이상 점수가 높은 디바이스 조회
def fetch_top_anomalous_devices(hours=1):
    from datetime import datetime, timedelta
    end_time = datetime.utcnow()
    start_time = end_time - timedelta(hours=hours)
    params = {
        'starttime': start_time.isoformat() + 'Z',
        'endtime': end_time.isoformat() + 'Z',
        'order': 'desc',
        'sortby': 'score',
        'maxresults': 10
    }
    try:
        data = client.get('/devices', params=params)
        df = pd.DataFrame(data)
        print(f"탐지된 상위 {len(df)}개 디바이스:")
        print(df[['label', 'score', 'ip', 'devicetype']].to_string())
        return df
    except Exception as e:
        print(f"API 호출 실패: {e}")
        return None

# 실행
devices_df = fetch_top_anomalous_devices()

Step 4: 모델 위반(Mode Breach) 이벤트 구독 (웹훅 예제)

# Flask를 이용한 간단한 웹훅 수신기 (개발용)
from flask import Flask, request, jsonify
app = Flask(__name__)

@app.route('/darktrace-webhook', methods=['POST'])
def handle_breach():
    data = request.json
    # 보안: Darktrace 서명 검증 로직 추가 필요
    breach_id = data.get('bid')
    score = data.get('score')
    device_label = data.get('deviceLabel', 'Unknown')
    model_name = data.get('model', {}).get('name', 'Unknown Model')
    print(f"[ALERT] 모델 위반 감지 | ID: {breach_id} | 점수: {score}")
    print(f"       디바이스: {device_label} | 모델: {model_name}")
    # 여기에 Slack, 이메일, SIEM 전송 로직 추가
    return jsonify({'status': 'received'}), 200

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)

외부 통합 SIEM/SOAR 연동 CEF/LEEF 포맷, Syslog 또는 API 웹훅 /integrations/siem?format=cef&target=syslog-server:514
API 보안 토큰 권한 범위 최소 권한 원칙: 읽기 전용 토큰과 쓰기/조치 토큰 분리 관리 콘솔 UI에서 구성
모델 튜닝 Exception List 정상적인 비즈니스 활동(예: 백업 서버)에 대한 예외 추가 POST /models/exceptions {"deviceid": "xyz", "reason": "Backup Server"}

6. 실무 팁 & 트러블슈팅

문제: 너무 많은 False Positive 알림

원인: 학습 기간 부족, 네트워크 변화(신규 서비스 도입), 예외 설정 미비.

해결책:

  • 학습 기간을 2-4주로 확보.
  • 정상적인 주기적 활동(배치 작업, 백업)에 대해 예외(Exceptions) 등록.
  • 모델의 '민감도(Sensitivity)' 설정을 일시적으로 낮춤.

문제: AI가 중요한 위협을 놓침

원인: 데이터 수집 불완전(특정 세그먼트 미수집), 암호화 트래픽 분석 한계.

해결책:

  • 모든 네트워크 세그먼트에서 NetFlow/패킷 데이터가 전송되는지 확인.
  • 엔드포인트(EDR) 및 클라우드 로그 통합을 추가하여 보완.
  • 위협 인텔리전스 피드와의 상관관계 분석 설정.

문제: Antigena 자동 조치가 비즈니스 중단 유발

원인: 과도한 공격성(Active-High) 설정, 중요 시스템에 대한 예외 미설정.

해결책:

  • 중요 서버/대상 그룹에 대해 'Passive Mode' 또는 'Active-Low'로 시작.
  • Antigena 네트워크 세그먼트를 나누어 단계적으로 적용.
  • 조치 전 '승인 대기(Hold for Approval)' 모드 활용.

팁: 효과적인 모니터링 대시보드

  • 핵심 지표: 총 모델 위반 수, 평균 이상 점수, Antigena 조치 수.
  • 집중 조사: 점수가 급상승한 디바이스, 새로운 외부 연결 도메인.
  • 통합: Darktrace 알림을 SIEM 대시보드에 임베드하여 단일 창구 구축.
  • 정기 검토: 주간 False Positive 리포트를 통해 모델/예외 지속적 개선.
성공 키포인트: Darktrace는 '설치 후 망각' 솔루션이 아닙니다. 초기 학습과 지속적인 예외 관리를 통해 정확도를 높이고, 보안 팀의 역량을 증폭시키는 도구로 활용해야 합니다.
