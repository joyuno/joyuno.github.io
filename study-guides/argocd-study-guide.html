1. ArgoCD란?

ArgoCD는 쿠버네티스 환경에서 GitOps 접근법을 구현하기 위한 선언적(Declarative), GitOps 지속적 딜리버리(Continuous Delivery) 도구입니다.

정의

Git 저장소를 "단 진리 소스 오브 트루스(Single Source of Truth)으로 간주하여,쿠버네티스 클러스터의 상태를 지속적으 로 동기화하고 원하는 상태를 유지합니다.

핵심 능력

  • Git 저장소와의 상태 동기
  • 자동 동기/롤백/롤
  • 헬치 다시 보드/웹 보드
  • 다중 클러스터 관라
  • RBAC, SSO 통합

2. 왜 써야 하나요?

기존의 CI/CD 파이프라인과 수동 kubectl 적용 방식의 한계를 해결합니다.

비교 항목 기존 방식 (CI/CD + kubectl) ArgoCD (GitOps)
배포 주기 CI 파이프라인 트리거 시 (주로 수동) Git 커밋 시 자동 (선언적 상태 동기)
관리 포인트 다중 (여러 클러스터/네임스페이스) 중앙 집중식 웹 UI, CLI
드리프트 감지 수동 감지/대응 자동 감지 및 시각화, 자동 복구 옵션
롤백 복잡한 수동 절차 Git 리비전 간 단계 롤백
보안/감사 개별 툴/스크립트 의존 Git 커밋 기반 감사 로그

3. 핵심 개념

ArgoCD 아키텍처의 기본 구성 요소와 데이터 흐름을 이해합니다.

                 +-----------------+
                 |   Git 저장소    |
                 | (선언적 매니페스트) |
                 +--------+--------+
                          |
                          | (Push/Pull)
                          v
                 +--------+--------+
                 |   ArgoCD 서버    |
                 | (컨트롤러) |
                 +--------+--------+
                          |
                          | (쿠버네티스 API)
                          v
                 +-----------------+
                 | 버네티스 클러스터 |
                 | (실제 상태) |
                 +-----------------+

Application

배포할 애플리케이션의 논리적 단위. Git 저장소의 특정 경로 (소스)와 쿠버네티스 클러스터 내 대상 (대상)을 연결합니다.

AppProject

애플리케이션의 논리 그룹. RBAC, 소스/대상 제한, 정책을 정의합니다.

Sync Policy

동기 동작 방식을 정의 (자동/수동, 수동 동기 시 자동 동기/병합 등).

4. 시작하기

CLI 명령어를 통한 빠른 시작 가이드.

1. ArgoCD 설치 (쿠버네티스 클러스터 내)

kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

2. CLI 도구 설치 (로컬)

# 맥OS
brew install argocd

# 리눅스 (예: Ubuntu)
curl -sSL -o /usr/local/bin/argocd https://github.com/argoproj/argo-cd/releases/latest/download/argocd-linux-amd64
chmod +x /usr/local/bin/argocd

주의: 설치 후 초기 비밀번호는 ArgoCD 서버 포드 로그에서 확인하거나 kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 --d 명령어로 얻을 수 있습니다.

3. 웹 UI 접속

# 포트 포워딩
kubectl port-forward svc/argocd-server -n argocd 8080:443

# 브라우저에서 접속: https://localhost:8080
# 사용자: admin, 비밀번호: 위에서 확인한 초기 비밀번호

5. 핵심 설정

애플리케이션을 정의하고 관리하는 주요 설정들입니다.

설정 설명 예시/기본값
source.repoURL 매니페스트가 위치한 Git 저장소 URL https://github.com/my-org/my-app.git
source.targetRevision 동기할 Git 브랜치/태그/커밋 해시 main, HEAD
destination.server 대상 쿠버네티스 클러스터 API 서버 주소 https://kubernetes.default.svc
destination.namespace 대상 쿠버네티스 네임스페이스 default
syncPolicy.automated 자동 동기 여부 true (권장)
syncPolicy.automated.prune Git에 정의되지 않은 리소스 자동 삭제 여부 false (주의 요구)

애플리케이션 생성 예시 (YAML)

apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: my-nginx-app
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/my-org/my-nginx-conf.git
    path: k8manifests/
    targetRevision: HEAD
  destination:
    server: https://kubernetes.default.svc
    namespace: production
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
    - CreateNamespace=true

6. 실무 팁 & 트러블슈팅

자주 겪는 문제 + 해결책

문제: Application이 "OutOfSync" 상태지만 Sync 실패.

원인/해결: 매니페스트 오류, k8bectl apply --dry-run=client -f .

헬름 차트 연동

팁: 헬름 차트를 사용할 경우 source.helm 섹션을 사용하고, valueFiles로 환경별 값을 관리하세요.

대형 배포 관리

팁: AppProject와 애플리케이션 세트를 활용하여 팀/환경별로 논리적 격리와 권한 관리를 구현하세요.

헬스 체크 커스터마이즈

팁: ArgoCD의 헬치 다시 보드는 버네티스의 Liveiness/Readiness 프로브에 의존합니다. 정확한 헬치를 위해 커스텀 헬스 스크립트를 고려하세요.

주의: Sync 실패 시 즉시 "자동 복구 (selfHeal)를 시도하지 마세요. 먼저 Git 저장소의 매니페스트 오류를 확인하고,수동 Sync"를 통해 신중히 조치하세요.
