1. MITRE ATT&CK Framework란?

MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge)는 사이버 공격자의 행위(Tactics, Techniques, Procedures - TTPs)를 체계적으로 분류한 지식 베이스이자 프레임워크입니다. 실제 관찰된 공격 데이터를 기반으로 공격 라이프사이클을 기술하며, 방어 측의 위협 모델링, 탐지 및 대응 능력을 강화하는 데 목적이 있습니다.

📊 지식 베이스 (Knowledge Base)

공격자의 전술(Tactics), 기술(Techniques), 절차(Procedures)에 대한 상세 설명, 예제, 탐지 및 대응 방안을 포함한 구조화된 데이터셋.

🛡️ 평가 프레임워크 (Evaluation Framework)

보안 제품의 성능을 ATT&CK 매트릭스 기반으로 평가하고 벤치마킹하는 표준 방법론을 제공 (예: MITRE Engenuity).

🔬 공격 에뮬레이션 (Adversary Emulation)

특정 공격 그룹(APT)의 TTPs를 재현하는 레드팀 플랜 수립 및 방어 체계 검증에 활용.

2. 왜 써야 하는가? 기존 대안과 비교

기존의 보안 프레임워크(예: Kill Chain)와 비교했을 때 ATT&CK의 가장 큰 차별점은 실제 관찰 가능한 행위(Techniques)에 초점을 맞추고 있으며, 상세한 기술 설명과 탐지 로직을 제공한다는 점입니다.

프레임워크 초점 강점 ATT&CK 대비 한계
Lockheed Martin Kill Chain 공격 단계(7단계)의 선형 진행 공격 흐름의 이해, 단계별 방어점 식별 너무 고수준, 구체적 기술 및 탐지 방법 부재
STRIDE 위협 모델링(6가지 위협 유형) 디자인 단계의 체계적 위협 식별 실제 공격 행위 매핑 어려움, 실행 단계 부족
NIST CSF 보안 관리 프로세스(Identify, Protect 등) 거버넌스, 위험 관리, 컴플라이언스 공격자 관점의 구체적 기술 정보 부재
MITRE ATT&CK 공격자의 실제 Tactics & Techniques 실행 가능한 탐지/대응 로직, 공격 그룹 TTPs 매핑, 풍부한 데이터 학습 곡선, 방대한 데이터 관리 필요

💡 AI 엔지니어에게 ATT&CK가 중요한 이유

ATT&CK는 레이블링된 공격 시나리오 데이터를 제공하여 지도 학습 기반 이상 탐지 모델의 훈련 데이터 구축에 활용될 수 있습니다. 또한, 특정 Technique(예: T1059.003 - Windows Command Shell)에 대한 탐지 로직을 자동 생성하는 AI 모델의 기준 지식으로 사용 가능합니다.

3. 핵심 개념: 매트릭스, Tactic, Technique, Sub-technique

ATT&CK의 계층적 구조를 이해하는 것이 실무 적용의 첫걸음입니다.

┌─────────────────────────────────────────────────────────────────────────┐ │ MITRE ATT&CK ENTERPRISE │ ├─────────────────────────────────────────────────────────────────────────┤ │ │ │ TACTICS (공격자의 전술적 목표 - "Why") │ │ ┌──────────┬──────────┬──────────┬──────────┬──────────┐ │ │ │Recon │Resource │Initial │Execution │Persistence│ ... (14개) │ │ │(정찰) │Development│Access │(실행) │(지속성) │ │ │ │TA0043 │(자원개발)│(초기침입)│TA0002 │TA0003 │ │ │ └──────────┴──────────┴──────────┴──────────┴──────────┘ │ │ │ │ │ │ │ │ │ TECHNIQUES (목표를 달성하는 구체적 방법 - "How") │ │ ┌──────────┬──────────┬──────────┬──────────┬──────────┐ │ │ │Active │Phishing │Exploit │Command │Registry │ ... (200+) │ │ │Scanning │(피싱) │Public- │& Scripting│Run Keys │ │ │ │T1595 │T1566 │Facing App│Interpreter│T1547.001 │ │ │ └──────────┴──────────┴──────────┴──────────┴──────────┘ │ │ │ │ │ SUB-TECHNIQUES (기술의 세부 변종 - "How in detail") │ │ ├─ T1547.001: Registry Run Keys / Startup │ │ ├─ T1547.002: Authentication Package │ │ └─ T1547.003: Time Providers │ │ │ │ PROCEDURES (실제 관찰된 구현 사례 - "How in reality") │ │ └─ APT29: "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion│ │ \Run"에 "rundll32.exe stager.dll" 추가 │ └─────────────────────────────────────────────────────────────────────────┘

주요 구성 요소

  • Tactic (TAxxxx): 공격자의 단기적 전술 목표 (예: TA0001 - Initial Access).
  • Technique (Txxxx): Tactic을 달성하기 위한 구체적 방법 (예: T1190 - Exploit Public-Facing Application).
  • Sub-technique (Txxxx.yyy): Technique의 세부 구현 (예: T1059.003 - Windows Command Shell).
  • Procedure: 실제 공격 그룹이 사용한 구체적 도구, 명령어, 스크립트.
  • Mitigation (Mxxxx): 해당 Technique을 방지하거나 어렵게 하는 대응책.
  • Detection (DSxxxx): Technique을 탐지하기 위한 데이터 소스 및 분석 로직.

4. 퀵스타트: Python & CLI로 시작하기

공식 Python 라이브러리(`attackcti`)와 STIX/TAXII를 활용하여 ATT&CK 데이터를 프로그래밍 방식으로 조회하고 분석할 수 있습니다.

4.1 환경 설정 및 라이브러리 설치

Terminal
# 필수 라이브러리 설치
pip install attackcti pandas jupyter
# 또는 컨테이너 환경에서
docker run -it --rm -p 8888:8888 -v $(pwd):/workspace python:3.11-slim bash -c "pip install attackcti pandas && jupyter notebook --ip=0.0.0.0 --allow-root"

4.2 첫 번째 예제: 모든 Enterprise Technique 조회

Python (attackcti)
from attackcti import attack_client
import pandas as pd

# ATT&CK STIX 데이터 로드 (로컬 캐시 또는 TAXII 서버)
lift = attack_client()

# 모든 Enterprise Technique 가져오기
techniques = lift.get_techniques()
print(f"총 Techniques 수: {len(techniques)}")

# DataFrame으로 변환하여 주요 정보 확인
tech_list = []
for t in techniques:
    tech_list.append({
        'ID': t['external_references'][0]['external_id'],
        'Name': t['name'],
        'Tactic': ', '.join([phase['phase_name'] for phase in t['kill_chain_phases']]),
        'Description': t['description'][:150] + '...' if t['description'] else ''
    })

df_tech = pd.DataFrame(tech_list)
print(df_tech.head())

# 특정 Tactic(예: Persistence)의 Technique 필터링
persistence_techs = df_tech[df_tech['Tactic'].str.contains('persistence', case=False)]
print(f"\nPersistence Tactic 관련 Techniques: {len(persistence_techs)}개")

4.3 CLI를 통한 빠른 조회 (mitre-attack CLI 도구)

Terminal
# mitre-attack CLI 설치 (Node.js 기반)
npm install -g @mitre/attack-cli

# 기본 조회
attack search "credential dumping"
# 특정 그룹(APT29)의 Techniques 조회
attack group "APT29" --techniques
# Technique 상세 정보 (T1055 - Process Injection)
attack technique T1055 --details --subtechniques

5. 핵심 설정 레퍼런스

ATT&CK를 실무에 통합할 때 주요하게 설정해야 하는 요소들입니다.

설정 항목 권장값 / 옵션 설명 적용 예시
데이터 소스 Enterprise (기본), ICS, Mobile 분석할 환경에 맞는 매트릭스 선택 기업 내부망 → Enterprise, OT 환경 → ICS
STIX/TAXII 서버 https://cti-taxii.mitre.org 공식 실시간 데이터 피드 엔드포인트 attack_client(latest=True)로 최신 데이터 동기화
로컬 캐시 주기 24시간 (기본), 7일 (낮은 빈도) 네트워크 부하 감소를 위한 캐싱 정책 스케줄러로 매일 02:00에 데이터 갱신
매핑 필드 external_id, name, kill_chain_phases 로그/알람과 ATT&CK를 매핑할 키 필드 SIEM에서 alert.technique_id = T1059.003
AI 모델 입력 Technique 설명, 탐지 예제, 절차 ML 모델의 특징(feature) 또는 레이블 소스 텍스트 임베딩 → Technique 유사도 분석
시각화 도구 ATT&CK Navigator, Attack Flow 매트릭스 시각화 및 공격 시나리오 작성 JSON Layer 파일로 커버리지 히트맵 생성

6. 실무 팁 & 트러블슈팅

⚠️ 자주 겪는 문제 1: 데이터 양이 너무 방대하다

증상: 200개가 넘는 Techniques를 모두 모니터링하려 하니 리소스 부족과 피로도 상승.
해결책: 조직의 위협 모델과 자산 중요도에 기반한 Technique 우선순위화 수행. MITRE의 ATT&CK Evaluations 결과를 참고하여 실제 탐지율이 낮은 Techniques에 집중.

💡 팁 1: AI/ML 파이프라인에 ATT&CK 통합하기

ATT&CK Technique 설명과 절차 데이터를 NLP 모델(BERT, SentenceTransformer)로 임베딩하여, 신규 로그/이벤트와의 유사도를 계산. 이를 통해 미리 정의되지 않은 변종 공격(Variants) 탐지에 활용.

# 간단한 유사도 매칭 예시 (의사 코드)
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('paraphrase-MiniLM-L6-v2')
# ATT&CK Technique 설명 임베딩
tech_descriptions = {t.id: t.description for t in techniques}
tech_embeddings = model.encode(list(tech_descriptions.values()))

# 새 이벤트 로그 임베딩
new_event_embedding = model.encode("PowerShell encoded command execution observed")
# 코사인 유사도 계산 → 가장 가까운 Technique ID 반환

⚠️ 자주 겪는 문제 2: 탐지 로직과 실제 Technique 매핑이 모호하다

증상: 하나의 탐지 규칙이 여러 Techniques에 걸쳐 있어 정확한 보고가 어려움.
해결책: 탐지 규칙을 가장 구체적인 Sub-technique 수준으로 분해. 예: "스크립트 실행" → "T1059.001 PowerShell", "T1059.003 Windows Command Shell", "T1059.004 Unix Shell"로 세분화.

💡 팁 2: ATT&CK Navigator 레이어로 커버리지 시각화

보유한 탐지 규칙, EDR 알람, 레드팀 테스트 결과를 ATT&CK Navigator Layer JSON으로 변환하여 시각적 격차 분석 수행. 이를 통해 방어의 빈틈을 명확히 식별하고 투자 우선순위 결정.

# Navigator Layer 생성 예시 (Python dict)
layer = {
    "name": "Our Detection Coverage",
    "versions": {"attack": "13", "navigator": "4.8.0"},
    "techniques": [
        {
            "techniqueID": "T1059.003",
            "score": 85,  # 탐지 커버리지 %
            "color": "#3fb950",  # 초록색
            "comment": "Detected via process command-line monitoring"
        },
        {
            "techniqueID": "T1566.001",
            "score": 40,
            "color": "#f0883e",  # 주황색
            "comment": "Partial coverage via email gateway"
        }
    ]
}
# JSON 파일로 저장 후 https://mitre-attack.github.io/attack-navigator 에 업로드
